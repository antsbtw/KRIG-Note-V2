为对齐范式不写。代码注释已涵盖说明。

- **W5-A View 边界全程合规**:DictionaryPanel + learning-integration + help-panel-registrations
  没有 view 直 import `@drivers/*`(text-editing 走 capability 间接);

- **Q1-Q8 决策全 A 默认落实**:

  - Q2 = A 模块单例 controller(对齐 popupController)

  - Q6 = A 删自渲染 close,用 shell 提供的 ×
  - Q7 = A popup vs help-panel 两套独立(允许同时打开)
  - Q8 = A 全局单例(对齐 popupController);per-workspace 隔离作单独 stage 全部 4 个 controller 一并迁

## 3. 用户验收清单(请按顺序点检)

### 3.1 启动 + 健康检查

1. `npm start` → 控制台无报错
2. F12 console 找 `[L4] alive | ...` 日志,应包含:
   - `helpPanel: 1`(NEW,新增的 7 号 interaction-registry)
   - `popup: 3`(顺手补的;link / color / note-link 三个,dictionary 已迁出)
   - 其他 6 号(contextMenu / slash / handle / floatingToolbar / overlay / popup)+ navSide / toolbar / keymap 仍在

### 3.2 dictionary 形态 — 长侧栏(L4.1 关键差异)

3. 打开任一笔记,输入英文段落 `The dictionary panel should appear on the right side.`
4. 双击选中 `dictionary` → 右键菜单仍然有 **📖 查词** + **🌐 翻译**
5. 点 **📖 查词** →
   - **关键差异**:面板从**屏幕右侧 360px 宽**长条滑出(顶到底,不再是选区下方的小卡)
   - Header 显 "📖 词典" + 右上 × 关闭按钮
   - Body 显 lookup 内容(词头 + 音标 + 🔊 + 释义 + 翻译 + 加生词本)
6. 点 × → 面板关闭
7. 重开 → Esc → 关闭
8. 重开 → 点面板外区域(笔记区域)→ 关闭

### 3.3 popup vs help-panel 两套独立

9. 选段文字 → Cmd+K(Mod+k)→ 弹 LinkPanel(popup)
10. 不关 LinkPanel,在另一处选词 → 右键 → 📖 查词 → dictionary help-panel 滑出
11. **预期两个同时显示**(popup 在选区下方,help-panel 在右侧;互不互斥)
12. 点屏幕中间 → LinkPanel 先关(它先消化点击)/ help-panel 关 — 两个独立池

### 3.4 互斥(本池内)

13. dictionary help-panel 显示中 → 不需要现在能验,因为 V2 还只有 dictionary 一个 help-panel。
   未来 latex/mermaid 加入时该项才能验。

### 3.5 B3.20b 复测(行为应 100% 不变)

15. TTS:点 🔊 播放 ✅

### 3.6 跨 workspace 互斥(Q8=A 行为验证)

21. 打开 dictionary help-panel,显示中 → 切到另一 workspace tab → 面板**消失**(因为 help-panel
