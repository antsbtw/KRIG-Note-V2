## 4. 已知约束

### 4.1 macOS-only 字典(决策 Q5=A)

非 macOS 用户调 `dictionaryLookup` → swift CLI 失败 → 静默 fallback Google
Translate(返 zh-CN 翻译作为"释义")。质量较 macOS 系统词典低。
跨平台 provider(StarDict / Wiktionary / etc.)留 Phase D。

### 4.2 Google 反爬(translate / TTS 风险)

translate.googleapis.com / translate.google.com 是免费端点,某些 IP 段会被限流
(429 / 403)→ 静默返 null。UI 层(B3.20b dictionary-panel + B3.19 video-block 翻译字幕)
做"翻译失败"提示。长期方案接 LLM 翻译留 Phase D。

### 4.3 vocab.json 规模

几千条以下毫秒级 IO 无瓶颈。规模超 5000 时:
- 写入逐渐变慢(全量序列化整 JSON)
- 内存 Map 占用增大(每个 entry ~200 字节)
- 留 Phase E:分片或迁 SQLite / 等

### 4.4 没有 vocab 导出 / 导入 UI

JSON 文件用户可手动备份,但无应用内 UI。Phase D 加(预期跟设置面板一起)。

### 4.5 TTS 大文本拼接慢

splitTextForTTS 按 180 字符切块,每块串行 fetch。1000+ 字符场景几秒延迟。
B3.20b 在 dictionary-panel TTS 按钮加 loading 状态(本段不优化)。
