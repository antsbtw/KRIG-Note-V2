#### main 侧(分层方向遵守 — main 不反向 import capability,P1 修正)

V2 既有约定(见 `src/platform/main/preload/main-window-preload.ts:131` `onYtdlpDownloadProgress` 用 `unknown` 而非 `@capabilities` 类型 / `electron-api.d.ts` 内联 shape):
**main 层 / preload / electron-api / capability 三方都对结构兼容,任一方不跨边界 import 类型**。本段沿用此模式。

`src/platform/main/ytdlp/fetch-transcript.ts`(NEW):
```typescript
import { fetchTranscript as ytFetchTranscript } from 'youtube-transcript';

// main 层本地定义返回类型(不反向 import capability;capability 侧自己定义结构兼容的 FetchTranscriptResult)
interface FetchTranscriptOutput {
  transcriptText: string | null;
  error: string | null;
}

export async function fetchYouTubeTranscript(url: string): Promise<FetchTranscriptOutput> {
  try {
    const segments = await ytFetchTranscript(url);
    if (!segments || segments.length === 0) {
      return { transcriptText: null, error: '字幕不可用(视频未提供字幕)' };
    }
    const transcriptText = segments
      .map((seg: { text: string; offset: number }) => {
        const s = Math.floor(seg.offset / 1000);
        const mm = String(Math.floor(s / 60)).padStart(2, '0');
        const ss = String(s % 60).padStart(2, '0');
        return `[${mm}:${ss}] ${seg.text}`;
      })
      .join('\n');
    return { transcriptText, error: null };
  } catch (e) {
    const msg = e instanceof Error ? e.message : String(e);
    return { transcriptText: null, error: msg || '抓取字幕失败' };
  }
}
```

**结构契约**(三方对齐):
- main 返 `{ transcriptText: string | null; error: string | null }`(本地 `FetchTranscriptOutput`)
- preload 透传 `Promise<unknown>` 或重声明等价 inline shape
- `electron-api.d.ts` 内联 inline shape 给 renderer 端类型检查
- capability `types.ts` 自己声明 `FetchTranscriptResult`(同 shape,renderer 端用)

三处独立声明,结构兼容,**不跨层 import**。

`src/platform/main/ytdlp/handlers.ts` 加 ipcMain.handle:

ipcMain.handle(IPC_CHANNELS.YTDLP_FETCH_TRANSCRIPT, async (_e, url: string) => {

export async function fetchTranscript(url: string): Promise<FetchTranscriptResult> {
  if (!window.electronAPI?.ytdlpFetchTranscript) {

## 6. 风险

### 6.1 youtube-transcript 失败模式
videos 禁字幕 / 字幕语言不匹配 / API rate limit 等。**fetchYouTubeTranscript 全部捕获**,返 `{transcriptText: null, error: 详情}`。

### 6.2 translate batch 失败处理
某 batch 失败时,V1 直接把 batch 原文塞回结果(降级)。本段沿用 — 用户拿到部分翻译总比全部失败好。

### 6.3 翻译 Tab 数量无上限
用户反复切语言,Tab 越来越多。V1 也无上限,**沿用**。如真出问题加最大数量限制(e.g. 5 个)再说。

### 6.4 downloader.ts 与新独立 handler 行为对齐
downloader.ts 内部抓字幕逻辑跟新 fetchYouTubeTranscript **不共用**(Qb-8=A)。两条路径功能等价但代码重复 ~10 行。**接受重复**(改动安全,降低 risk)。
