## 3. 协议核心(PROTOCOL.md 摘要)

### 三类管道

| 管道 | 形态 | 用途 |
|---|---|---|
| **Channel** | emit / subscribe,有 lastValue | view 状态广播(锚定场景) |
| **Capability Request** | request → Promise<Result<T>>,1 handler | 调用别 view 能力(触发/发送) |
| **Slot Control** | bus.openRight / closeRight / closeLeft 内置 API | 改容器(只 right 可改;left 只能 close 触发升级) |

### 9 条铁律

1. 三类管道,各司其职(不混用)
2. Workspace scope only(跨 Workspace 用 IPC)
3. dev mode typeof 校验(prod 0 开销)
4. Manifest 分散到各 view/capability(`channels.ts`)
5. 主 view 锁:bus 不提供 openLeft;`slot.left.*` handler 拒绝
6. Slot Control 是框架级保留指令;`slot.*` 前缀禁注册
7. left 关闭时 right→left 升级 + view 实例不重建
8. 最后一个 view 不可关:Workspace 必有 left view
9. NavSide 切主 view 自动关 right(承袭 V1 契约)

详见 [PROTOCOL.md](../../../src/slot/workspace-bus/PROTOCOL.md)。

## 5. 阶段中遇到 / 解决的问题

### 5.1 Bug:viewTypeRegistry.getAll() useSyncExternalStore 死循环

**症状**:启动后整个 React 树空白(标题栏在,内部都没渲染),console 看到所有 alive 但 UI 不出。

**根因**:SlotArea 用 useSyncExternalStore 订阅 `viewTypeRegistry.getAll()`,每次返回 `Array.from(map.values())` 新数组,React getSnapshot `===` 比较失败 → 重渲 → 又调 getAll → 又新数组 → Maximum update depth exceeded → React 整树 unmount。

这是 L3 WorkspaceManager.getAll() 同款 bug,L4 / L3.5 又踩。

**修复**(commit `6822f8d`):
- viewTypeRegistry 加 `cachedAll: ViewDefinition[] | null`
- `getAll()` 返回缓存数组(数据未变同一引用)
- `notify()` 失效缓存

**教训**:任何 useSyncExternalStore 订阅的 getter 都必须返回稳定引用。Map.get 天然稳定,但 Array.from(map.values()) 必须缓存。下次新建 Registry 默认配 cached + notify 失效模式。

### 5.2 设计调整:三类管道 vs 两类(用户洞察)

文档 v0.1 草稿原计划两类(Channel + Request),用户提出"slot.openRight 类操作不该让 view 注册 handler 改写"——容器控制是框架级保留指令,与能力调用本质不同。

**调整**:协议正式分**三类**(加 Slot Control 独立类),内置 API 不通过 request 走,view 不能注册 `slot.*` 前缀 handler。

修订理由文档化在 PROTOCOL.md § 1。

### 5.3 设计调整:left 关闭场景(用户洞察)

文档 v0.1 草稿"left 永远不变"过于简化。用户提出:用户主动关 left 时,right 应升级成 left。

**调整**:加铁律 7(right→left 升级)+ 铁律 8(最后一个 view 不可关,UI 灰显)。

实施细节:Grid 布局 + 扁平视图列表,left/right 切换由 grid-column 控制,React 实例不重建。

### 5.4 设计调整:Workspace 不允许全空(用户洞察)

文档 v0.1 草稿场景 C 允许"空 Workspace 显示占位"。用户提出:像浏览器关最后一个 tab 后必有新空白页一样,Workspace 工作位不该全空。

**调整**:铁律 8 实施 — `closeLeft()` 在 right=null 时返回 fail,UI 按钮 disabled。
