## 0. 上下文

### 0.1 V1 download 状态机(对齐迁)

```
idle ─(点击)─> [check ytdlp installed]
              ├─ 否 ─> installing ─(成功)─> idle ─(再点)─> downloading
              │                  └(失败)─> idle + ❌ 2s
              └─ 是 ─> downloading ─(complete)─> done(显 📁)
                              └(error)─> idle + ❌ 2s

done + 点击 ─> showItemInFolder(localFilePath)
```

完成时:
- localFilePath = result.filename(写 attrs)
- 若 result.subtitleText 且 transcriptText 为空 → 自动填入 transcriptText
- 若 attrs.translationTexts 有翻译 → 遍历调 ytdlp.saveSubtitle 落 .srt

### 2.2 actions/download-button.ts

```typescript
export interface DownloadButton {
  el: HTMLButtonElement;
  destroy(): void;
}

export function createDownloadButton(deps: {
  /** 取当前 src */
  getSrc: () => string | null;
  /** 取已下载本地路径(从 attrs.localFilePath)*/
  getLocalFilePath: () => string | null;
  /** 取当前 transcriptText */
  getTranscriptText: () => string | null;
  /** 取 translationTexts(JSON 解析后的 map)*/
  getTranslations: () => Record<string, string>;
  /** 写 localFilePath / 可选 transcriptText 到 attrs */
  onUpdateAttrs: (patch: { localFilePath?: string | null; transcriptText?: string | null }) => void;
  /** 设置进度条可见性 + 百分比(交给 progress-bar)*/
  onProgress: (visible: boolean, percent: number) => void;
}): DownloadButton;
```

行为:
- 初始 disabled,checkStatus 异步完成后 enable;tooltip 根据 installed 显 "Download" / "Click to install yt-dlp"
- 点击:
  - 未装 → install();成功 enable + 状态切回 idle;失败 ❌ 2s
  - 已 done → showItemInFolder(localFilePath)
  - downloading → 忽略
  - idle → start download
- 下载中:订阅 onDownloadProgress 推 onProgress;done 后 attrs.localFilePath 写 + 按钮变 📁
- 完成时:若 result.subtitleText 且 transcriptText 为空 → onUpdateAttrs 写 transcriptText
- 完成时:若 translations 非空 → 遍历调 saveSubtitle 落 .srt(失败静默)
- 错误:❌ 2s 后回 ⬇️

**仅 YouTube 源启用 ytdlp 路径**;direct/vimeo/generic 源禁用此按钮(direct 用 B3.16 内置下载按钮,vimeo/generic 占位无下载需求)。

## 6. 风险

### 6.1 onDownloadProgress 跨 video-block 实例
所有 video-block 实例都订阅 `ytdlp.onDownloadProgress`,但回调里要 `progress.url === src` 过滤(对齐 V1 already)。多实例同时下载不冲突 — 各自按 url 过滤事件。

### 6.2 install 期间用户切笔记
导致 download-button destroy 但 install promise 还在跑。**接受**:install 是 main 进程任务,完成后写入二进制,下次 checkStatus 仍能识别;UI 这边不需特殊处理。

### 6.3 done 后 src 仍指 YouTube,iframe 不切 local
预期行为(Q-e-1=A)。用户在 block 内看的是 YouTube embed,在 Finder 里看到的是本地文件。**用户需理解这个分离**(Data Tab 文案需明确说明)。

### 6.4 translation .srt 路径安全
ytdlp.saveSubtitle main 侧已校验 `isAbsolute + 不含 .. + langCode 正则`(L5-B3.17 落地)。**复用既有约束**。
