## 4. 关键设计决策落地

### 4.1 三态状态机
- **placeholder**:`!src` → 🖼 + Upload + Embed link 双按钮
- **普通图**:`src && !isSvg` → `<img>` + 左右 resize handles
- **SVG**:`src && isSvg` → `<div>` + safe innerHTML + resize handles

`update()` 检测态切换 → 返回 false 让 PM 重建 NodeView(三态切换不会卡到上一态的 DOM 结构)。

### 4.2 SVG 安全(改进 V1)
V1 直接 `innerHTML = svgText`,`<script>` / `on*` 事件可能执行(同源策略)。
V2 `injectSvgStringSafe`:
- 临时 div innerHTML
- TreeWalker 遍历清洗:删 `<script>` / `<foreignObject>` / `on*` 属性 / `javascript:` URL
- 移动子节点到 container

不引 DOMPurify(~50KB 重 + V2 暂无外部 npm 加包习惯),手写轻量级 sanitize ~30 行。

### 4.3 Resize handle
- 左右 2 个 handle,hover 才显示
- mousedown 进入 resize 模式 → 鼠标移动改 target.style.width → mouseup 写回 PM attrs.width
- 最小 40px

### 4.4 反向驱动证明
md-to-pm `PM_NODE_REGISTRY.image` 从 `❌` 改 `✅`,**md-to-pm 主体逻辑不动**。
原本走 unknown 占位的图,现在 image schema 加进 ENABLED_BLOCKS 后**直接渲染为真实 image 节点**。
印证 L5-B4.3.1 的"输出目标 schema 节点 + unknown 占位 + 反向驱动"路线。

**接受 staleness**:已存的 unknown 节点不会升级,新转出来的 markdown 才用 image。
完成报告明示;后续如有真实数据问题,可写一次性 migration。
