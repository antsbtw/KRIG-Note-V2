## 1. 实际改动 vs 设计 — 偏差点

### 1.1 fileBlock / externalRef 走 atom: true(决策 Q2 = A 验证通过)

设计 § 6 Q2 倾向先试 atom: true,V1 走 `content: 'text*'` 是为给 PM 派 mousemove 给
block-handle hover。**V2 测试 atom: true 不阻塞 typecheck**(实际 hover 行为待用户实测,
若有 bug 退回 'text*' 即可)。

### 1.2 抽 insertAtomBlock 通用辅助

设计 § 5.3 各 view-side command 单独写 insertBlock。**实际**在 driver api.ts 抽
`insertAtomBlock(view, nodeName)` 通用函数,fileBlock / externalRef 共用 — 减少重复 +
保持"空段落替换 / 否则段后插入 + isTitle 守卫" 逻辑一处。

### 1.3 main 侧 MEDIA_DIR 安全校验加严

设计 § 9.1 风险:`media://../../etc/passwd` 越界。
**实际**:`resolveMediaPath` 加双重防护:
- `path.resolve` 后必须 `startsWith(MEDIA_DIR + path.sep)` 或 === MEDIA_DIR(防 sibling 路径如 `.../media-evil`)
- `fs.existsSync` 校验文件存在(避免暴露"是否存在"信息也被利用)

### 1.4 fileLink leafText 模式微调

设计 leafText 用 `[[label]]`-like 格式。**实际**:`📎<filename>`,跟图标渲染对应,
复制到外部应用时也是合理 fallback(对齐 V1 textContent 行为)。

## 3. 用户验证清单

`npm run dev`,Note view,以下逐项测:

### fileBlock 主流程(judge #2 - #8)
1. 打开任一现有笔记 — 不白屏
2. 输入 `/` 唤 slash menu — 看到 "File attachment" 一项
3. 选中 → 插入 fileBlock placeholder(📎 + Choose file 按钮 + URL 输入)
4. 点 "Choose file" → 系统弹文件选择器,选一个 PDF
5. 切 card 态:📕 + 文件名 + `application/pdf · XX KB` + [打开] [在 Finder 显示]
6. 点 [打开] — 系统默认 PDF 阅读器打开该文件
7. 点 [在 Finder 显示] — Finder 高亮该文件(在 KRIG media store 内)
8. **完全退出 + 重启应用 + 重开笔记** — fileBlock card 仍显示 / 仍可打开(media:// 持久化)

### externalRef 主流程(#9 - #13)
9. slash → "External reference" — 插入 externalRef placeholder(🔗 + Pick a file + URL 输入)
10. **kind=file**:点 "Pick a file" → 选一个 PDF 文件
    - 切 card 态:📁 + 文件名 + 完整路径 + [打开] [在 Finder 显示]
    - 点 [打开] → 系统 PDF 阅读器
    - 点 [Finder 显示] → 文件原路径 Finder 高亮
11. **kind=url**:再插一个 externalRef → URL 输入框输 `https://example.com` 按 Enter
    - 切 card 态:🌐 + "example.com" 摘要 + 仅 [打开] 按钮(无 Finder 显示)
    - 点 [打开] → 系统浏览器打开
12. **失败提示**:placeholder "Pick a file" 时如选某些非 disk 来源(若 mac 测得到,
    或拖入来自浏览器的图)— placeholder 应红字显"无法解析文件路径,请改用下方 URL 输入"

### fileLink(本阶段不进 slash,验证 schema 注册)

14. fileLink 由未来 paste/drag 路径创建 — 本阶段验证仅靠"schema 加载不报错"(已 typecheck OK)

## 4. 已知约束 & 后续优化

### 4.1 atom: true vs content: 'text*' 待用户实测

设计 § 6 Q2 验证:V2 用 atom: true 实施;若 block-handle hover 在 fileBlock /
externalRef 上不显示 → 退回 V1 同款 content: 'text*'(NodeView 仍不暴露 contentDOM 保持只读)。

### 4.2 Windows 路径未支持

externalRef `fileToFileHref` 走 POSIX `/` 分隔符,Windows `C:\...` 路径不支持。
本阶段 macOS-only,留 issue 等 Windows 测试时补 `path.sep` 兼容。

### 4.3 fileLink 暂无创建路径

V1 fileLink 通过 paste/drag 流程产生(从 fileBlock 转 inline 等)。V2 本阶段 schema
注册了但**没写创建路径**。后续 Phase B+ 加:
- paste/drag 流程把 application/octet-stream 的文件 → fileLink(段落中)
- fileBlock context menu "转 inline" → fileLink

### 4.4 file:// embed 在 placeholder 输入路径

URL 输入框可输 `file:///path` 但需用户手输完整 URI。Phase B+ 改进:输 `/abs/path`
自动 prefix `file://`(externalRef 已做了 — fileBlock 没做,小遗憾)。
