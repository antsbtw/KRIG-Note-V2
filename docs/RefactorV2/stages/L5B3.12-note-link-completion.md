## 1. 实际改动 vs 设计 — 偏差点

### 1.1 driver 不直接 import noteStore — 走 LinkClickHandler 反向取

**设计(§ 2 NodeView)**:NodeView 直接 `noteStore.get(noteId)` 同步函数

**实际**:driver 不能 import `@views/note/note-store`(违反铁律 — driver → view 反向依赖)。
改用扩展 `LinkClickHandler` 接口加 `resolveNoteTitle?(noteId): string | null` 字段:
- driver `getLinkClickHandler` getter 暴露
- view 在 `link-click-integration.ts` 注入实现(用 noteStore.get)
- NodeView 通过 handler 间接查 title(返回 null = "未找到"红色态)

**代价**:driver 加 9 行 + view 加 9 行接入,跟原有 `setLinkClickHandler` 一脉相承,清晰。

### 1.2 search panel 拆出独立 integration.ts 文件

**设计**:#4 NoteLinkSearchPanel 含 popup 启动逻辑 + UI

**实际**:把 popup 启动 / 关闭逻辑(setNoteLinkSearchHandler 接入)拆到
`note-link-search/integration.ts`(45 行),NoteLinkSearchPanel 纯 UI(140 行)。
- 对齐 link-click-integration.ts 的"view 层接 driver 拓展点"模式
- registerNoteLinkSearchIntegration 在 view/index.ts 副作用调用

### 1.3 plugin 加 active view 模块单例(getNoteLinkActiveView)

**原因**:popup 组件无法靠 props 拿 EditorView(popup 是 popupRegistry 注册全局组件,
mount 时不知道是哪个 PM view)。

**方案**:plugin 在 `onOpen` 时把 view 写入模块级单例,panel mount 时 getNoteLinkActiveView()。
单 React tree + 同时只有一个 active popup,模块级单例安全。

## 3. 用户验证清单

打开 V2 → 进入 Note view,以下逐项测:

### 基础流(judge #2 - #6)
1. **加载现有笔记** — 之前的笔记打开正常,不白屏(schema 加 noteLink 后向兼容)
2. **输入 `[[`** — 期望:`[[` 留在文档中,光标下方弹出搜索面板,列出所有笔记(标题 + 📄 图标)
3. **继续输 `[[红楼`** — 期望:面板过滤,只显示 title 含"红楼"的笔记
4. **按 ↓ ↑** — 期望:面板内高亮选项跟着移动
5. **鼠标 hover** — 期望:hover 项也变高亮
6. **按 Enter / 点击选中项** — 期望:`[[红楼` 文本被删掉,原位插入 `📄 <笔记标题>`(蓝色)
7. **再输 `[[`** — 重新弹,验证可以反复触发

### 路由 + 失效(#7 - #8)
8. **点击已插入的 noteLink** — 期望:当前 ws 切到目标笔记(覆盖左栏当前)
9. **删掉一个被引用的笔记** — 在 NavSide 删 note-X,然后回到含 `[[note-X]]` 的笔记,
   期望:noteLink 显**红色** "📄 ... (未找到)"

### leafText 复制(#9)
10. **选中 noteLink + Cmd+C** — 期望:剪贴板含 `[[<label>]]` 而不是空文本(leafText 生效)

### 关闭路径(#10)
11. **`[[` 弹出后按 Esc** — 期望:面板关,`[[` 留在文档中
12. **`[[` 弹出后输入 `]]`** — 期望:面板自动关(plugin 检测到 ]] 触发 close)
13. **`[[abc` 状态下退格删掉 `[[`** — 期望:面板自动关

### 边界
14. **`[a]` 单 `[`** — 期望:不触发面板(plugin 严格判 `[[`)
15. **空 noteStore 时 `[[`** — 期望:面板显"无匹配笔记"

## 4. 已知约束

### 4.1 title 同步是 mount-once(决策 Q2 = A)

NodeView mount 时同步一次 title;**正在打开的 note 内的 noteLink 不会因为目标 note 改名实时刷新**。
关闭重开当前笔记后会同步到最新 title。这是 V1 一致行为,实时 subscribe 留 sub-stage(决策推迟到性能边界明确)。

### 4.2 popup 跟 slash menu 冲突

设计 § 8.3 提到的预案:noteLink popup show 时 popupController 自动单例化(其他 popup 自动关)。
slash menu 不归 popupController(它是独立 controller),实操路径:
- 用户先按 `/` 弹 slash menu,再输 `[[` — slash menu 仍在
- 这种用例罕见,本阶段保留;真出问题再加 popupController.show 时 slashMenuController.hide()
