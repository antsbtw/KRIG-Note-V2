## 1. 实际改动 vs 设计 — 偏差点

### 1.1 binary-manager.ts 加防重入(决策 Q3)

设计 Q3 推荐 main 侧 `installPromise` 单例。**实际**:
```ts
let installPromise: Promise<YtdlpStatus> | null = null;

export async function install(onProgress?): Promise<YtdlpStatus> {
  if (installPromise) return installPromise;
  installPromise = doInstall(onProgress).finally(() => {
    installPromise = null;
  });
  return installPromise;
}
```

V1 没这个保护;V2 加上避免文件竞争。

### 1.2 saveSubtitle 安全校验加严(决策 Q4)

设计 Q4 推荐 isAbsolute + 不含 ..,**实际**额外加 langCode 正则约束:
```ts
if (!path.isAbsolute(videoFilePath) || videoFilePath.includes('..')) return null;
if (!/^[a-z]{2,5}(-[A-Za-z]{2,4})?$/.test(langCode)) return null;
```

防 ../ 通过 langCode 注入文件名(.srt 路径拼接走 langCode)。理论场景小(只有视频
block 调),但 main 进程边界值得防呆。

### 1.3 progress 推送走 webContents.send(对齐 V2 现有约定)

设计提到 progress 推送方式不明确。**实际**走 `event.sender.send(channel, payload)`
+ renderer 侧 `ipcRenderer.on(channel, handler)`,跟 V2 现有 `WINDOW_FULLSCREEN_CHANGED`
完全同模式。

订阅函数 `onYtdlpInstallProgress / onYtdlpDownloadProgress` 返回 unsubscribe 函数
(对齐 `onFullscreenChanged`),支持多订阅。

## 4. 已知约束 & 后续

### 4.1 macOS-only(本阶段)

V1 沿用,Windows / Linux 用户 install 会失败(URL 不匹配 yt-dlp_macos)。
Phase E 加跨平台 binary 区分。

### 4.2 yt-dlp 自身需要更新

二进制装好后跑了几个月,YouTube 反爬升级 → yt-dlp 老版本失效。
本阶段不做自动更新,用户重新调 install 重置二进制。

### 4.3 fetchTranscript 失败静默

某些 YouTube 视频禁用字幕 → fetchTranscript throw。V1/V2 都 try/catch 忽略,
不影响视频下载本身。

### 4.4 未来消费方排期

- **L5-B3.18 tweet-block** — 推文视频下载(消费 download / getInfo)
- **L5-B3.19 video 字幕系统** — WebVTT cue / CC 浮层 / 翻译(消费 saveSubtitle)
- **Phase D web-bridge** — 跟 browser-capability + ai-note-bridge 一起完整迁
- **Phase E** — 跨平台 binary / 自动更新 / yt-dlp 内置 -U 包装
