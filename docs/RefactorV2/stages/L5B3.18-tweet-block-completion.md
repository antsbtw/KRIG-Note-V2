## 1. 实际改动 vs 设计 — 偏差点

### 1.1 driver 走 capability **模块级 import**(对齐 W5 严格态 A 边界)

设计 § 3.3 提到"driver 内部消费 capability,优先通过 capability API 保持口径统一"。
**实际**:
- `import { fetchTweetData } from '@capabilities/tweet-fetcher'`
- `import { checkStatus, install, download } from '@capabilities/ytdlp'`

**未**走 `requireCapabilityApi`(view 才需要,driver 内部直 import 等价合规,W5 边界 A)。
DESIGN.md 已强化"禁止绕过 capability 直 import @platform/main/* 或 IPC channel"约束。

### 1.2 view install 加了 media-storage(本阶段顺手补)

设计 § 1.1 只列 `ytdlp` + `tweet-fetcher` 两个新加。**实际**还加了 `media-storage` —
note-view 长期 driver 内部消费 media-storage(file-block / image / audio / video),
之前 install 列表漏了它(audit 老遗留),本阶段顺手补,install-coverage 自检更完整。

### 1.3 CSS iframe scale + 宽度补偿

V1 用 `transform: scale(0.75)` 缩小 iframe(Twitter embed 默认偏大)+ `width: 133.33%`
(`1/0.75`)补 scale 后视觉宽度。V2 直迁同模式。

## 4. 已知约束 & 后续

### 4.1 ⚠️ 临时 capability 红线

`tweet-fetcher` 是**临时 capability**(用户红线 — 避免临时能力长期化):
- DESIGN.md 顶部 banner 标识 + 不扩展项目清单(明确禁:Twitter timeline / 搜索 /
  其他社交平台 scraping / 通用 DOM 框架 / 媒体下载)
- Phase D browser-capability 正式化时被吸收为"DOM scraping"子能力
- **本阶段后续 audit 必检**:tweet-fetcher 是否被新功能扩展(任何扩展走 Phase D 通道)

### 4.2 Twitter SPA 反爬升级

EXTRACT_TWEET_JS 基于 Twitter `data-testid="tweet"` / `User-Name` / `tweetText` 等选择器。
Twitter 改名 / 改结构 → scraping 失败,Fetch 显 ❌(不影响 Browse Tab iframe)。
后续维护走小补丁更新选择器,不影响主功能。

### 4.3 t.co 短链

V1 NodeView extractTweetId 正则只匹配 twitter.com / x.com,不支持 t.co 短链。
V2 直迁同限制。

### 4.4 视频字幕系统(留 L5-B3.19)

tweetBlock 下载的视频,字幕系统(WebVTT cue / CC 浮层 / 翻译)留 L5-B3.19 video 字幕系统
(消费 ytdlp.saveSubtitle)。

### 4.5 BrowserWindow 创建限制(打包后)

打包后 BrowserWindow 创建可能受沙盒限制 → fetcher.ts try/catch 兜底返 success:false。
本阶段开发环境验证;打包验证留 Phase D 集成测试。
