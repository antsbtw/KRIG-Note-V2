## 0. 目标与价值

V1 web view 4 大变体之一(对照 L5-B4 实现的 base WebView)。**双屏同步翻译**是 KRIG
学习/翻译/AI 工作流的核心入口:

- 左栏原文 webview + 右栏翻译 webview
- 用户在左 / 右任一侧滚动 / 点击 / 输入 / 选中,**对面同步**(scroll / click / input / selection)
- 右栏自动注入 Google Translate 翻译成目标语言(中/日/韩/英 等)
- 用户左右对照看英文原文 + 中文翻译,提升阅读效率

**典型场景**:打开英文文档 → 点 WebToolbar 翻译按钮 → 右栏弹出翻译 webview → 双向同步导航。

## 1. V1 实现盘点

### 1.1 模块组成(~677 行)

| 文件 | 行数 | 职责 |
|---|---|---|
| `sync/sync-protocol.ts` | 15 | 5 个 ViewMessage action(NAVIGATE / REQUEST_URL / READY / SYNC_EVENTS / TAKE_CONTROL) |
| `sync/sync-driver.ts` | 384 | renderer 侧同步引擎(controller/passive 模型 + 80ms poll + 7 种事件应用) |
| `sync/sync-inject.js` | 410 | 注入到 guest webview 的脚本(抓 click/scroll/input/selection 事件入 `__mirroSyncQueue`)|
| `translate/translate-driver.ts` | 107 | 渲染侧翻译引擎(注入 Google Translate element.js + CSP 旁路 + 暗色模式) |
| `translate/google-translate-inject.js` | 107 | 注入到 guest 的 Google Translate 启动器 |
| `components/TranslateWebView.tsx` | 171 | 右栏翻译 webview React 组件 |

### 1.2 关键架构(V1)

**控制权模型(单向通信防回环)**:
- 同一时刻只有单向同步:controller 发,passive 收
- 用户在 X 侧操作(poll 抓到事件)→ X 侧自动 takeControl + 通知对面 yield
- 防止两侧互发滚动事件造成抖动

**同步事件类型(7 种)**:
- `scroll-delta`:相对滚动(滚轮/键盘)
- `scroll-anchor`:绝对滚动(锚点元素 + offsetRatio,跨页面定位)
- `click`:链接/按钮 click(配 toggleState 防 toggle 类按钮重复触发)
- `input`:input/textarea/select 同步
- `input-enter`:输入框 Enter(支持回调翻译值后再设到对面 — 输 EN → 对面收 ZH)
- `submit`:表单提交
- `selection`:选区高亮(对面 webview 加 outline)

**翻译注入 5 步**:
1. 移除 CSP meta(防 Google CDN 被 block)
2. fetch element.js(IPC 走 main 拿)
3. 注入 google-translate-inject.js 启动器(替换 `__KRIG_TARGET_LANG__`)
4. 执行 element.js(Google CDN 代码)
5. 暗色模式 meta 注入

**两 webview 通信(V1 走 main 进程)**:
- 左 webview renderer.send → main 转发 → 右 webview renderer.on
- IPC channel: `IPC.VIEW_MESSAGE_SEND` / `viewAPI.sendToOtherSlot`
- main 进程 slot-manager 知道当前 active ws 的 left/right webContents

### 1.3 V2 可复用 / 缺失

| 资源 | V2 现状 |
|---|---|
| `<webview>` tag | ✅ L5-B4 已启 |
| WebView 组件 | ✅ L5-B4 |
| slot 系统 | ✅ slotBinding.left / right |
| **跨 slot 通信(IPC)** | ❌ V2 单 React tree,直接走模块级 event bus 即可,不需要 IPC |
| **Google Translate element.js IPC fetch** | ❌ 需补 IPC |
| **CSP 旁路(disablewebsecurity)** | ❌ webview 当前无此属性 |
| translate-driver / sync-driver / 注入脚本 | ❌ 全新迁 |

## 3. 决策点(请用户拍板)

### Q1:跨 slot 通信用 V2 模块级 event bus 还是补 IPC?

- **A. 模块级 event bus(推荐)**:V2 单 React tree,左右 webview 在同一 renderer,
  直接走 JS 模块单例,免 IPC 复杂度
- B. 走 IPC(对齐 V1 main 转发模式):多余,V2 不需要

**推荐 A** — V2 架构红利

### Q2:sync-inject.js 命名空间(`window.__mirroSyncQueue`)

- **A. 改 `__krigSyncQueue` (推荐)**:命名空间隔离,避免跟 V1 doc 残留冲突 +
  KRIG 命名一致性
- B. 沿用 `__mirro`:零改动

**推荐 A**

### Q3:Google Translate element.js IPC

- **A. 走 main 进程 IPC fetch + cache(对齐 V1)**:首次 fetch translate.googleapis.com/element.js
  缓存到内存,后续直接给
- B. webview 内直接 script src:浏览器加载
  - 风险:CSP 拦,V1 经验是必须 main fetch 注入

**推荐 A**(V1 同款)

### Q4:翻译目标语言

- **A. 4 种(中/日/韩/英)**(推荐):覆盖主要场景
- B. V1 同款 7 种(中/日/韩/英/法/德/西)
- C. 用户配置

**推荐 A** — 简单,后续按需扩

### Q5:WebView 内翻译按钮放哪里?

- **A. WebToolbar 右侧 actions 区(推荐)**:对齐 V1
- B. 框架 toolbar(view-web 注册一项 button,但 view-web 当前空 toolbar)

**推荐 A**(L5-B4 已留 actions 占位区,正好填)

### Q6:web-translate-view 是否独立 view 注册?

- **A. 是(推荐)**:registerView 'web-translate-view',跟 web-view 平等;
  link 路由 / WebToolbar 翻译按钮 触发 slotBinding.right = 'web-translate-view'
- B. 不注册,只作 web-view 的内嵌组件:复杂,且 slot 系统不知道它

**推荐 A**

### Q7:input-enter 翻译回调本阶段做不做?

V1 行为:用户在右栏(翻译版)输入框输 ZH 文字 + Enter → 调翻译 API ZH→EN → 对面输入框设 EN 值
- **A. 不做(推荐)**:本阶段砍掉,等 viewAPI 翻译 IPC 阶段
- B. 做 placeholder(input-enter 同步原值,不翻译)

**推荐 A** — 减依赖,核心同步导航功能不阻塞

## 6. 风险 + 开放问题

### 6.1 sync-inject.js 注入时机

V1 在 did-finish-load 时调 `injectSyncScript`,但页面还在初始化时部分 DOM 还没建好。
V1 用 80ms poll + DOM 操作的 try/catch 容忍。本阶段 1:1 沿用 V1 容错策略。

### 6.2 disablewebsecurity 安全风险

- 只在 translate-view 启,base WebView 不启
- 用户感知:翻译 view 不要登录敏感账号(留 README 提示)
- 后续可考虑只放开 translate.googleapis.com 域名(CSP 白名单),但 V1 用粗粒度
  disablewebsecurity 已满足需求

### 6.3 跨 webview Cookie 隔离

V1 left/right webview 都用 `WEBVIEW_PARTITION='persist:webview'` 共享 session。
本阶段沿用 — translate-view 跟 web-view 共享 Cookie/cache。

### 6.4 SyncDriver controller 切换的边界 case

V1 经验:页面跳转 / 长加载时控制权易错乱。V1 用 `remoteNavUntilRef` 时间窗口
(Date.now() < remoteNavUntilRef + 2000)区分"对面触发的导航 vs 用户主动导航"。
本阶段 1:1 沿用。

### 6.5 google-translate-inject CDN 依赖

需要外网访问 translate.googleapis.com,**国内网络环境可能失败**。
V1 没处理 fallback,V2 同款 — 出错就静默不翻译,不影响 sync 同步。
