## 3. 阶段中遇到 / 解决的问题

### 3.1 白屏:PM content 表达式不支持节点 name 含短横线

**现象**:加 6 个 block 后启动白屏,console 报 `SyntaxError: Unexpected token '-'`
**根因**:节点 id 用了 `bullet-list` 风格,PM 解析 content expression 时把 `-` 当减号
**修法**:新建 block id 全部改驼峰 `bulletList` / `orderedList` / `listItem` / `taskList` / `taskItem` / `blockquote` / `codeBlock` / `horizontalRule`
**例外**:既有 `text-block`(L5-A 已存)保留,`listItem.content: 'block+'` 用 group 'block' 通配引用避免直接 ref `text-block`
**对应 commit**:`e223495`,记忆已落 [feedback_pm_schema_naming.md](memory)

### 3.2 input rule 触发后光标停留位置错

**现象**:输 `- ` 触发后光标跑到 list 之外 / 不在新 listItem 内
**修法**:wrapInListRule 内手算路径偏移(blockStart + 3:进 list +1 / 进 listItem +1 / 进 textBlock +1)+ TextSelection.near 收敛
**对应 commit**:`4e2d7a4`

### 3.3 handle 渗入 list — 整 list 一个 handle

**现象**:鼠标悬停 list 第二项时 handle 仍指向 list 整体(顶层 ul/ol)
**根因**:handle 探测用顶层 block 维度,把 list 当作单一 block,内部 listItem 不被探测
**修法**:在 listItem / taskItem 节点也注册 handle(每项独立)
**对应 commit**:`fc4665f`

### 3.4 ordered-list `attrs.start` 视觉失效

**现象**:输 `5. ` 触发后 attr 是 5,但渲染仍从 1 开始
**根因**:CSS `counter-reset: ordered-item` 默认从 0 起,toDOM 只输出 HTML `start` 属性 CSS 不读
**修法**:toDOM 注入 inline `style="counter-reset: ordered-item ${start - 1}"` 覆盖 CSS 默认
**对应 commit**:`e15ecaa`

### 3.5 task-list 嵌套双倍缩进

**现象**:嵌套 task-list 比预期深一倍(checkbox 不在最左)
**根因**:通用 `ul ul / ul ol / ol ul / ol ol` 嵌套 24px 规则**也吃 task-list**;且通用 `li { padding-left: 24px }`(给 ::before 留位)对 task-item flex 布局多余
**修法**:用更具体选择器 `ul.krig-task-list ul.krig-task-list { padding-left: 0 }` 覆盖,task-item 单独 `padding-left: 0`(checkbox 自占)
**对应 commit**:`e15ecaa`

### 3.6 task-item 缺 V1 业务特色(deadline / 时间戳 / 超期红字)

**初版**:V2 taskItem 仅 `{ checked: false }` 单 attr,checkbox toggle 无时间戳
**对照 V1**:taskItem 5 attrs(checked / createdAt / completedAt / deadline)+ NodeView 有时间标签 / date picker / 超期标记 — KRIG 业务特色
**修法**:扩 attrs + 重写 NodeView(时间标签 + date input + 自动 createdAt + checked/overdue 同步)
**对应 commit**:`e15ecaa`

### 3.7 horizontalRule 缺 atom: true 导致光标陷入

**初版**:hr nodeSpec 仅 `group: 'block'`,光标按 ↓ 进 hr 内"消失"
**根因**:PM 叶子节点(无 content)必须 `atom: true`,否则不被当作不可拆分单元
**修法**:补 atom: true + selectable: true + CSS 选中高亮
**对应 commit**:`e15ecaa`

### 3.8 codeBlock 缺 keyboard plugin → Enter 出 codeBlock

**现象**:在 codeBlock 内按 Enter 直接跳出变 paragraph(PM 默认行为对 code: true 块不友好)
**修法**:新建 `buildCodeBlockKeymap` 移植 V1 的 codeBlockKeyboardPlugin(Enter 插 \n / 双 Enter 跳出 / Tab 2 空格 / Shift-Tab 反缩进 / Backspace 空块清退)
**对应 commit**:`e15ecaa`
