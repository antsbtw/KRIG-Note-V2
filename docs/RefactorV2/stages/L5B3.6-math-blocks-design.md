## 2. mathBlock 设计

### 2.1 NodeSpec

V1:`content: 'text*'`,`code: true`,`defining: true`,`marks: ''`,attrs `color` / `bgColor`。

V2 直迁:

```ts
const mathBlockNodeSpec: NodeSpec = {
  content: 'text*',
  group: 'block',
  code: true,
  defining: true,
  marks: '',
  attrs: {
    color:   { default: null },
    bgColor: { default: null },
  },
  parseDOM: [{
    tag: 'div.krig-math-block',
    preserveWhitespace: 'full',
    getAttrs(dom) {
      const el = dom as HTMLElement;
      return {
        color: el.getAttribute('data-color') || null,
        bgColor: el.getAttribute('data-bg-color') || null,
      };
    },
  }],
  toDOM(node) {
    const attrs: Record<string, string> = { class: 'krig-math-block' };
    if (node.attrs.color) attrs['data-color'] = node.attrs.color as string;
    if (node.attrs.bgColor) attrs['data-bg-color'] = node.attrs.bgColor as string;
    return ['div', attrs, ['pre', { class: 'krig-math-block__code' }, 0]];
  },
};
```

### 2.2 NodeView 行为(对齐 V1)

**两态切换**:
- **rendered 态**(默认):KaTeX 渲染 LaTeX,点击进入 edit
- **edit 态**:显示原始 LaTeX 源码 + 实时预览;Esc 或 点外部退出

**性能优化(V1 已有,直迁)**:
- KaTeX 渲染缓存:LaTeX 未变化跳过重复渲染(`lastRenderedLatex` ref)
- 共享 `IntersectionObserver`:所有 mathBlock 复用一个,200px rootMargin lazy 渲染
- 共享全局 mousedown 监听:所有进入 edit 的 mathBlock 共享一个 listener

**砍 V1 的 help panel**:
- 编辑头不放 `?` 按钮,或显示但点击无效(占位 + console.warn)
- LaTeX 速查面板留 L5-B+

### 2.3 CSS

直迁 V1 `.math-block*` selector,加 `krig-` 前缀:
- `.krig-math-block` / `.krig-math-block--editing`
- `.krig-math-block__rendered`(KaTeX 容器,字号 1.4em)
- `.krig-math-block__editor`(edit 态:header + code + live preview)
- `.krig-math-block__code`(LaTeX 源码 pre,字号 13px)
- `.krig-math-block__preview`(实时预览,字号 1.3em)
- `.krig-math-block__empty` / `.krig-math-block__error`

## 3. mathInline 设计

### 3.1 NodeSpec

```ts
const mathInlineNodeSpec: NodeSpec = {
  inline: true,
  group: 'inline',
  atom: true,           // 不可拆,光标不进入
  attrs: { latex: { default: '' } },
  parseDOM: [{ tag: 'span.krig-math-inline' }],
  toDOM() { return ['span', { class: 'krig-math-inline' }]; },
  leafText: (node) => `$${node.attrs.latex as string}$`,  // textBetween / 复制成纯文本时
};
```

### 3.2 NodeView 行为

- **render**:KaTeX inline 渲染(displayMode: false)
- **空 latex**:显示 "New equation" 占位,单击编辑
- **双击编辑**:弹出绝对定位的小编辑器(input + live preview)
- **保存**:Enter / 点外部 → setNodeMarkup 写 latex
- **取消**:Esc → 关闭弹窗,不保存

V1 thoughtMark 集成砍掉(V2 没 thought mark 系统)。

### 3.3 CSS

- `.krig-math-inline`(inline 元素,KaTeX 渲染)
- `.krig-math-inline--empty`(占位样式)
- `.krig-math-inline-editor`(绝对定位弹窗)+ input + preview

## 4. 决策点

### Q1:KaTeX 安装 npm 包还是直迁本地副本?

V1 走 `katex` npm 包,~280KB(JS + CSS + 字体)。

- **A. npm 安装**(V2 同样 `npm install katex`):跟 V1 一致,Vite 自动 tree-shake / 字体 bundling
- **B. 直接 vendor 进 src/**:不依赖 npm,但字体 / CSS 等资源管理麻烦

**推荐 A** — 标准做法,Vite 处理字体好

### Q2:LaTeX 速查面板(help panel)是否本阶段做?

V1 mathBlock 编辑头有 `?` 按钮,点击弹出 LaTeX 速查参考(~520 行 latex-data + panel UI)。

- **A. 不迁,这次砍掉**:`?` 按钮注释或不显示,留 L5-B+
- **B. 直迁全套**:多 ~600 行,但用户体验完整
- **C. `?` 按钮显示但点击 toast "速查面板留后续"**:占位提醒,未来补

**推荐 A** — 速查面板对熟悉 LaTeX 的用户没用,新用户可以查文档,不挡核心闭环

### Q3:thoughtMark 集成是否保留 attrs

V1 mathInline 的 `syncThoughtMark` 同步 thought mark 到 DOM(显示标注线)。
V2 没 thought mark 系统,**直接砍 syncThoughtMark 调用**。

但 NodeSpec 里 V1 没 thoughtMark attr(thought 是独立 mark),所以 NodeSpec 不需要改,
只 NodeView 不调 syncThoughtMark。

**已确认**:NodeView 不调,NodeSpec 不动。

### Q4:slash menu 顺序

V1 mathBlock order=10,mathInline order=14,夹在基础 block 中间。V2 现在 slash 项:

```
Paragraph 10 / H1 20 / H2 30 / H3 40 / Bullet 50 / Ordered 60 / Task 70 /
Quote 80 / Code 90 / Divider 100 / Callout 110 / Toggle 120 / Image 130
```

L5-B3.5 image 在 130。math 加在哪?

- **A. Math Block 140 / Inline Math 150**(放最后,跟 image 一起作为"高级 block")
- **B. Math Block 95(在 codeBlock 之后)/ Inline Math 95.5**(对齐 V1 在前面)

**推荐 A** — 现在顺序按 V2 实际 sub-stage 出现顺序(image 130 已在 codeBlock 之后),math 跟着来

**实施时调整**:用户验收阶段提出 Inline Math 应该在 floating toolbar 而不是 slash —
理由:slash 是"插 block",floating toolbar 是"改选中文字"语义,行内公式属于后者
(选中 LaTeX 源 → 转公式)。最终方案:
- **Math Block(块级)** → slash menu `/math`(order 140)
- **Inline Math(行内)** → floating toolbar `∑`(order 70),选区有内容时
  转换;无选区时插空 atom 单击编辑

### Q5:md-to-pm 升级时机

L5-B4.3.1 md-to-pm 已经 emit `mathBlock` / `mathInline` 节点(走 unknown 占位)。

- 本阶段升级 PM_NODE_REGISTRY 标记 `mathBlock: '✅'` / `mathInline: '✅'`
- 验证:写一段 markdown 含 `$$x^2$$` / `$y^2$` → 转出来直接渲染

**已确认**:跟 L5-B3.5 image 同款"反向驱动证明"

## 7. 风险

### 7.1 KaTeX CSS 资源加载

V1 `import 'katex/dist/katex.min.css'` 走 Vite CSS 处理,字体文件(woff2)走相对路径。
V2 同样 import,Vite 自动 bundle。**风险点**:Electron renderer CSP `default-src 'self'`
可能阻字体文件加载(KaTeX 字体走 css `@font-face url(...)`)。

**预案**:如果字体加载失败,在 index.html CSP 加 `font-src 'self' data:`(L5-B3.5 已加
img-src / connect-src 白名单,这次顺便加 font-src)。

### 7.2 codeBlock vs mathBlock 区分

V2 已有 codeBlock (`content: 'text*'`, `code: true`)。V1 mathBlock 也是 code+text*,
两者节点 spec 类似。在 V2 schema 里两个节点 id 不同(`codeBlock` vs `mathBlock`)
不会冲突,但 V1 用了 group 'block' 都注册到 doc content 'block+',OK。

**确认**:无冲突

### 7.3 mathInline 编辑弹窗定位

V1 弹窗 `position: fixed; top/left = rect.bottom/left`。如果 mathInline 靠近 viewport
底部,弹窗可能溢出屏幕。V1 没处理。

**预案**:V2 直迁 V1 行为,溢出问题留 L5-B+
