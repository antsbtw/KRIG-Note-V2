## 3. 用户验证清单

`npm run dev`,Note view,以下逐项测:

### 基础流(judge #2 - #4)
1. **打开任一现有笔记** — 不白屏(plugin 装载 OK)
2. **截屏(Cmd+Shift+4)→ 在空段落 Cmd+V** — 期望:空段落被替换为 image(不留空段落)
3. **截屏 → 在含文字的段落 Cmd+V** — 期望:该段保留,image 插入到段后

### 嵌套 + title 守卫(#5 - #6)
4. **截屏 → 光标在 title(首块,空标题)Cmd+V** — 期望:**title 不被替换**,image 插到 title 后(B3.11 守卫)
5. **建一个 callout / blockquote,光标在内,截屏 → Cmd+V** — 期望:image 落在 callout/blockquote 内(嵌套定位)
6. **建表格,光标在某个 cell,截屏 → Cmd+V** — 期望:image 落在该 cell 内(table cell 也是嵌套容器)

### mediaStore 持久化(#7)
7. **粘贴一张图,等 1-2 秒,检查 image 节点 attrs**(可以 F12 看 DOM 的 `data-src`)— 期望:`media://<hash>...`,**不是 `data:image/...;base64,...`**
8. **完全重启应用 + 重开笔记** — 期望:粘贴的图仍然显示(刷新不丢)

### HTML 让步(#8)
9. **从 Excel 复制一个表格 → Cmd+V** — Excel 通常会同时附 PNG fallback;期望:**不**抢成图,落 PM 默认粘贴(可能是破碎 HTML,但保留结构语义,Phase E smart-paste 接入后会变好)

### 边界(#9)
10. **复制一段纯文本 → Cmd+V** — 期望:plugin 不拦截(handlePaste return false),PM 默认粘贴文本

## 4. 已知约束 & 后续优化

### 4.1 PM pos 漂移(异步窗口期)

mediaPutBase64 异步窗口典型 100-500ms,期间用户继续编辑可能让 insertedPos 失效。
当前用 `nodeAt(pos).type.name === 'image'` 二次校验 + try/catch 兜底,失败留 dataUrl
(session 内可见,刷新丢)。

**更稳方案(未来)**:给 image schema 加 `id` attr,异步用 id 重新定位 pos。
本阶段不做,低概率边界场景容忍。

### 4.2 大图阻塞 FileReader

5MB+ 大图 readAsDataURL 可能卡 100-500ms。本阶段不优化(用户场景多是 < 2MB 截屏);
Phase E 可改 ArrayBuffer + Blob URL 异步管线。

### 4.3 多图粘贴丢失

仅取第一个 image/* 文件。多图粘贴是边界场景,用户反馈再做。
