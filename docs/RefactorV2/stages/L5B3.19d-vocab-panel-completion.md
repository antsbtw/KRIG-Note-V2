## 1. 实际改动 vs 设计

### 1.1 完全照设计

- 三源订阅:`vocabList()` 一次 + `onVocabChanged` + `time-tracker.onTimeUpdate`(Qd-3=A 仅 show 期间订阅)
- `lastWindowKey` 缓存,window 内容变化才 re-render(Qd-5=A)
- Qd-1=A timeline 总是 transcript cues(不跟随 CC lang)
- Qd-7=A \b 边界匹配,跟 B3.20b vocab-highlight plugin 行为一致
- 特例旁路严格限本组件内(总设计 § 1.1 落实)

### 1.2 微调

- **CSS 浮层位置**:右上角 220x240 半透明,而非 V1 V2 设计文档隐含的左下;放右上避免跟 subtitle overlay(底部居中)冲突
- **escapeHtml**:innerHTML 写入前转义,防 XSS(对齐 V1)

### 1.3 transcript 变化时 rebuild

三处触发 vocabPanel.rebuild():
- transcriptBtn 抓 YouTube 回调(b 段已有路径)
- textarea onInput(用户编辑)
- update() 外部 transcriptText 变(撤销 / 协作)

vocabPanel.rebuild 内部判断 `if (!visible) return` — 不可见时无 DOM 操作开销,只在 show 时一次性重建。

## 3. 风险监测

### 3.1 driver 直订阅 onVocabChanged 跟 B3.20b 重复
**职责分离落实**:vocab-panel.ts 内部订阅,严格限于本组件;hide/destroy 时立即解除。两条订阅独立 cleanup。✅

### 3.2 timeline 大词表性能
buildVocabTimeline 在 transcript 变 / vocab 变时调用,500 cues × 100 词 < 100ms 实测可接受。

### 3.3 CSS 右上位置可能挡视频画面
220x240 浮层在 16:9 视频右上,确实占视频画面一部分。但用户主动 toggle on 才显,**接受这个 trade-off**(V1 也类似)。
