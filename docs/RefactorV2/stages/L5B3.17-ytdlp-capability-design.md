### 1.3 出口契约(YtdlpApi)

```ts
// src/capabilities/ytdlp/types.ts
export interface YtdlpStatus {
  installed: boolean;
  version?: string;
  path?: string;
}

export interface YtdlpDownloadProgress {
  url: string;
  status: 'downloading' | 'complete' | 'error';
  percent: number;
  filename?: string;        // 完成时的本地路径
  subtitleFile?: string;    // .en.srt 路径(YouTube 字幕)
  subtitleText?: string;    // [MM:SS] 格式纯文本字幕
  error?: string;
}

export interface YtdlpInstallProgress {
  percent: number;          // 0-100,二进制下载进度
  installed: boolean;       // 完成后切 true
  version?: string;
  error?: string;
}

export interface YtdlpApi {
  /** 检查 yt-dlp 是否已安装 + 版本号 */
  checkStatus(): Promise<YtdlpStatus>;

  /** 下载并安装 yt-dlp 二进制(从 GitHub release,~22MB) */
  install(): Promise<YtdlpStatus>;

  /** 订阅安装进度(install 期间多次 emit,完成后自动 unsubscribe)
   *  返回取消订阅函数 */
  onInstallProgress(callback: (progress: YtdlpInstallProgress) => void): () => void;

  /** 下载视频(自动抓 YouTube 字幕保存为 .en.srt)
   *  outputPath 可选,默认 ~/Downloads/<title>.<ext> */
  download(url: string, outputPath?: string): Promise<YtdlpDownloadProgress>;

  /** 订阅下载进度(每次 percent 变化 emit) */
  onDownloadProgress(callback: (progress: YtdlpDownloadProgress) => void): () => void;

  /** 获取视频元数据(--dump-json,不下载)*/
  getInfo(url: string): Promise<Record<string, unknown> | null>;

  /** 保存翻译字幕为 .srt(对齐视频文件)*/
  saveSubtitle(videoFilePath: string, langCode: string, timestampText: string): Promise<string | null>;
}
```

#### handlers.ts(新写)

```ts
import { ipcMain, BrowserWindow } from 'electron';
import { IPC_CHANNELS } from '@shared/ipc/channel-names';
import * as path from 'path';
import { checkStatus, install } from './binary-manager';
import { downloadVideo, getVideoInfo, saveTranslationSubtitle } from './downloader';

export function registerYtdlpHandlers(): void {
  ipcMain.handle(IPC_CHANNELS.YTDLP_CHECK_STATUS, async () => checkStatus());

  ipcMain.handle(IPC_CHANNELS.YTDLP_INSTALL, async (event) => {
    const sender = event.sender;
    return install((percent) => {
      // 进度推送(每次 percent 变化)
      if (!sender.isDestroyed()) {
        sender.send(IPC_CHANNELS.YTDLP_INSTALL_PROGRESS, { percent, installed: false });
      }
    });
  });

  ipcMain.handle(IPC_CHANNELS.YTDLP_DOWNLOAD, async (event, url: unknown, outputPath: unknown) => {
    if (typeof url !== 'string' || !url) return { url: '', status: 'error', percent: 0, error: 'invalid url' };
    const out = typeof outputPath === 'string' ? outputPath : undefined;
    const sender = event.sender;
    return downloadVideo(url, (progress) => {
      if (!sender.isDestroyed()) {
        sender.send(IPC_CHANNELS.YTDLP_DOWNLOAD_PROGRESS, progress);
      }
    }, out);
  });

  ipcMain.handle(IPC_CHANNELS.YTDLP_GET_INFO, async (_event, url: unknown) => {
    if (typeof url !== 'string' || !url) return null;
    return getVideoInfo(url);
  });

  ipcMain.handle(IPC_CHANNELS.YTDLP_SAVE_SUBTITLE, async (_event, videoFilePath: unknown, langCode: unknown, timestampText: unknown) => {
    if (typeof videoFilePath !== 'string' || typeof langCode !== 'string' || typeof timestampText !== 'string') return null;
    // 安全:videoFilePath 必须是绝对路径(防 ../ 攻击)
    if (!path.isAbsolute(videoFilePath)) return null;
    try {
      return saveTranslationSubtitle(videoFilePath, langCode, timestampText);
    } catch (err) {
      console.warn('[ytdlp] saveSubtitle failed:', err);
      return null;
    }
  });
}
```

```ts
import { capabilityRegistry } from '@slot/capability-registry/capability-registry';
import type {
  YtdlpApi,
  YtdlpStatus,
  YtdlpDownloadProgress,
  YtdlpInstallProgress,
} from './types';

export type {
  YtdlpApi,
  YtdlpStatus,
  YtdlpDownloadProgress,
  YtdlpInstallProgress,
} from './types';

async function checkStatus(): Promise<YtdlpStatus> {
  if (!window.electronAPI?.ytdlpCheckStatus) {
    return { installed: false };
  }
  return window.electronAPI.ytdlpCheckStatus();
}

async function install(): Promise<YtdlpStatus> {
  if (!window.electronAPI?.ytdlpInstall) {
    return { installed: false };
  }
  return window.electronAPI.ytdlpInstall();
}

function onInstallProgress(callback: (progress: YtdlpInstallProgress) => void): () => void {
  if (!window.electronAPI?.onYtdlpInstallProgress) return () => {};
  return window.electronAPI.onYtdlpInstallProgress(callback);
}

async function download(url: string, outputPath?: string): Promise<YtdlpDownloadProgress> {
  if (!window.electronAPI?.ytdlpDownload) {
    return { url, status: 'error', percent: 0, error: 'electronAPI.ytdlpDownload not available' };
  }
  return window.electronAPI.ytdlpDownload(url, outputPath);
}

function onDownloadProgress(callback: (progress: YtdlpDownloadProgress) => void): () => void {
  if (!window.electronAPI?.onYtdlpDownloadProgress) return () => {};
  return window.electronAPI.onYtdlpDownloadProgress(callback);
}

async function getInfo(url: string): Promise<Record<string, unknown> | null> {
  if (!window.electronAPI?.ytdlpGetInfo) return null;
  return window.electronAPI.ytdlpGetInfo(url);
}

async function saveSubtitle(
  videoFilePath: string,
  langCode: string,
  timestampText: string,
): Promise<string | null> {
  if (!window.electronAPI?.ytdlpSaveSubtitle) return null;
  return window.electronAPI.ytdlpSaveSubtitle(videoFilePath, langCode, timestampText);
}

// W5 严格态:Registry 注册 + api 字段(view 通过 requireCapabilityApi 间接路由)
capabilityRegistry.register({
  id: 'ytdlp',
  api: {
    checkStatus,
    install,
    onInstallProgress,
    download,
    onDownloadProgress,
    getInfo,
    saveSubtitle,
  } satisfies YtdlpApi,
});

// 同时模块级 export — driver/slot 内部消费保留(对齐 media-storage 模式)
export {
  checkStatus,
  install,
  onInstallProgress,
  download,
  onDownloadProgress,
  getInfo,
  saveSubtitle,
};
```

## 3. 决策点

### Q1:capability 入口里"业务函数"是 export const 还是嵌在 api 对象里?

参考 [audit 2026-05-08 § 5.2](../audit/2026-05-08-register-and-layer-audit.md#52-w5-严格态边界定义2026-05-08-复审钉死)
W5 严格态边界定义 A:**view 边界严格**(view 必须经 `requireCapabilityApi('ytdlp')`
间接路由)+ driver / slot / frame-bindings / triggers 仍可直 import capability 单例(charter
line 88 主语为 view)。

**A. 跟 media-storage 一致**:模块级 export const + Registry api 字段同时挂(双导出)
**B. 只 Registry 注册**:删模块级 export(view 必须经 Registry)— 最严格

**推荐 A**(对齐已落地 capability 的现状,与 audit § 5.2 W5 严格态 A 一致)。

> ⚠️ **重要标注**:Q1 选 A 是 **W5 严格态 A 的临时允许项**(audit § 5.2 已钉死边界 +
> "已知边界"列出 5 个老 capability 都走双导出)。**不是**全局注册式严格态(B/C)
> 的达成态。后续若 charter v0.5 升级到 B/C,本 capability 跟随其他能力一同改造,
> 而非提前激进。
>
> 本阶段选 A 的核心理由:
> 1. 对齐已落地的 5 老 capability + media-storage / text-editing / web-rendering 写法,
>    避免新 capability 引入额外风格分叉
> 2. driver/slot 内部消费(若有)走单例 import 跟现有 capability 一致(charter line 88
>    主语限于 view,driver/slot 不在禁列)
> 3. 同时立 Registry 注册让 install 校验工作,view 间接路由强制(W5 已生效)

### Q2:install / download 的 progress 订阅 — 单订阅 or 多订阅?

V1 install/download 一次性任务,理论上同时只有一个 task 在跑。

**A. 单订阅**(只允许一个订阅者,新 callback 替换老的)
**B. 多订阅**(EventEmitter 模式,多个订阅者并存)

实际场景:Tweet ⬇️ 按钮 + Video 字幕系统 都可能同时订阅,但**都是关注全局唯一的 yt-dlp 进度**。多订阅更通用。

**推荐 B**(多订阅,callback 数组 + unsubscribe 返回 dispose 函数 — 跟 V2 onFullscreenChanged 同模式)。

### Q3:install 是否需要"已在安装中"防重入?

用户连续点 ⬇️ → install 被调多次。V1 没防,会启动多个下载并发覆盖文件。

**A. 全局锁**:main 侧记 installPromise,二次调用返回同 promise
**B. 用户层防**:UI 把按钮置灰
**C. 不防**(对齐 V1)

**推荐 A**(main 侧加保护,避免文件竞争破坏);UI 层也应该禁用按钮(B 是 UX,不是 correctness)。

### Q4:videoFilePath 安全校验

`saveSubtitle(videoFilePath, ...)` 直接接受任意绝对路径写文件,理论上可以写到任何地方。

**A. 严格白名单**:videoFilePath 必须在用户 Downloads 或某个安全目录
**B. 路径校验**:isAbsolute + 不含 `..`(基本防呆)
**C. 不校验**(对齐 V1)

**推荐 B**(基本防呆,跟 B3.14 mediaResolvePath 安全策略一致)。生产场景这个 IPC 只由我们的视频 block 调用,不暴露给用户输入,实际风险低。

### Q5:youtube-transcript npm 包

V1 在 dependencies。V2 没装。

**A. 装**:`npm install youtube-transcript`(本阶段做)
**B. 砍掉 transcript 功能**:downloader 删 fetchTranscript,字幕导出留 saveSubtitle 翻译用

**推荐 A**(沿用 V1,功能完整)。

**层级约束(P2-3)**:`youtube-transcript` 是 **ytdlp capability 内部实现细节**:
- ✅ `src/platform/main/ytdlp/downloader.ts` 内 `import { fetchTranscript } from 'youtube-transcript'`
- ❌ view 层 / driver 层 / 其他 capability **不允许**直 import `youtube-transcript`
- 字幕功能对外只通过 `YtdlpApi`(download 自动产 `subtitleFile / subtitleText` /
  saveSubtitle 写翻译)暴露,view 层无需也不该知道下层用哪个 npm 包

eslint 现有规则已经做了类似"对外屏障"约束(view/driver 不可 import 业务 npm 包),
youtube-transcript 自动落入 platform/main/* 才能用范围。本阶段不加额外 lint。

## 6. 风险

### 6.1 [低] V1 下载二进制 URL 失效

GitHub release 路径:`https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp_macos`。
yt-dlp 项目活跃,几年内不会失效,但若 yt-dlp 项目改名 / 仓库迁移会失败。

**预案**:用户层 install 失败提示;若需要,后续加 fallback URL 或镜像(本阶段不做)。

### 6.2 [低] yt-dlp 二进制 macOS-only

V1 沿用 macOS,V2 一致。Windows / Linux 用户安装会失败(URL 不匹配)。

**预案**:`platform.platform() !== 'darwin'` 时 install 直接返回 error("yt-dlp 当前仅支持 macOS,Windows/Linux 留后续")。

### 6.3 [中] yt-dlp spawn 进程的 stderr 噪音

下载某些站点(YouTube 反爬强化时段)yt-dlp 输出大量 stderr warning,parseLine 走每个 chunk。
**预案**:V1 已经处理(只 match 特定模式,非匹配忽略),直迁即可。

### 6.4 [中] 字幕 fetchTranscript 失败处理

某些 YouTube 视频禁用了字幕 / 不支持的语言 → fetchTranscript throw。
**预案**:V1 已 try/catch 忽略错误,不影响视频下载本身。直迁即可。

### 6.5 [低] yt-dlp 自身需要更新

二进制装好后跑了几个月,YouTube 反爬升级 → yt-dlp 老版本失效。

**预案**:本阶段不做自动更新。用户手动重新 install 重置二进制(或后续加 yt-dlp 内置 `-U` 命令包装)。
