## 4. 阶段中遇到 / 解决的问题

### 4.1 link mark 实质是大工程(降级处理)

**初版调研**:list 内说 link 是"小工作量"
**实际**:linkMark spec 简单(href + title attrs),但 V1 link 联动:
- LinkPanel 三 Tab(笔记/文件/网页)— 调 viewAPI.noteList / noteLoad / fileOpenDialog / mediaPutFile
- link-click plugin — 5 协议路由(krig://note / krig://block / https / file / media)+ 同文档 anchor 滚动 + 历史栈
- right-slot-routing 跨 view 路由系统
- Cmd+[/Cmd+] 历史前进/后退

**决策**:link 升格独立阶段 L5-B3.4(等 viewAPI IPC 补齐 / 跨 view 路由设计)

### 4.2 ColorPicker UI 跟 link popup 共依赖 popup 基础设施

**V1**:floating-toolbar 含 LinkPanel + ColorPicker 两个 popup(同套基础设施)
**V2**:FloatingToolbarItem 接口仅 `{label, command, activeWhen}`,**无 popup 类型**
**决策**:Plan C-1 缩水 — textStyle / highlight 落 mark spec(数据层 1:1)+ floating-toolbar 单
按钮 cycle 6 色(UI 层缩水);完整 ColorPicker UI 留 L5-B3.4 跟 link popup 共建 popup 基础设施

### 4.3 V2 BlockSpec containerRule 没为 inline 留档

**现象**:hardBreak 是 inline node(group: 'inline'),但 V2 BlockSpec.containerRule 类型是
`'leaf' | 'inline-only' | 'block+'` 三档,没有 'inline-leaf'
**临时处理**:塞 'leaf'(hardBreak 本质是叶子节点,语义勉强对)
**记录**:留 BLOCK-SPEC.md L5-B+ 协议升级时考虑加 'inline-leaf'

### 4.4 V1→V2 命名空间不互通(已知不破坏)

**V1**:div.callout / div.toggle-list(无前缀)
**V2**:div.krig-callout / div.krig-toggle-list(命名空间隔离)
**影响**:V1 doc 粘贴到 V2 不会被 parseDOM 识别(降级为 paragraph)
**处理**:V1→V2 文档兼容性 / 反序列化器统一留 L5-B3.4 评估;短期接受
