#### vocab-store.ts(JSON 文件版)

V1 SurrealDB 改 JSON 文件 + 内存 Map 缓存(对齐 mediaStore.json 索引模式):

```ts
const VOCAB_DIR = path.join(app.getPath('userData'), 'krig-data', 'learning');
const VOCAB_FILE = path.join(VOCAB_DIR, 'vocab.json');

interface VocabFile {
  version: '1';
  entries: Record<string, VocabEntry>;  // id → entry
}

class VocabStore {
  private cache: Map<string, VocabEntry> = new Map();
  private loaded = false;

  private async ensureLoaded(): Promise<void> {
    if (this.loaded) return;
    fs.mkdirSync(VOCAB_DIR, { recursive: true });
    if (fs.existsSync(VOCAB_FILE)) {
      try {
        const data = JSON.parse(fs.readFileSync(VOCAB_FILE, 'utf-8')) as VocabFile;
        if (data.version === '1' && data.entries) {
          for (const [id, entry] of Object.entries(data.entries)) {
            this.cache.set(id, entry);
          }
        }
      } catch (err) {
        console.warn('[vocab-store] load failed:', err);
      }
    }
    this.loaded = true;
  }

  private save(): void {
    const data: VocabFile = {
      version: '1',
      entries: Object.fromEntries(this.cache),
    };
    // atomic write:tmp → rename
    const tmp = VOCAB_FILE + '.tmp';
    fs.writeFileSync(tmp, JSON.stringify(data, null, 2), 'utf-8');
    fs.renameSync(tmp, VOCAB_FILE);
  }

  async add(word: string, definition: string, context?: string, phonetic?: string): Promise<VocabEntry | null> {
    await this.ensureLoaded();
    const normalized = word.toLowerCase().trim();
    if (!normalized || !definition) return null;
    const id = `${Date.now()}-${Math.random().toString(36).slice(2, 8)}`;
    const entry: VocabEntry = {
      id,
      word: normalized,
      definition,
      context,
      phonetic,
      createdAt: Date.now(),
    };
    this.cache.set(id, entry);
    this.save();
    return entry;
  }

  async remove(id: string): Promise<void> {
    await this.ensureLoaded();
    if (this.cache.delete(id)) this.save();
  }

  async list(): Promise<VocabEntry[]> {
    await this.ensureLoaded();
    return Array.from(this.cache.values()).sort((a, b) => b.createdAt - a.createdAt);
  }

  async has(word: string): Promise<boolean> {
    await this.ensureLoaded();
    const normalized = word.toLowerCase().trim();
    for (const entry of this.cache.values()) {
      if (entry.word === normalized) return true;
    }
    return false;
  }
}

export const vocabStore = new VocabStore();
```

#### handlers.ts(7 invoke + 1 推送)

```ts
// src/platform/main/learning/handlers.ts
import { ipcMain, BrowserWindow } from 'electron';
import { IPC_CHANNELS } from '@shared/ipc/channel-names';
import { vocabStore } from './vocab-store';
import { lookupWord } from './dictionary-service';
import { googleTranslate, googleTTS } from './providers/google-translate';

function broadcastVocabChanged(): void {
  vocabStore.list().then((entries) => {
    for (const win of BrowserWindow.getAllWindows()) {
      if (!win.isDestroyed()) {
        win.webContents.send(IPC_CHANNELS.LEARNING_VOCAB_CHANGED, entries);
      }
    }
  }).catch((err) => console.warn('[learning] broadcast failed:', err));
}

export function registerLearningHandlers(): void {
  ipcMain.handle(IPC_CHANNELS.LEARNING_VOCAB_ADD, async (_e, word, def, ctx, phon) => {
    if (typeof word !== 'string' || typeof def !== 'string') return null;
    const entry = await vocabStore.add(
      word, def,
      typeof ctx === 'string' ? ctx : undefined,
      typeof phon === 'string' ? phon : undefined,
    );
    if (entry) broadcastVocabChanged();
    return entry;
  });

  ipcMain.handle(IPC_CHANNELS.LEARNING_VOCAB_REMOVE, async (_e, id) => {
    if (typeof id !== 'string') return;
    await vocabStore.remove(id);
    broadcastVocabChanged();
  });

  ipcMain.handle(IPC_CHANNELS.LEARNING_VOCAB_LIST, () => vocabStore.list());
  ipcMain.handle(IPC_CHANNELS.LEARNING_VOCAB_HAS, async (_e, word) => {
    if (typeof word !== 'string') return false;
    return vocabStore.has(word);
  });

  ipcMain.handle(IPC_CHANNELS.LEARNING_LOOKUP, (_e, word) => {
    if (typeof word !== 'string' || !word) return null;
    return lookupWord(word);
  });

  ipcMain.handle(IPC_CHANNELS.LEARNING_TRANSLATE, (_e, text, targetLang) => {
    if (typeof text !== 'string' || !text) return null;
    return googleTranslate(text, typeof targetLang === 'string' ? targetLang : 'zh-CN');
  });

  ipcMain.handle(IPC_CHANNELS.LEARNING_TTS, async (_e, text, lang) => {
    if (typeof text !== 'string' || typeof lang !== 'string' || !text) return null;
    const buf = await googleTTS(text, lang);
    if (!buf) return null;
    // Buffer 转 ArrayBuffer 给 renderer(IPC 序列化兼容)
    return buf.buffer.slice(buf.byteOffset, buf.byteOffset + buf.byteLength);
  });
}
```

## 3. 决策点

### Q1:vocab JSON 文件原子写

`fs.writeFileSync` 直接覆盖有"写一半挂掉文件损坏"风险。
- **A. tmp + rename**(原子 — 推荐):写到 `vocab.json.tmp` 再 `fs.renameSync`(POSIX 保证原子)
- **B. 直接覆写**:简单,小文件(几 MB)+ 短写时间(< 100ms)风险低

**推荐 A**(对齐 mediaStore.json 同模式;JSON 文件几 KB ~ 几 MB,损坏代价高 — 用户半年生词全没)。

### Q2:vocab broadcast 给所有 BrowserWindow 还是仅 mainWindow

V1 走多 view 架构(每个 ws 独立 BrowserWindow),broadcast 遍历 children。
V2 单 BrowserWindow,但理论上以后可能多窗口。

- **A. 遍历 BrowserWindow.getAllWindows()**(V1 等价):未来多窗口也 OK
- **B. 仅 mainWindow**(getMainWindow):V2 当前简单

**推荐 A**(防御性,无成本)。

### Q3:tts 返回 ArrayBuffer 还是 base64 字符串

main 进程 net.fetch 拿 `Buffer`,IPC 传 renderer 需要选格式。

- **A. ArrayBuffer**:二进制原生,renderer 直接 `new Blob([ab])` 创建 audio URL,小开销
- **B. base64 字符串**:跨平台 IPC 兼容性最好,但传输大小 4/3 倍

**推荐 A**(Electron IPC 支持 ArrayBuffer/Uint8Array 直传)。

### Q4:google providers 内部 import

V1 `google-translate.ts` 用 `import { net } from 'electron'`。V2 main 进程同样可用。

- **A. 同 V1**:`import { net } from 'electron'` — 简单
- **B. 注入 fetch**:让 service 不直接耦合 electron(可测试)

**推荐 A**(直迁 V1,不过早抽象;V2 现有 mediaStore 也直接用 `net`)。

### Q5:macOS-only 提示

macos-dictionary 在非 macOS 平台返 null。dictionary-service 走 fallback Google。
但用户体验上不知道为什么没词典 — 是否要加日志 / 提示?

- **A. 静默**(对齐 V1,fallback 自然降级)
- **B. 一次性 console.warn**:启动检测一次平台,非 macOS 警告 "macOS dict 不可用,用 Google fallback"

**推荐 A**(用户体验上看到 Google 译文也是有结果,无需额外提示)。

## 6. 风险

### 6.1 [低] swift CLI 在打包后路径

V1 用 `swift -e <SCRIPT>`,需要系统装 Xcode Command Line Tools。普通用户没装 → spawn 失败 → fallback Google。

**预案**:macos-dictionary try/catch 兜底返 null(已有);用户安装 Xcode CLT 后自动可用。

### 6.2 [中] Google 翻译被反爬

V1 用 `translate.googleapis.com` 免费端点,某些 IP 段会被限流(429 / 403)。

**预案**:googleTranslate 失败返 null;fallback null 让 UI 层做"翻译失败"提示。

### 6.3 [低] vocab.json 损坏 / 丢失

ensureLoaded JSON.parse 失败 → console.warn + 内存缓存空(等于"全没")。
**预案**:atomic write 防写损坏;用户备份 vocab.json 自管(导入导出 UI 留 Phase D)。

### 6.4 [低] vocab 数量过大(几万条)

JSON 文件每次写全量 → IO 增大 + 内存 Map 占用增。**几千条以下毫秒级 IO**。
**预案**:当前不优化;用户量超 5000 时再加分片或迁 DB(memo 项)。

### 6.5 [低] tts 大文本分块拼接

V1 splitTextForTTS 对 > 180 字符按句子分块,逐块 fetch 拼 Buffer。**网络慢时会卡数秒**。
**预案**:UI 层加 loading 状态(20b dictionary-panel 处理),本段不优化。
