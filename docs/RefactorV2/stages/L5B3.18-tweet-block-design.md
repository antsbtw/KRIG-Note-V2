## 0. 背景

### 0.1 V1 状态

V1 tweet-block(`src/plugins/note/blocks/tweet-block.ts` 456 行)+ scraping(`src/plugins/web/main/ipc-handlers.ts:1335-1500` ~170 行 fetchTweetData + ~80 行 EXTRACT_TWEET_JS)。

**V1 NodeView 完整能力**(456 行直迁):
- 双 Tab UI(Browse iframe / Data 离线缓存卡片)
- Fetch 按钮 — 调 fetchTweetData,抓元数据填 attrs(authorName / metrics / 引用 / 媒体 / 时间 等)
- Download 按钮(⬇️)— 调 ytdlp.download 下载推文里的视频
- iframe postMessage 自适应高度(twttr.private.resize)
- 打开原文(Open original ↗)
- Caption(PM contentDOM)

**V1 main 侧 fetchTweetData**(170 行 + 80 行 EXTRACT_TWEET_JS):
- 创建隐藏 BrowserWindow(800x900,show:false,nodeIntegration:false,contextIsolation:true)
- loadURL(tweetUrl) → 等 SPA 渲染(轮询最多 10 秒,检查 `article[data-testid="tweet"]`)
- executeJavaScript(EXTRACT_TWEET_JS) — 基于 Twitter `data-testid` 属性 DOM scraping:
  - User-Name / Tweet-User-Avatar(头像 / 名字 / @handle)
  - tweetText(正文 + lang)
  - time(createdAt)
  - tweetPhoto / video(媒体 URL)
  - role="group"(metrics:replies / retweets / likes / views)
  - quoteTweet / socialContext(引用 / inReplyTo)
- finally:win.destroy()

### 0.2 V2 现状

- ytdlp capability 已落(L5-B3.17)
- audio/video block 模式确立(`platform/main/<module>/` + `capabilities/<module>/`)
- 没有 browser-capability(audit § 5.1 列为 Phase D 项,~7600 行)
- W5 严格态 A 已生效(view 必须 requireCapabilityApi 间接路由)

### 0.3 方案 A 核心约束

**最小化 = 只迁 tweet 自身需要的 scraping**:
- 新建 `src/platform/main/tweet-fetcher/` — **临时模块**,仅做 fetchTweetData 一个事
- 新建 `src/capabilities/tweet-fetcher/` — capability 门面 + Registry 注册

  统一搬家或被吸收",避免临时能力长期化(用户红线)

**不**做的:
- ❌ 建 `capabilities/browser-capability/` 抽象层(等 Phase D)
- ❌ 写通用 BrowserWindow scraping 框架(Lease / PageRegistry / 等 — V1 那 7600 行的雏形)
- ❌ 字幕系统(WebVTT cue / CC / 翻译 — 留 L5-B3.19)
- ❌ Twitter oEmbed API 路径(V1 实现了但 NodeView 没用,留 Phase D)

## 4. 决策点

### Q1:fetchTweetData 失败处理

V1 BrowserWindow 创建 + loadURL 可能失败(网络断 / Twitter 反爬 / SPA 没渲染好)。

**A. 静默失败**:UI 显 "Fetch failed" 红字提示
**B. 抛错让 view 兜**:NodeView try/catch
**C. 同 V1**:result.success 字段 — `{ success:false, error: string }`

**推荐 C**(对齐 V1 + 跟 ytdlp 的返回模式一致)。NodeView 检查 result.success,失败时 Fetch 按钮显 ❌ + 2s 后恢复。

### Q2:Download 按钮的 ytdlp 安装提示

V1:用户首次点 ⬇️ 时 — 没装 → 提示 → 装 → 装完 → 再点 ⬇️ 真下载。两次点击。

**A. V1 行为**(两次点击):简单
**B. 一次点击**:首次点 ⬇️ 直接进入"装 → 立即下载"(用户感知"点了 ⬇️ 就开始")
**C. 启动时静默装**:打开 note 时检查 ytdlp,没装就静默装(用户无感)

**推荐 A**(对齐 V1 + 用户控制权)。NodeView 实现:`if (!status.installed) { install(); return; }`(第一次点装,装完用户再点真下载)。

### Q3:Fetch 时的 NodeSelection 状态

V1 Fetch 按钮 mousedown 时 e.preventDefault() + e.stopPropagation(),防止 PM 把 NodeView 选中。

**A. 同 V1**(prevent + stop):简单
**B. 不拦,让 PM 自然选中**

**推荐 A**(对齐 V1,Fetch 是动作不是选中行为)。

### Q4:iframe 自适应高度

V1 监听 window message 拿 Twitter postMessage `twttr.private.resize` 调 iframe.style.height。

**A. 直迁**:V1 行为对齐
**B. 砍掉,固定 max-height**:简单但 UX 差(推文长短不一)

**推荐 A**(直迁,iframe 体验好)。需注意 destroy 时 removeEventListener('message')(V1 已做)。

### Q5:Browse Tab default vs Data Tab default

V1 attrs.activeTab default `'browse'`。

**A. 同 V1**:browse 优先(用户首次看官方推文)
**B. 改 data 优先**:已有 attrs 时 data Tab(离线优先)

**推荐 A**(对齐 V1)。但 NodeView 渲染时若没有 tweetUrl,placeholder 直接显;有 tweetUrl 但没 attrs 数据,Browse Tab(iframe)是默认。

### Q6:CSP frame-src

V2 CSP 已有 `frame-src 'self' https://www.youtube.com https://www.youtube-nocookie.com`(L5-B3.16 加的)。Twitter iframe 走 `https://platform.twitter.com/embed/Tweet.html`。

**A. 加 platform.twitter.com 白名单**:必须,否则 iframe 被 CSP block
**B. 不加**

**推荐 A**(必须加,否则 iframe 不能加载)。

## 7. 风险

### 7.1 [中] Twitter SPA 反爬升级

EXTRACT_TWEET_JS 基于 `data-testid="tweet"` / `User-Name` / `tweetText` 等选择器。Twitter 改名 / 改结构 → scraping 失败。

**预案**:
- fetchTweetData 失败时 success:false + error,UI 友好降级到 Browse iframe(不影响阅读)
- 选择器更新走小补丁 commit,不阻塞主功能

### 7.2 [低] BrowserWindow 创建失败

某些环境(打包后 / 沙盒限制)BrowserWindow 创建可能失败。

**预案**:fetcher.ts try/catch 兜底返 success:false + error。

### 7.3 [中] iframe postMessage 跨 origin 安全

监听 window message 接收推文 iframe 的 resize 事件。**只过滤 event.source === iframe.contentWindow**(V1 已防)— 第三方 message 不影响。

### 7.4 [低] CSP frame-src 跟 platform.twitter.com

L5-B3.16 加了 youtube,本阶段加 twitter。CSP meta 累加合理,无副作用。

### 7.5 [中] 临时能力长期化(用户红线)

**预案**(用户红线):
- tweet-fetcher DESIGN.md 顶部 ⚠️ banner 标"临时 capability,Phase D 正式化后被吸收"
- 后续审计**专项检查** tweet-fetcher 是否被新功能扩展(任何扩展走 Phase D 通道)
- audit 文档加一行"tweet-fetcher 临时能力,跟踪 Phase D"

### 7.6 [低] Twitter 短链(t.co)

V1 NodeView extractTweetId 正则只匹配 twitter.com / x.com — 不支持 t.co 短链。
**预案**:同 V1 限制,不优化。
