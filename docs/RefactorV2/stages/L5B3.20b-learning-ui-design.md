### 0.2 关键架构决策

#### dictionary-panel 走 popup-registry(对齐 V2 LinkPanel / NoteLinkSearch / ColorPickerPanel 模式)

V1 用自建的 help-panel-core 做"互斥外部面板"框架,V2 没有 help-panel,但有
`popup-registry / popup-controller / PopupBinding`(L5-B3.4 落地,LinkPanel /
NoteLinkSearch / ColorPickerPanel 已用)。

popup-registry 的契约**完全适配** dictionary-panel:
- anchor-positioned 弹层(选词时用鼠标位置作 fake anchor,对齐 NoteLinkSearch 模式)
- 同时只一个 popup(controller 单例)
- 点外/Esc 关闭(已有)
- React 组件由 view 提供(本段写 `DictionaryPanel.tsx`)

V2 version:**改写为 React 组件**(对齐 LinkPanel.tsx)。V1 是纯 DOM,因为当时 view 不强求 React;V2 既然走 popup-registry,自然走 React。

#### vocab-highlight-plugin 是 PM plugin → driver 层

vocab-highlight-plugin 是 ProseMirror plugin(decorations + handleDOMEvents),走 driver 内部:
- `src/drivers/text-editing-driver/plugins/build-vocab-highlight-plugin.ts`(driver 内部 plugin builder)
- `editor-view-builder.ts` 的 plugins 链装上
- driver 内部消费 `@capabilities/learning`(W5 严格态 A 允许)

vocab 词表更新的数据流:
```
vocabStore (main) → broadcast LEARNING_VOCAB_CHANGED
  → renderer onVocabChanged(callback)
  → callback 调 driver 暴露的"更新所有 PM 实例 vocab"接口
  → driver 遍历 instanceRegistry → 每个 view dispatch vocabHighlightPluginKey meta
  → plugin apply 重建 decorations
```

driver 层加 setVocabWords(words) API(对齐 setLinkClickHandler 模式)。

#### 触发点 1:右键菜单(选词 / 选句子)

V1 ContextMenu 加两项:
- `查词`(选区 ≤ 1 词时显)→ showDictionaryPanel(word, contextSentence)
- `翻译`(选区 > 1 词时显)→ showTranslationPanel(text)

V2 走 `contextMenuRegistry` + 加新 `enabledWhen` 条件(`'has-word-selection'` / `'has-text-selection'`?或者用 `has-selection` 简化)。

**简化**:统一用 `'has-selection'` 条件,菜单内部判断选区单词 vs 多词:
- 单词:`查词`(panel 走 lookup 模式)
- 多词:`翻译`(panel 走 translate 模式)

更朴素:**两项都用 has-selection,各自判断**:
- `📖 查词`:enabledWhen='has-selection',内部判断单词才显
- `🌐 翻译`:enabledWhen='has-selection',内部判断多词才显

V1 NoteView contextMenu 已有 visibleWhen 支持(参考 has-link),加两个新 enabledWhen 枚举值 `'is-single-word-selection'` / `'is-multi-word-selection'`。

但其实**简化最大化**:两项都用 `'has-selection'`,**显示并存,用户挑**(单词查词,多词翻译)— 用户选错 panel 内部容错(单词翻译也能跑,只是质量降级)。

#### 触发点 2:vocab broadcast 自动更新

renderer 启动时 + onVocabChanged 触发时,把 vocab 词表 dispatch 给所有 PM 实例。

放在哪里订阅?
- **A. note-view/index.ts 副作用**:学习能力是 note 业务,view 注册时启动订阅
- **B. driver 层**:driver 自己订阅 capability,view 不动
- **C. capability 层** vocab-highlight-plugin:plugin 内部订阅(模块级 listener)

V2 既有约定下,**A 最合理**(view 是消费方,view 启动时订阅 capability;driver 只暴露 setVocabWords,不直接 import learning capability — driver/learning 跨域依赖避免)。

但实际 W5 严格态 A 允许 driver 内部 import capability(如 ytdlp / tweet-fetcher),所以 **C 也合理**。

**推荐 A**:view 层订阅 → 调 driver `setVocabWords` API → driver 内部分发给所有 PM 实例。driver 不直接 import learning capability,职责单一。

  // 1. 更新 vocabDefs(给 tooltip 用)

### 2.2 view:DictionaryPanel.tsx(React 改写)

`src/views/note/dictionary-panel/DictionaryPanel.tsx`:

```tsx
import { useEffect, useMemo, useState } from 'react';
import type { PopupCloseProps } from '@slot/interaction-registries/popup-registry/popup-types';
import { requireCapabilityApi } from '@slot/capability-registry/get-capability-api';
import type {
  LearningApi,
  DictionaryLookupResult,
  VocabEntry,
} from '@capabilities/learning/types';
// W5-A View 边界严格态:view 通过 requireCapabilityApi 间接路由,
// 不直 import @capabilities/learning 运行时函数(types 是类型 only,纯擦除可 import)

// 模块级"待显示"状态(由 showDictionaryPanel / showTranslationPanel 设定后由 popup 渲染时读)
let pendingMode: 'lookup' | 'translate' = 'lookup';
let pendingText = '';
let pendingContext: string | undefined;

export function setPanelInitial(mode: 'lookup' | 'translate', text: string, context?: string): void {
  pendingMode = mode;
  pendingText = text;
  pendingContext = context;
}

export function DictionaryPanel({ onClose }: PopupCloseProps) {
  // W5-A:间接路由取 capability API
  const learning = useMemo(() => requireCapabilityApi<LearningApi>('learning'), []);

  const [tab, setTab] = useState<'lookup' | 'vocab'>('lookup');
  const [mode] = useState(pendingMode);
  const [text] = useState(pendingText);
  const [context] = useState(pendingContext);
  const [lookup, setLookup] = useState<DictionaryLookupResult | null>(null);
  const [translation, setTranslation] = useState<string | null>(null);
  const [loading, setLoading] = useState(true);
  const [vocab, setVocab] = useState<VocabEntry[]>([]);
  const [filter, setFilter] = useState('');

  // 初次加载:并行 lookup + translate(lookup 模式)/ 仅 translate(translate 模式)
  useEffect(() => {
    if (mode === 'lookup') {
      Promise.all([learning.dictionaryLookup(text), learning.translate(text)]).then(([d, t]) => {
        setLookup(d); setTranslation(t?.text ?? null); setLoading(false);
      });
    } else {
      learning.translate(text).then((t) => { setTranslation(t?.text ?? null); setLoading(false); });
    }
  }, [mode, text, learning]);

  // 切到 vocab Tab 时取列表
  useEffect(() => {
    if (tab === 'vocab') learning.vocabList().then(setVocab);
  }, [tab, learning]);

  // 用户点 + 添加到生词本时调 learning.vocabAdd / 删除调 learning.vocabRemove /
  // 🔊 按钮调 learning.tts 创 Blob URL 播放
  // 渲染 ... (双 Tab + 添加生词本 + TTS + 删除 + 搜索过滤)
}
```

### 2.4 view 启动 learning-integration

`src/views/note/learning-integration.ts`(NEW):

```ts
import { textEditingDriverApi } from '@drivers/text-editing-driver';
import { requireCapabilityApi } from '@slot/capability-registry/get-capability-api';
import type { LearningApi } from '@capabilities/learning/types';
import { popupController } from '@slot/triggers/popup-controller';
import { setPanelInitial } from './dictionary-panel/DictionaryPanel';

const POPUP_ID = 'note-view.popup.dictionary';

export function registerLearningIntegration(): void {
  // W5-A 间接路由(view 层强制经 Registry,types 是类型 only 可 import)
  const learning = requireCapabilityApi<LearningApi>('learning');

  // 1. 启动取一次全量 → 分发到 PM
  learning.vocabList().then((entries) => {
    textEditingDriverApi.setVocabWords(entries);
  });

  // 2. 订阅变化(任何 add/remove 都推全量 list)
  learning.onVocabChanged((entries) => {
    textEditingDriverApi.setVocabWords(entries);
  });
}

/** 给 contextMenu cm-dictionary-lookup / cm-translate-text 命令调 */
export function showDictionaryPanel(word: string, context?: string, anchorRect?: DOMRect): void {
  setPanelInitial('lookup', word, context);
  showAtAnchor(anchorRect);
}

export function showTranslationPanel(text: string, anchorRect?: DOMRect): void {
  setPanelInitial('translate', text);
  showAtAnchor(anchorRect);
}

function showAtAnchor(rect?: DOMRect): void {
  const fake = document.createElement('div');
  fake.style.position = 'fixed';
  if (rect) {
    fake.style.left = `${rect.left}px`;
    fake.style.top = `${rect.bottom}px`;
  } else {
    // fallback:屏幕中心
    fake.style.left = `${window.innerWidth / 2}px`;
    fake.style.top = `${window.innerHeight / 2}px`;
  }
  fake.style.width = '1px';
  fake.style.height = '1px';
  fake.style.pointerEvents = 'none';
  document.body.appendChild(fake);
  popupController.show(POPUP_ID, fake);
  window.setTimeout(() => fake.remove(), 0);
}
```

## 3. 决策点

### Q1:dictionary-panel 用 popup-registry vs help-panel

V1 用 help-panel-core(自定义互斥外部面板框架)。V2 没 help-panel。

- **A. popup-registry**:对齐 V2 既有(LinkPanel / NoteLinkSearch / ColorPickerPanel)
- **B. 新建 help-panel-core**:对齐 V1
- **C. 直接 DOM body**:简单但绕过 V2 既有架构

**推荐 A**:popup-registry 完全合适(anchor-positioned + 单例 + 点外/Esc 关闭),无需新建框架。help-panel 留 Phase D 处理 latex 速查 / mermaid 速查 等真正"长期常驻"需求。

### Q2:V1 纯 DOM → V2 React?

V1 dictionary-panel 是纯 DOM(`document.createElement` + innerHTML),原因是 V1 view 没强制 React。

- **A. React 改写**:对齐 V2 popup 模式(LinkPanel.tsx / NoteLinkSearch.tsx 等);
  - 优点:统一风格,JSX 比 innerHTML 易读
  - 缺点:436 行 V1 → ~400 行 React,改写工作
- **B. 保持纯 DOM**:V1 直迁,但跟 V2 popup 风格不一致

**推荐 A**(对齐 V2 既有 popup 都 React)。

### Q3:vocab-highlight-plugin 装载位置

- **A. driver/plugins/**(对齐 build-link-click-plugin / build-slash-plugin 等):
  driver 内部 plugin 体系
- **B. 单独抽出**:capability/learning/plugin?(违反"capability 不依赖 PM" 边界)

**推荐 A**(driver 内部最合理,vocab-highlight 跟 link-click 同性质)。

driver 内部直 import `@capabilities/learning`(W5 严格态 A 允许)。

### Q4:vocab broadcast 订阅位置

- **A. view 层 learning-integration**:view 是消费方,view 启动订阅 → 调 driver setVocabWords
- **B. driver 模块级**:driver 自己订阅 capability,view 不参与
- **C. capability 模块级**:learning capability 自己订阅自己 broadcast(?递归)

**推荐 A**:driver 不直接 import learning capability(避免 driver/learning 跨域强耦合),view 层做协调。

实际上 vocab-highlight-plugin 内部消费 tts(B3.19.1 类似设计 — driver 内部直 import capability 也允许),但订阅 onVocabChanged 仍走 view 层(职责更清晰)。

### Q5:contextMenu 项条件

- **A. has-selection 统一**:两项都显,用户选(单词/多词都能用)
- **B. has-single-word vs has-multi-word**:精确条件,但要扩展 EnabledWhen 枚举

**推荐 A**(简单 + 容错,V1 实际行为也是两项都显)。Phase D 真有体验问题再加细分。

### Q6:tooltip 位置 — popup-registry 还是模块级 DOM

vocab-highlight 的 hover tooltip 只是个小 hover 提示(显释义 + TTS 按钮),不是"打开面板"那种半受控弹层。

- **A. 模块级 DOM**(对齐 V1):document.body 加临时 div,生命周期模块级
- **B. popup-registry**:复用基础设施

**推荐 A**(tooltip 是即用即关,popup-registry 单例特性会冲突 — 用户 hover 不同生词时 popup-registry 切换会闪)。模块级 DOM 简单合适。

## 6. 风险

### 6.1 [低] popup-registry fake-anchor 模式

跟 NoteLinkSearch 一样用临时 fake div 作 anchor,有先例,无新风险。

### 6.2 [中] vocab broadcast → setVocabWords 性能

文档很长(几千 block)+ vocab 几百词时,每次 vocab 变化(add/remove)→ broadcast → 重建 decorations。
buildDecorations 走 doc.descendants + RegExp 全文扫,**长 note ~50ms**(可接受)。

**预案**:本段不优化;Phase D 真有性能问题加增量更新(只扫变化的 block)。

### 6.3 [中] V1 ContextMenu 的 contextSentence 提取

V1 单词查词时取"整句"作 context 传给 vocabAdd。V2 简化暂不传,**只传 word + definition**(对齐 vocabStore.add 的 context 字段可选)。

**影响**:vocab.json 里 entry 缺 context 字段(可选)— 不影响功能,只是后续看 vocab 列表时少一行"句子上下文"。

**Phase D 增强**:从 PM doc selection 取所在 block 的 textContent 作 context(需要从 view → driver 取选区 anchor block — 不复杂,但本段不做)。

### 6.4 [低] tooltip 位置溢出 viewport

V1 已处理(`getBoundingClientRect` + 边界翻边)。直迁。

### 6.5 [低] TTS 多次点击

V1 模块级 ttsAudio + ttsObjectUrl,下次点先 pause + revoke。直迁。
