## 3. B3.19 整体验收清单(a~e 五段统一,用户新工作流约定)

按用户拍板的新工作流,a/b/c/d/e 5 段不单独验收,本段 e merge 后给完整清单。请按顺序点检:

### 3.1 启动 + a 段 字幕底座

1. `npm start` → console 无报错
2. 任一笔记新建 video block(/video 或粘贴 URL)→ 显 🎞 placeholder
3. URL 输入 `https://www.youtube.com/watch?v=dQw4w9WgXcQ` Enter → block 形态变化:
   - 顶栏显 `[Video] [Meta] [EN]` tabs(Video active 蓝色下划线)
   - 右侧 actionBar 显完整 7 个按钮:**`CC ⏮ 🧠 ⏭ 🌐 📝 ⬇ 📖 ⛶`**
4. 切 `Meta` Tab → 显 Title 输入框 + Source readonly + Download 状态行(初始 "未下载")
5. 切 `EN` Tab → 显 textarea + 占位提示
6. 在 EN 贴入:
   ```
   [00:05] First subtitle line
   [00:30] Second line
   ```
7. 切回 Video,点播放,**默认 CC 关浮层不显**
8. 点 [CC] → dropdown 显 [EN] [OFF];选 EN → CC 变 "CC✓",浮层启
9. 播到 5s+ 显 "First subtitle line";30s+ 切 "Second line"
10. [CC] → OFF → 浮层立即消失
11. [⛶] → 全屏(字幕仍同步)
12. **直接 mp4 视频**(http(s) / media://):同样 Tab 框架 + CC 体验

### 3.2 b 段 字幕产线

13. YouTube Tab `EN`:点 [📝] → ⏳ → 字幕自动填入 textarea + 切到 `EN` Tab
    (如果视频禁字幕 / API 失败 → ❌ 显 2s 回 📝,tooltip 显错误)
14. 点 [🌐] → ⏳ → 创建 `ZH-CN` Tab + textarea 显中文翻译 + 自动切到 ZH Tab
15. CC dropdown 现在显 `[EN] [ZH-CN] [OFF]`
16. CC 选 ZH-CN → 浮层显中文字幕
17. 切笔记切回 → translationTexts 持久化(ZH Tab 仍在)

### 3.3 c 段 Memory Mode

18. actionBar 点 [🧠] → dropdown 显 `30s / 60s / 90s / 120s`
19. 选 60s → 视频 seekTo 0 开始播,🧠 文字变 `🧠 1/N`(N = ceil(duration/60))
20. 播到 60s 边界附近自动 advance,seek 到段 0 重播段 [0,1]
21. 点 [⏭] 跳下一 step;[⏮] 跳上一 step
22. 再点 [🧠] → stop;⏮/⏭ disabled
23. 切笔记切回 → segmentDuration + memoryLastStep 持久化(active 不持久,需重新点 🧠 启动)

### 3.4 d 段 Vocab Panel

24. 点 [📖] → 视频右上角显 vocab panel(220x240 半透明卡片)
25. 在生词本(右键查词加词)加几个英文词,且这些词在 transcript 里有出现
26. vocab panel 显 ±5 entry,active 高亮跟随 currentTime 推进
27. 在生词本删词 → vocab panel 立即刷新(onVocabChanged 触发)
28. 编辑 transcript textarea → vocab panel 重 build(若可见)
29. 再点 [📖✓] → 隐藏

### 3.5 e 段 Download + Meta(本段)

30. YouTube 视频:点 [⬇] →
    - 若 yt-dlp 未装 → ⏳ → 装完 → 回 ⬇(再点开始下载)
    - 已装 → ⏳ + 顶部进度条显进度 + Meta Tab Download 行显 "下载中... 45%"
31. 完成后 ⬇ 变 📁(绿色),进度条隐
32. Meta Tab Download 行变绿色,显本地路径
33. 点 [📁] → Finder 打开文件位置
34. **transcript 自动填**:若下载前 EN textarea 为空 → 完成后自动填字幕(不覆盖已有)
35. **翻译 .srt 落盘**:若下载前已有 ZH-CN 翻译 Tab → 完成后 `<videofile>.zh-CN.srt` 静默生成
36. 切笔记切回 → localFilePath 持久化(再开仍显 📁)
37. **直接 mp4 视频**:[⬇] 灰显(仅 YouTube),direct video 用 video 元素自带的下载按钮(B3.16 既有 mediaDownload)

### 3.6 内存安全 / 错误恢复

38. 切笔记后 console 跑 `document.querySelectorAll('.krig-video-block').length` → 0(NodeView destroy 干净,无内存泄漏)
39. 下载失败(关网测试)→ ⬇ 显 ❌ 2s 回 ⬇(可重试)
40. install 失败 → 同上恢复

### 3.7 跟其他功能不冲突

41. CC 浮层(底部居中)跟 vocab panel(右上)同时显示不挤压
42. Memory Mode 跟字幕 / vocab panel 三者同时启用,300ms 单源轮询不重复(time-tracker 共享)
43. 多视频实例:开 2 个 YouTube video block 各自下载,互不干扰(progress 按 url 过滤)

## 4. 已知短板

### 4.1 Vimeo / generic embed 不渲染
显占位"暂不支持(Phase D)"。Phase D 需做 iframe sanitizer 才能开。

### 4.2 ytdlp.download 返回本地路径不入 mediaStore
Q-e-1 决策落实:src 不切 media://,localFilePath 仅记录。如要桥接到 mediaStore 需另做"ytdlp → mediaPutFile"capability 协作,留 Phase D 评估。

### 4.3 download 撤销 / 协作不主动同步 button 内部状态
update() 仅同步 dataTab 显示;downloadBtn 内部 phase 不重 sync。少见场景,用户重点 ⬇ 即恢复正确状态。

### 4.4 YouTube autoplay policy
Memory Mode 调 play() 在 iframe 上可能被 Chrome 86+ 静音 / 阻拦。需用户先手点 iframe 一次。**接受**(V1 已踩过)。

### 4.5 attrs 体积膨胀
translationTexts(多语言全 transcript)+ transcriptText 持久化到 PM doc。500 cues × 5 语言可能 ~15KB。可接受范围,后续如出问题加 gzip。
