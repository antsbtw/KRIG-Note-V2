## 1. 实际改动 vs 设计 — 偏差点

### 1.1 重构 image insertImageAtSelection 复用通用辅助

设计 § 6.3 提到"考虑抽 insertWithCaptionBlock 共享"。**实际**做了:
- image / audioBlock / videoBlock 三个 insert API 都调 `insertWithCaptionBlock(view, nodeName)`
- 原 image 实现的 ~40 行内联逻辑被抽走,行为完全等价(空段落替换 / 否则段后插入 / 光标进 caption)
- 副作用:image 也加了 isTitle 守卫(原代码缺,本次顺手加)— 跟 insertAtomBlock 一致

### 1.2 NodeView update 加局部刷新优化

设计的 update 一律 paint 整体重渲。**实际**:
- src/embedType 变 → 整体重渲
- 仅 title 变 → 只更新 .krig-{audio,video}-block__title 的 textContent
  避免 audio/video 元素重建(用户感觉播放卡一下),局部更新更优雅

### 1.3 stopEvent 放行播放器控件

PM NodeView 默认会拦截内部 DOM 事件。audio/video 控件(play/pause/volume slider)
需要 PM 不接管。`stopEvent` 返回 true 让 PM 跳过该事件。

### 1.4 dispose 释放音视频资源

NodeView destroy 时调 `disposeAudio() / disposeVideo()`(pause + src=''),
对齐设计 § 10.2 风险预案。**实测**:切笔记 / 删 block 都能立即停播放(无残留声音)。

## 3. 用户验证清单

`npm run dev`,Note view,以下逐项测:

### audioBlock 主流程(judge #2 - #8)
1. 打开任一现有笔记 — 不白屏
2. 输入 `/` → slash menu 含 "🎵 Audio"
3. 选中插入 audioBlock placeholder(🎵 + Upload + URL 输入框)
4. 点 Upload → 选一个 mp3 → 切 player 态(title + audio 控件 + 可编辑 caption)
5. 点 audio 播放/暂停 — 正常播放(浏览器原生 controls)
6. F12 看 audio[src] — 是 `media://...`
7. **完全退出 + 重启应用 + 重开笔记** — audio 仍可播放
8. 再插一个 audioBlock → URL 输入 `https://www.soundjay.com/free-music/sounds/the-lounge-1.mp3`(或任意 https mp3)→ Enter
   - 切 player 态 + 显 ⬇ 下载按钮
9. 点 ⬇ → 显 ⏳ → 完成后 ✅ → 自动恢复 ⬇
   - F12 看 audio[src] — 已变成 `media://...`
   - ⬇ 按钮**消失**(因为已经是 media://)

### videoBlock 主流程(#9 - #12)
10. slash → "🎞 Video" → 插入 videoBlock placeholder
11. 上传 mp4 → 切 direct 播放器(`<video controls>` 16:9 区域)
12. 再插一个 → URL 输入 `https://www.youtube.com/watch?v=dQw4w9WgXcQ` → Enter
    - 切 youtube embed(`<iframe>` 16/9 比例)
    - **YouTube 视频能正常播放**(若 CSP block 见风险 § 10.1)
13. URL 输入 `https://youtu.be/<id>` 短链 — 也识别为 youtube
14. URL 输入 `.../watch?v=<id>?something=else` — youtube 解析出 ID OK

### caption(PM contentDOM,#13)
15. audio/video player 下方有 caption 区(灰色斜体 placeholder)
16. 点 caption → 可写文字(PM 接管,**支持 mark 富文本如 bold/italic**)
17. caption 内 Enter 跳出 audio/video,在外面新段落继续写

### destroy 释放资源(#14)
18. audio 播放中切到另一个笔记 → 立即停止(无残留声音)
19. 删 audio block(handle 菜单 → 删除 Block)→ 也立即停

### 边界
20. 选中已有 audio/video block → handle 菜单可移到其他位置(draggable=true)
21. block 选中时蓝色外框(ProseMirror-selectednode 样式)

## 4. 已知约束 & 后续优化

### 4.1 Vimeo / generic embed 留 Phase D

V2 video 仅支持 youtube + direct,Vimeo / 其他平台 embed 留 D 阶段(用户少 +
iframe 安全 sanitizer 设计成本)。

### 4.2 字幕系统留 Phase D

V1 video 有完整字幕(WebVTT cue / CC 浮层 / YouTube transcript / 翻译),~700 行,
全留 Phase D(等 thought + AI 集成成型)。

### 4.3 大文件 FileReader 阻塞

50MB+ 视频上传 readAsDataURL 会卡 1-3s。本阶段不优化(同 paste-media / fileBlock);
Phase E 改 ArrayBuffer + Blob URL。

### 4.4 audio/video block 暂无 thumbnail / poster

V2 不预留 poster attr(决策 Q1 = A)— 未来加缩略图提取无破坏性 schema 升级。

### 4.5 YouTube iframe CSP

风险 § 10.1 — 若被 block 需 main 进程 webRequest 加白名单。**待用户实测确认**。
