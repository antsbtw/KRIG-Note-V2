### 1.1 W5-A 边界遵守

driver 层允许直 import capability(对齐 B3.20b 现有约定)。涉及:
- `actions/translate-button.ts` → `import { translate } from '@capabilities/learning'`
- `actions/transcript-button.ts / download-button.ts` → `import * as ytdlp from '@capabilities/ytdlp'`
- `components/vocab-panel.ts` → `import { vocabList, onVocabChanged } from '@capabilities/learning'`(**本地订阅,特例旁路** — 见下方说明)

view 层(`src/views/note/...`)本期不需要任何新文件 — driver NodeView 自包含。

#### 跟 B3.20b vocab 订阅链路的口径分离(避免审计争议)

B3.20b 已经存在一条 vocab 订阅链路,职责是"vocab 变化 → driver.setVocabWords → 全 PM instance 重建 decoration"(给所有 NoteView 文档高亮服务)。
**本期 video-block 内部 `components/vocab-panel.ts` 直接订阅 `learning.onVocabChanged` 是另一条独立链路**,职责是"vocab 变化 → 单个 video-block 实例的 timeline 卡片刷新"。

| 维度 | B3.20b 订阅链(已有) | 本期 video-block 订阅链(新增) |
|---|---|---|
| 触发位置 | `src/views/note/learning-integration.ts`(view 层 integration) | `components/vocab-panel.ts`(driver 层 NodeView 子模块) |
| 消费者 | driver.setVocabWords → instanceRegistry → 全 PM 实例 decoration | 单个 video-block NodeView 实例的 vocab-panel 局部刷新 |
| 生命周期 | view register 时启动,view 范围 | NodeView mount 时订阅 + destroy 时 cleanup |

**两条链路并存合规**(W5-A 允许 driver 直 import capability;两个订阅各自独立 cleanup,不会跨干扰)。
**做这条特例旁路的理由**:vocab-panel 是 video-block 局部 UI,不需要"全 PM 推送"链;走 view integration → driver IPC → vocab-panel 是绕路,且会跟 driver 的 timeline 状态耦合。直订阅最自然。
**约束**:driver 层订阅严格限于 video-block(且未来其他类似"内部需要 vocab 实时刷新的 NodeView"也允许同模式),不得在 view 层 / 顶层模块级出现 — 否则跟 B3.20b 链路会重复推送语义。

## 6. 风险 / 偏差

### 6.1 attrs 体积膨胀
subtitleCues + translationTexts(多语言)持久化到 PM attrs,可能让单笔记 JSON 显著变大(几 MB transcript 实测)。
**应对**:translationTexts 用 base64 gzip 压缩(可选,B3.19.b 决策)。本期 v0.1 不做,留监测。

### 6.2 cross-origin postMessage 兼容
YouTube 偶尔改 IFrame API。V1 用 raw `event:listening` + `infoDelivery`,V2 沿用相同。如果某天失效,可改用 `youtube` npm 包(无破坏性切换)。

### 6.3 Memory Mode 跟 YouTube 自动播放策略冲突
Chrome 86+ 对 iframe autoplay 严格,Memory Mode 调 `playVideo` 可能被静音 / 阻拦。V1 已踩过,需要用户先手点过 iframe 一次,之后允许自动播放。**接受现状,UI 提示**。

### 6.4 翻译 Tab 多语言无上限
用户反复点 🌐 不同语言会一直叠 Tab。V1 没限制。**沿用 V1 — 无限叠**。

### 6.5 placeholder 与 Data Tab 合一(Q-e-2)
当前 V2 placeholder 是 `playerWrap` 覆盖式 UI;改成 Data Tab 内部能力后,无 src 时默认切到 Data Tab。可能造成新建视频时第一感觉是空白(playerWrap 空),需 hint 提示"切到 Meta Tab 填 URL"。**v0.1 接受,如体验差段 e 设计时再调**。
