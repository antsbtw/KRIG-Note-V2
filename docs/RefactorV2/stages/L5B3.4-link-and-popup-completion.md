## 4. 阶段中遇到 / 解决的问题

### 4.1 V2 故意暂缺 rightActiveNoteId(架构空缺)

**现象**:V1 link 行为是"krig://note 开右栏",但 V2 的 noteWsState 只有一个
activeNoteId,没有 rightActiveNoteId。
**根因**:V2 故意暂缺(workspace/README.md 注:V1 教训散落字段,V2 等
ActiveResourceManager 抽象层)。
**降级**:本阶段 link 路由 → 切**当前 ws 左栏 activeNoteId**(覆盖当前编辑笔记)。
**记录**:链接跳转用户体验跟 V1 不完全一致;真右栏 routing 留 ActiveResourceManager
抽象到位后补。已记 link-click-integration.ts 注释 + 完成判据 #10 备注。

### 4.2 driver 不能 import slot/popup-controller(铁律 — 依赖反转)

**问题**:Cmd+K 想触发 popup,但 driver keymap plugin 不该知道 slot 层有 popup。
**修法**:Cmd+K 不走 driver keymap,改在 view 层 NoteView 组件挂 useEffect window
keydown(参考 use-slash-trigger 全局键盘模式)。
**收益**:driver/slot 解耦保持完整;view 自己处理"什么键弹什么 popup"。

### 4.3 重复声明 Window.electronAPI

**现象**:在 build-link-click-plugin.ts 加 `declare global { interface Window {...} }`
覆盖了 src/shared/ipc/electron-api.d.ts 已有的声明,导致其他 alive 文件 type 错。
**根因**:TS 全局声明合并需 modifiers 一致,我用了 `?` 可选,跟原声明的非可选冲突。
**修法**:删 plugin 内的重复声明,统一在 electron-api.d.ts 加 openExternal / openPath。
**记录**:**electronAPI 类型只在 electron-api.d.ts 一处声明**,其他文件不要重复。

### 4.4 popup 跟 floating-toolbar 共存时点击 anchor 关 popup

**预期**:点 anchor 自己 → toggle(关 popup);非 anchor 区域 → 关 popup
**实现**:PopupBinding 内 mousedown handler 检查点击位置是否在 anchorRect 内,在 → 不主动关
(让 anchor 自己 toggle)。

### 4.5 selection 焦点丢失问题(LinkPanel 输入框获取焦点时 PM selection 清空)

**预期**:LinkPanel apply 时 PM selection 仍保留,setLink 才能加 mark。
**实现**:popup 渲染时 onMouseDown stopPropagation(避免冒泡到 PM 触发 blur);apply 后
view.focus() 恢复编辑器焦点。
**对齐 V1 同款防御:** V1 用 `onMouseDown={(e) => e.preventDefault()}`,V2 用 stopPropagation。

## 6. 遗留问题 / 未来阶段

### 6.1 跨 ws 路由 + 真右栏 routing(等 ActiveResourceManager)

V2 当前 noteWsState 只有 activeNoteId 一个字段,无 rightActiveNoteId。link 跳转目前
切左栏覆盖。等 ActiveResourceManager 抽象到位后补真右栏。

### 6.2 LinkPanel 文件 Tab(等 viewAPI fileOpenDialog IPC)

V1 三 Tab 之一的"文件 Tab"依赖 viewAPI.fileOpenDialog / mediaPutFile,V2 当前无文件 IPC。
留独立阶段 — 同时也解锁 audio/video/file-block 迁移。

### 6.3 V1 doc 兼容性(callout / toggle-list V1→V2 命名)

V2 命名空间隔离 div.krig-callout / div.krig-toggle-list,V1 doc 粘贴会降级 paragraph。
留 viewAPI 阶段时考虑加双 parseDOM 兼容。

### 6.4 笔记历史栈仅含 link 跳转(NavSide 切不进栈)

V1 用 viewAPI.noteOpenInEditor 作统一入口,NavSide 点击 / link 跳转都走它,
所以历史栈包含所有切换路径。

V2 当前 NavSide 走 `note-view.set-active` 命令 → setActiveNote(不推 back 栈);
只有 link-click-integration 走 navigateToNote(推栈)。

**用户感知**:Cmd+[/Cmd+] 只在 link 跳转链中工作,NavSide 切笔记后按 Cmd+[ 静默无响应。

**修复方向**(留后续):把 `note-view.set-active` 改用 navigateToNote,让 NavSide 切笔记
也进栈(对齐 V1)。本阶段降级处理,不阻塞 link 主功能。

### 6.5 cycle 命令保留作快捷键备份

`note-view.cycle-text-color` / `note-view.cycle-highlight` 命令保留,本阶段未绑快捷键。
后续可 Cmd+Shift+H / Cmd+Shift+L 之类按需绑。
