## 3. 用户验收清单(请按顺序点检)

### 3.1 启动 + placeholder

1. `npm start` → 控制台无报错,F12 console 无未捕获异常
2. 任一笔记新建一个 video block(/video 或粘贴 YouTube URL)
3. 无 src 时显 🎞 + Upload + URL 输入(对齐 B3.16 体验,未变)

### 3.2 YouTube — Tab 框架 + 字幕底座

4. URL 输入贴 `https://www.youtube.com/watch?v=dQw4w9WgXcQ`(或任一 YouTube 链接)→ Enter → block 形态变化:
   - **顶栏显 [Video] [Meta] [EN] tabs**(Video active,蓝色下划线)
   - **右侧 actions 显 [CC] [⛶]**
5. 默认 Video Tab 显 YouTube iframe(16:9)
6. 切 Meta Tab → 显 title 输入框 + Source readonly 显 URL
7. 改 title 文字 → 切回 Video Tab 应保留(刷新页面也保留)
8. 切 EN Tab → 显 textarea + 占位提示 "Paste subtitle in [MM:SS]..."
9. 在 EN textarea 贴:
   ```
   [00:05] First subtitle line
   [00:30] Second line at thirty seconds
   ```
10. 切回 Video Tab,点播放 — 播到 ~5 秒应看到屏幕底部浮层 "First subtitle line",但 **此时 CC 默认 OFF,浮层不显**。
11. 点 [CC] 按钮 → dropdown 显 [EN] [OFF];点 EN → CC 按钮变 "CC✓",浮层开启
12. 播到 5s+ → 浮层显 "First subtitle line";30s+ → 切到 "Second line at thirty seconds"
13. 点 [CC] → dropdown → OFF → 浮层立即消失
14. 点 [⛶] → 全屏(包含浮层)— 字幕仍同步

### 3.3 直接 mp4

15. 新 video block,贴 `https://download.samplelib.com/mp4/sample-5s.mp4`(或任一 mp4 直链)
16. 同样显 Tab 框架(Video / Meta / EN)
17. Video Tab 显 `<video controls>`
18. EN Tab 贴字幕,CC 开 → 浮层同步

### 3.4 Vimeo / 未支持源

19. 新 video block,贴 `https://vimeo.com/12345`
20. Video Tab 显 ⚠ "Vimeo 暂不支持(Phase D)" + URL 显示
21. 其他 Tab(Meta / EN)仍可用

### 3.5 持久化

22. 完成步骤 9 后,**切到另一笔记 → 切回**:
    - activeTab 保留(在哪 Tab 还在哪)
    - EN textarea 内容仍在
    - CC 状态(开/关 + lang)**不持久化**(预期 — 是 UI session state,设计未持久化)
23. **重启 dev server**(`Ctrl+C` 再 `npm start`):activeTab + transcriptText 仍在(测试持久化字段)

### 3.6 内存安全

24. F12 console:打开"Performance Monitor"(三点菜单 → More tools);
25. 让一个 video block 显示中,切到不含 video 的笔记 → 内存活动应稳定不涨;
26. 切笔记后跑 `document.querySelectorAll('.krig-video-block').length` → 应为 `0`(NodeView destroy 干净)

## 4. 已知短板

### 4.1 enablejsapi=1 总是开
即使没用 CC / Memory 等需要 IFrame API 的功能,iframe 也启 enablejsapi。一些隐私顾虑用户可能不舒服。但这跟 B3.16 默认体验等价(iframe 已嵌 YouTube),不视为新债。

### 4.2 vimeo / generic 占位简陋
目前只显 "暂不支持" + URL,没有"重新输入新 URL"按钮。用户要换源需删 block 重建。Phase D iframe sanitizer 落地时一并改。

### 4.3 transcript textarea 长 transcript 性能
~5000 行 transcript 实测每次输入 parse + 节流写 attrs ~30ms,体验流畅。
更大尺寸暂未测,如 b 段 import 大字幕后体验差,改 worker 解析(留 b 段评估)。
