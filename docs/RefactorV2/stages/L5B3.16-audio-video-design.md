## 1. 三个简化决策

### 1.1 字幕系统坚决留 Phase D

V1 video-block 含完整字幕系统(WebVTT cue 解析 / CC 浮层渲染 / YouTube transcript 自动抓取 /
跨语言翻译)。**全部留 Phase D**(等 thought / AI 集成成型一起做),本阶段:
- video schema 不留 cues / transcripts attrs(等 D 阶段加,无破坏性 schema 升级 — 只加 attrs default null)
- NodeView 不渲染 CC

### 1.2 砍 V1 createRenderBlockView 共享基类

V1 audio/video 都用 `createRenderBlockView(renderer, atomType)` + `RenderBlockRenderer` 接口。
V2 image / fileBlock / externalRef 已经各自独立 NodeView(对齐 L5-B3.5 模式),
**audio/video 同模式独立写**(不抽 V2 RenderBlockRenderer 共享接口)。

### 1.3 generic embed 暂缓

V1 video 支持 4 种 embed:youtube / vimeo / direct(.mp4 等)/ generic(其他 iframe)。

V2 本阶段:
- **YouTube embed**(iframe)— 必做,常见
- **direct video**(`<video>` + media:// 或 https:// 直链)— 必做
- **Vimeo / generic iframe** — 砍掉留 Phase D(用户场景少 + iframe 安全风险需 sanitizer 设计)

## 3. audioBlock NodeView

两态(对齐 image / fileBlock):

**placeholder**(无 src):🎵 + Choose file + URL embed
**player**(有 src):title + `<audio controls preload="metadata">` + 下载按钮(http(s) 源)+ caption

```ts
export const audioBlockNodeView: NodeViewConstructor = (initialNode, view, getPos) => {
  const dom = document.createElement('div');
  dom.classList.add('krig-audio-block');
  dom.setAttribute('contenteditable', 'false');

  // contentDOM(caption)— PM 文本编辑由 PM 接管
  const contentDOM = document.createElement('div');
  contentDOM.classList.add('krig-audio-block__caption');

  let audioEl: HTMLAudioElement | null = null;

  function paint(n: PMNode): void {
    // 清掉旧的 player(保留 contentDOM)
    Array.from(dom.children).forEach((c) => {
      if (c !== contentDOM) c.remove();
    });
    if (audioEl) { audioEl.pause(); audioEl.src = ''; audioEl = null; }

    if (n.attrs.src) renderPlayer(n);
    else renderPlaceholder();

    dom.appendChild(contentDOM);
  }

  function renderPlayer(n: PMNode): void { /* title + audio + download */ }
  function renderPlaceholder(): void { /* file picker + URL embed,upload 走 mediaPutBase64 */ }

  paint(initialNode);

  return {
    dom,
    contentDOM,
    update(updated) {
      if (updated.type.name !== 'audioBlock') return false;
      const hadSrc = !!node.attrs.src;
      const hasSrc = !!updated.attrs.src;
      node = updated;
      if (hadSrc !== hasSrc) paint(node);
      else if (hasSrc) refreshPlayer(node); // 仅 title / src 更新
      return true;
    },
    destroy() {
      if (audioEl) { audioEl.pause(); audioEl.src = ''; }
    },
  };
};
```

### 3.1 上传路径(同 image / fileBlock)

```ts
// placeholder Choose file 按钮:
input.addEventListener('change', async () => {
  const file = input.files?.[0];
  if (!file) return;
  const reader = new FileReader();
  reader.onload = async () => {
    const dataUrl = reader.result as string;
    const r = await mediaPutBase64(dataUrl, file.type, file.name);
    if (r.success && r.mediaUrl) {
      updateAttrs({
        src: r.mediaUrl,
        title: file.name.replace(/\.[^.]+$/, ''),
        mimeType: file.type || null,
      });
    }
  };
  reader.readAsDataURL(file);
});
```

if (src.startsWith('http://') || src.startsWith('https://')) {

type EmbedType = 'youtube' | 'direct';

function detectEmbedType(url: string): EmbedType {
  if (/(?:youtube\.com\/(?:watch|embed|shorts)|youtu\.be\/)/i.test(url)) return 'youtube';
  return 'direct';
}

function extractYouTubeId(url: string): string | null {
  const m = url.match(/(?:youtube\.com\/(?:watch\?v=|embed\/|shorts\/)|youtu\.be\/)([\w-]{11})/);
  return m?.[1] ?? null;
}

function toYouTubeEmbedUrl(id: string): string {
  return `https://www.youtube.com/embed/${id}?rel=0`;
}

## 7. 决策点

### Q1:audio/video schema 加 thumbnail / poster attr

V1 没专门 thumbnail 字段。V2 是否预留 `poster`(video 缩略图)?

- **A. 不预留**:V2 直迁 V1 行为,简单
- **B. 预留 poster default null**:未来加 thumbnail 提取无破坏性

**推荐 A** — V2 不预留,等真需要再加(对齐"不过早抽象")。

### Q2:YouTube embed 是否禁用 enablejsapi

V1 用 `enablejsapi=1` 是为字幕同步(YouTube IFrame API 读播放进度)。
V2 砍字幕,不需要 jsapi。

- **A. 砍 jsapi**:`?rel=0` 即可(rel=0 = 不显示相关视频)
- **B. 留 jsapi**:对齐 V1,但本阶段无消费方

**推荐 A** — 砍掉,Phase D 加字幕时再加回。

### Q3:download 按钮在 audio/video 都加 vs 砍

V1 audio/video 都有 ⬇ 下载按钮(http(s) 源时显示),走 mediaDownload IPC。

- **A. 都保留**(对齐 V1):URL 媒体可以"导入到本地",对应 LinkPanel 文件 Tab 的 import 模式
- **B. 砍**:留 Phase D

**推荐 A** — V2 mediaDownload IPC 已有(L5-B4.3.1),直接调,无成本。

### Q4:audio/video block 内嵌 caption 走 PM contentDOM 还是独立 attr

image/V1 audio/video 都用 PM contentDOM(content='block' 单段 caption,可富文本编辑)。

- **A. 同 image / V1**:contentDOM
- **B. 改 attr 字符串**:简单但用户不能富文本

**推荐 A** — 对齐 image 模式,UX 一致。

### Q5:placeholder 上传时 caret 留位置

audio/video 上传成功后:
- **A. caret 留在节点上**(NodeSelection):同 image,用户按 Enter 续写
- **B. 自动跳到 caption 内**:用户立即写说明

V2 image 走 B(caret 进 caption)— 看 api.ts.insertImageAtSelection。**audio/video 也走 B**
(对齐 image,UX 一致)。

## 10. 风险

### 10.1 YouTube iframe CSP

V2 主窗口 CSP 是否允许 youtube.com iframe?V1 在 Electron 下默认放行,但 V2 若加严
CSP 可能 block。**预案**:实施时验证;若被 block,改 main 进程 webRequest 加 youtube.com
到 frame-src 白名单(对齐 web-translate 加 widget.io 经验)。

### 10.2 audio/video 内存泄漏

NodeView destroy 必须调 `audioEl.pause() + src=''` 释放(V1 写得清楚)。否则切笔记后
仍能听到声音(常见 bug)。**预案**:严格在 NodeView destroy 中清理。

### 10.3 大文件 FileReader 阻塞

50MB+ 视频上传卡。**预案**:同 paste-media / fileBlock,显 importing 状态;不优化 — 留 Phase E。

### 10.4 上传完 setNodeAttribute pos 漂移

异步路径与 paste-media 同(用户编辑期间 pos 失效)。
**预案**:不需要异步路径 — audio/video 上传是用户主动点 placeholder Choose file,
同步 placeholder 内,上传期间用户没法编辑外部(focus 在 input);完成时直接
updateAttrs(getPos 当时是有效的)。
