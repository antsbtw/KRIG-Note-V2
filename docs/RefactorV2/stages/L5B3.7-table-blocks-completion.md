### 6.1 cell 内 slash menu 上下文感知(已知限制,留 L5-B3.7.2)

**现状**:`tableCell.content = 'block+'`(对齐 V1),schema 上允许 cell 内插任意 block,
包括 table(嵌套 table)/ image / mathBlock / horizontalRule 等。

**问题**:
- 嵌套 table:schema 允许但用户体验糟(markdown 也不支持)
- image / mathBlock 在 cell 内可能 width 撑爆 / resize handle 异常
- horizontalRule 在 cell 内意义不明

**当前路径**(本阶段不处理):
- 用户主动在 cell 内 slash → `/table` 等 → schema 允许,会插进去
- markdown / AI 提取的 table 只含 text(常规情况),不会触发这些边界

**留 L5-B3.7.2 处理**:
- 扩展 V2 `SlashItem` 加 `visibleWhen?: (ctx) => boolean` hook
- cell 内时隐藏不合适的 slash 项(table / horizontalRule / 等)
- 同时考虑 cell 内 image / math 的尺寸约束 CSS 兜底

工作量预估:~50 行 SlashItem 扩展 + 各 slash 项 visibleWhen 配置。
