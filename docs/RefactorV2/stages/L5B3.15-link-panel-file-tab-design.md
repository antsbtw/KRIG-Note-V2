## 2. 实现

### 2.1 新增 FileTab 组件

```tsx
// src/views/note/link-panel/FileTab.tsx(新)
function FileTab({ onApply, onClose }: { onApply: (href: string) => void; onClose: () => void }) {
  const [pickedFile, setPickedFile] = useState<File | null>(null);
  const [mode, setMode] = useState<'link' | 'import'>('import');
  const [importing, setImporting] = useState(false);

  const pickFile = () => {
    const input = document.createElement('input');
    input.type = 'file';
    input.addEventListener('change', () => {
      const f = input.files?.[0];
      if (f) setPickedFile(f);
    });
    input.click();
  };

  const confirm = async () => {
    if (!pickedFile) return;
    if (mode === 'link') {
      // file:// 链接 — 不复制,需要 getFilePath 拿绝对路径
      const path = window.electronAPI?.getFilePath?.(pickedFile);
      if (!path) {
        // Blob URL / 从浏览器拖入等无 disk path 来源 → 失败提示
        alert('无法解析文件路径(可能是 Blob 来源)。请用"导入到媒体库"模式。');
        return;
      }
      const href = `file://${path.split('/').map(s => s ? encodeURIComponent(s) : '').join('/')}`;
      onApply(href);
    } else {
      // 导入 mediaStore — FileReader → mediaPutBase64
      setImporting(true);
      try {
        const dataUrl = await readFileAsDataUrl(pickedFile);
        const r = await mediaPutBase64(dataUrl, pickedFile.type, pickedFile.name);
        if (r.success && r.mediaUrl) {
          onApply(r.mediaUrl);
        } else {
          alert(`导入失败:${r.error || '未知错误'}`);
        }
      } finally {
        setImporting(false);
      }
    }
  };

  // ... UI:pick / 双模式 radio / 确认按钮
}

function readFileAsDataUrl(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(reader.result as string);
    reader.onerror = reject;
    reader.readAsDataURL(file);
  });
}
```

### 2.2 主 LinkPanel 加 'file' tab

```tsx
type LinkTab = 'note' | 'file' | 'web';

// initialTab 加 file 协议判断
const initialTab: LinkTab = (() => {
  if (!currentHref) return 'note';
  if (currentHref.startsWith('http')) return 'web';
  if (currentHref.startsWith('file://') || currentHref.startsWith('media://')) return 'file';
  return 'note';
})();

// Tab 渲染加 'file'
{(['note', 'file', 'web'] as const).map((key) => (

))}

{tab === 'file' && <FileTab onApply={handleApply} onClose={onClose} />}
```

### 2.3 link-click plugin 协议路由

V2 link-click 已经有(L5-B3.4):
- `file://` → `shell.openPath`(已有,B3.14 加 mediaResolvePath 时也用)
- `media://` → ❌ **当前没路由**!需补:`media://` → mediaResolvePath → openPath

LinkPanel "文件" Tab 输出 `media://...` href 后,用户点链接时 link-click 必须能打开。

**改 build-link-click-plugin.ts**:加 `media://` 协议路由(对齐 B3.14 fileBlock 打开逻辑):

```ts
// build-link-click-plugin.ts handleClick 补
if (href.startsWith('media://')) {
  // 跟 fileBlock 打开同路径:resolveMediaPath → openPath
  void mediaResolvePath(href).then((path) => {
    if (path) window.electronAPI?.openPath?.(path);
  });
  return true;
}
```

注意 driver 不能直接 import storage,但 mediaStore 在 storage 层,driver → storage
是允许的(B3.14 fileBlock NodeView 已经这么做)。

## 3. 决策点

### Q1:link 模式失败用 alert 还是 inline 提示

V1 ignore 失败(只是不 apply);B3.14 externalRef 改用红字 inline 提示(决策 Q4)。

- **A. inline 红字提示**(对齐 B3.14):UX 友好,不打断流程
- **B. alert**(对齐 V1 简单):弹原生对话框,简单粗暴

**推荐 A** — 对齐 B3.14 模式;失败时 FileTab 内显红字 + 引导用户改用 import 模式。

### Q2:import 模式失败处理

mediaPutBase64 失败(IPC 不可用 / 超大小限制 / 等)。

- **A. inline 红字提示**:同 Q1
- **B. alert**

**推荐 A**。

### Q3:default mode 是 link 还是 import

V1 default `'import'`(用户更可能想自包含)。

- **A. 同 V1 = import**:推荐路径(自包含)
- **B. link**:简单(不复制),但断链风险

**推荐 A**(同 V1)。

### Q4:link-click plugin 加 media:// 协议路由 vs 不加

LinkPanel 输出 media:// href 后,用户点链接需要能打开。

- **A. 加 media:// 路由**(本阶段做):完整闭环
- **B. 不加,留 issue**:用户能创建链接但点了打不开

**推荐 A**。强制做,否则功能不完整。

### Q5:file:// 路径编码 Windows 兼容

V1 + B3.14 都用 POSIX `/` 分隔符,Windows `C:\...` 不支持。

- **A. 同前**:macOS-only,Windows 留 issue
- **B. 加 path.sep 兼容**:复杂(renderer 没 Node path)

**推荐 A** — 跟 B3.14 决策一致。

## 6. 风险

### 6.1 link 模式 Blob 来源失败

`<input type="file">` 通常给 disk 文件,但某些场景(extension 注入 / 拖入 web 图片)
可能拿不到 disk path。getFilePath 返回空串 → 红字提示用户改 import 模式。

### 6.2 import 模式大文件阻塞

FileReader.readAsDataURL 对 50MB+ 文件可能卡 1-3 秒。本阶段不优化,跟 B3.14
fileBlock upload 行为一致(用户能感知 importing 状态)。

### 6.3 link mark 显示

V2 link mark 默认是文字加下划线,**没有 file/media 图标提示**。用户看到 link mark
不知道是什么协议。Phase B+ 可考虑给不同协议的 link 加协议图标(本阶段不做)。
