**关键策略转变(v0.2 — 用户拒绝 v0.1 降级方案)**:
- v0.1 想"降级为 text-block + linkMark"的方案被用户否决 — 永久损失语义
- v0.2:md-to-pm 输出**目标 V2 schema 节点名**(image / mathBlock / table 等驼峰命名),
  schema 暂未实现的节点用 `{ type: 'unknown', attrs: { originalType, missing, raw, error? } }`
  占位
- 加 `unknown` block spec 到 NoteEditor schema(本阶段顺手做),让 unknown 节点能渲染成
  "暂未支持: <originalType>" 占位卡片(橙黄虚线,hover 显示原始内容)
- L5-B4.3.3 闭环测试时 unknown 节点暴露 schema 缺口,反向驱动后续补齐

| Markdown 元素 | V2 输出 | schema 状态 |
|---|---|---|
| heading 1-3 | `text-block` attrs.level | ✅ 已实现 |
| paragraph | `text-block`(level=null) | ✅ |
| code block | `codeBlock` | ✅ |
| blockquote | `blockquote` | ✅ |
| bullet list | `bulletList > listItem > text-block` | ✅ |
| ordered list | `orderedList > listItem > text-block` | ✅ |
| task list | `taskList > taskItem > text-block` | ✅ |
| horizontal rule | `horizontalRule` | ✅ |
| math block (`$$...$$`) | `mathBlock` | ❌ → unknown |
| inline math (`$...$`) | `mathInline` | ❌ → unknown |
| block image (`![]()`) | `image` attrs.{src, alt} | ❌ → unknown |
| `!attach[]()` | `fileBlock` | ❌ → unknown |
| `!file[]()` | `externalRef` | ❌ → unknown |
| table | `table > tableRow > tableHeader/tableCell > text-block` | ❌ → unknown |

**md-to-pm.ts 内有 `PM_NODE_REGISTRY` 常量明示 ✅/❌ 状态**,schema 补齐时改 ❌ → ✅
即可,md-to-pm 不需要改(它输出目标节点名,schema 一旦注册就直接生效)。

**节点缺口清单(L5-B4.x 后续 schema 补齐工作)**:
1. `image` block(块级图片,接 mediaUrl)
2. `mathBlock` / `mathInline`(LaTeX 公式,可能用 KaTeX / MathJax 渲染)
3. `table` / `tableRow` / `tableHeader` / `tableCell`(表格)
4. `fileBlock`(附件,base64 → mediaStore)
5. `externalRef`(外部文件引用,绝对路径)

按 L5-B4.3.3 真实使用频次决定补齐顺序(image / mathBlock 大概率优先)。

## 4. V1 → V2 关键架构红利

| V1 路径 | V2 直接 | 收益 |
|---|---|---|
| putBase64 写 SurrealDB 索引(non-fatal 但增加复杂度) | 不写 index,纯 SHA256+文件存在性去重 | 简化 ~30 行,且 DB 不可用时行为一致 |
| md-to-pm 中 mediaSurrealStore.putBase64 直接调函数(main 进程) | renderer 走 IPC 调封装 | renderer ↔ main 边界清晰;markdownToProseMirror 现在是 renderer 异步 |
| 协议在 SurrealDB ready 后注册(V1) | 协议在 app.whenReady 第一时间注册 | 早于 webview 创建,无 ERR_FILE_NOT_FOUND 时序问题 |
