## 0. 背景

### 0.1 V1 help-panel 关键事实(已调研)

V1 在 `src/plugins/note/help-panel/`,其形态、契约、API 已完整摸清:

**形态**(关键差异点 — 不是 popup):
- `position: fixed; right: 0; top: var(--help-panel-top); width: 360px`
- 右栏定宽长侧栏(从顶到底),不跟选区/按钮锚点对齐
- 这是 V1 LaTeX / Mermaid / Math-Visual / Bookmarks / Dictionary 共用的容器形态

**互斥**:同时只一个面板可见;显式打开新面板自动关旧。

**两类面板**(都参与同一互斥池):
| 类型 | API | DOM 归属 | V1 用例 |
|---|---|---|---|
| Shell-managed | `createHelpPanel(config)` → `{el, headerEl, bodyEl, show, hide, destroy}` | core 建 + cache | LaTeX / Mermaid / Math-Visual / Bookmarks |
| External | `registerExternalPanel(id, hideFn)` + `notifyExternalShow/Hide` | 面板自建 | Dictionary(自带 search input + tab 切换 + 复杂 UI) |

**全局监听**(装一次):
- Esc → 关 active panel
- mousedown 点面板外 → 关(支持 `excludeFromClickOutside` 选择器白名单 — math-block 编辑器/inline 编辑器不算外)
- × 按钮 → 关

**V1 总规模** = `help-panel-core 226 + types 36 + index 18 + css 428 = 708 行`

### 0.2 V2 现状

L4 已有 6 个 interaction-registry([src/slot/interaction-registries/](src/slot/interaction-registries/)):
context-menu / slash / handle / floating-toolbar / overlay / popup。**没有 help-panel**。

L4 完成报告 [L4-slot-registry-completion.md:9-22](docs/RefactorV2/stages/L4-slot-registry-completion.md) 中健康检查行 `[L4] alive | ... overlay: N, ...` 列出已就位 registry,补 help-panel 后从 6 → 7。

**V2 frame mount 真实架构**(关键!不是 platform/renderer 顶层):
- 每个 `WorkspaceInstance` 渲染自己的 `OverlayFrames`(见 [src/workspace/workspace-instance/overlay-frames/index.tsx](src/workspace/workspace-instance/overlay-frames/index.tsx))
- 每 frame 对应一个 binding(`PopupFrame → PopupBinding`)
- 这意味着 help-panel 也要走"每 workspace 实例一份"的模式,而不是"全局单例 host"

**popup vs help-panel 边界**(语义清晰,两套并存):

| 维度 | popup-registry(已有)| help-panel-registry(本期建)|
|---|---|---|
| 形态 | 锚点 popup(贴选区/按钮)| 右栏侧栏(贴 viewport 右,定宽长条)|
| 大小 | 小卡片(`estimatedSize`)| 360 定宽,顶到底 |
| 数量 | 同时一个 | 同时一个(独立池)|
| 用例 | LinkPanel / ColorPicker / NoteLinkSearch / 当前 dictionary 临时 | 未来 LaTeX / Mermaid / Bookmarks + dictionary(本期迁入)|
| state 单例 | popupController | helpPanelController(NEW)|

**两套独立** — 同时打开 popup + help-panel **允许**(互不干涉)。help-panel 内部互斥,popup 内部互斥,跨池不互斥。

### 0.3 跟 L5-B3.20b 的衔接

B3.20b 已合并 dictionary 走 popup-registry(过渡)。本期完成后:
- `popupRegistry.register('note-view.popup.dictionary')` 删掉

## 2. API 设计

### 2.1 `help-panel-types.ts`

```typescript
import type { ComponentType } from 'react';

/** 子面板 Component 接收的 props */
export interface HelpPanelCloseProps {
  /** 面板自身请求关闭(× 按钮 / 内部确认完成等)*/
  onClose: () => void;
}

export interface HelpPanelItem {
  /** 面板 ID,跨 view 唯一(命名建议:`<view>.help.<name>`)*/
  id: string;

  /** view 作用域过滤(undefined = 全 view 可用)*/
  view?: string;

  /** Header 标题文字 */
  title: string;

  /** Body 内容组件 — view 实现 */
  Component: ComponentType<HelpPanelCloseProps>;

  /**
   * 点这些 CSS selector 匹配的元素时,不算"点外部"(不关闭面板)。
   * 面板自身始终被排除。
   * 对齐 V1:LaTeX 排除 `.math-block-wrapper` `.math-inline-editor`。
   */
  excludeFromClickOutside?: string[];
}
```

### 2.2 `help-panel-registry.ts`

完全 mirror popup-registry([src/slot/interaction-registries/popup-registry/popup-registry.ts](src/slot/interaction-registries/popup-registry/popup-registry.ts)):

```typescript
class HelpPanelRegistry {
  private items: Map<string, HelpPanelItem>;
  private listeners: Set<() => void>;

  register(item: HelpPanelItem): void;
  unregister(id: string): void;
  unregisterByView(viewId: string): void;
  get(id: string): HelpPanelItem | undefined;
  subscribe(listener: () => void): () => void;
  get count(): number;
}
export const helpPanelRegistry = new HelpPanelRegistry();
```

### 2.3 `help-panel-controller.ts`

跟 popupController 同模式,但 state 更简(无 anchorRect):

```typescript
interface HelpPanelState {
  visible: boolean;
  activeId: string | null;
}

class HelpPanelController {
  private state: HelpPanelState;
  private listeners: Set<() => void>;

  /** 显式 show(id) — 互斥(自动关旧)*/
  show(id: string): void;

  /** 显式 hide */
  hide(): void;

  /** 切换 — 同 id 关,不同 id 切到新 */
  toggle(id: string): void;

  getState(): HelpPanelState;
  subscribe(listener: () => void): () => void;
}
export const helpPanelController = new HelpPanelController();
```

注意:**全局 listener(Esc / click-outside)放在 HelpPanelBinding 内部**(对齐 PopupBinding),不放 controller —
原因:listener 需要 React DOM ref(读 popup-el contains target),controller 不持 DOM。

### 2.4 `HelpPanelBinding.tsx`

```tsx
export function HelpPanelBinding() {
  // 订阅 controller + registry version
  // state.visible + state.activeId 控可见
  // listener 1:Esc → controller.hide()
  // listener 2:document mousedown
  //   if (panelEl.contains(e.target)) return;
  //   if (item.excludeFromClickOutside.some(sel => closest(target, sel))) return;
  //   controller.hide();
  
  // render:
  if (!state.visible || !state.activeId) return null;
  const item = registry.get(state.activeId);
  if (!item) return null;
  
  return (
    <div className="krig-help-panel" ref={panelRef}>
      <div className="krig-help-panel__header">
        <span className="krig-help-panel__title">{item.title}</span>
        <button className="krig-help-panel__close-btn"
          onClick={() => controller.hide()}>×</button>
      </div>
      <div className="krig-help-panel__body">
        <item.Component onClose={() => controller.hide()} />
      </div>
    </div>
  );
}
```

## 4. 决策点(全 A 默认)

| Q | 选项 | 默认推荐 |
|---|---|---|
| **Q1**:registry 形态 | A. 完全 mirror popup-registry(扁平 register/get + Map)/ B. 仿 context-menu(group/order/enabledWhen)| **A** — 面板互斥不需要排序/分组,跟 popup 一致最干净 |
| **Q2**:controller 单例还是注入 | A. 模块单例(对齐 popupController)/ B. context-injected | **A** — 跨多 view/driver 的全局互斥用单例语义自然 |
| **Q3**:children 渲染策略 | A. activeId 变 → Component 重 mount(state 走面板模块级)/ B. 持久 mount(visibility 切)| **A** — 对齐 PopupBinding;DictionaryPanel 已用 setPanelInitial 模块级状态 |
| **Q4**:click-outside 实现 | A. document mousedown,排除自身 + entry.excludeFromClickOutside / B. focus-out / C. 全局 capture-phase | **A** — 对齐 V1 + PopupBinding |
| **Q5**:子面板覆盖范围 | A. 本期只迁 dictionary,latex/mermaid/math-visual/bookmarks 等用到时再迁 / B. 一锅端把 V1 四面板全迁 | **A** — V2 还没 latex/mermaid 编辑场景需求,等真要用时跟功能阶段一起迁 |
| **Q6**:dictionary close button | A. 删自渲染 close,用 shell 提供的 × / B. 保留 + shell 也有(双关闭)| **A** — V1 也是 shell 提供 close,子面板不重复 |
| **Q7**:popup vs help-panel 互斥 | A. 两套独立(允许同时打开)/ B. 跨池互斥(打 help-panel 关 popup,反之亦然)| **A** — 语义自然(popup = 锚点小卡,help-panel = 长侧栏,功能独立);用户实操时也极少需要同时弹两种 |
| **Q8**:controller 作用域 | A. 全局单例(对齐 popupController — 全 app 同时只显一个 help-panel,跨 workspace 也互斥)/ B. per-workspace 隔离(每个 WorkspaceInstance 独享一份 controller,不同 workspace 可各自显自己的 help-panel)| **A** — 对齐 V2 已有 popupController([src/slot/triggers/popup-controller.ts:64](src/slot/triggers/popup-controller.ts#L64))设计选择;改 per-workspace 隔离需一并修 popupController + 全部 4 个 controller,不在本期范围 |

> 选 A 的副作用 — 跟 popupController 同形:"打开 help-panel 在 A workspace,切到 B workspace,A 的 panel 视觉上随 A workspace 隐藏,B 不显;切回 A 重新显出"。已有 4 个 controller(popup / contextMenu / slash / handle) 都这样,本期不视为新债。如未来要做 per-workspace 隔离,作为单独 stage(改全部 4 个 controller 一并迁),不在 L4.1 范围。

## 6. 风险 / 偏差

### 6.1 顶部留空(`--krig-help-panel-top`)
V1 应用顶部有 toolbar 占空间。V2 当前没有顶部固定 toolbar,默认值 `0`。
shell.css 用 CSS 变量留位,后续如加 toolbar 由 platform 写入 root style。

### 6.2 Application Menu / 菜单"Help"项
本期不做 Application Menu 集成("Help" 菜单项打开 help-panel 等)— 留 L4.2 或合到具体面板任务里(如 LaTeX 时加 "Help → Formula Reference")。

### 6.3 多 workspace 共享 controller(Q8)
若日后改 per-workspace 隔离,popupController + helpPanelController 一起改 — 不在本期范围。

### 6.4 dictionary CSS 改动可能影响视觉
dictionary-panel.css 删 width/border/box-shadow/header 后,shell 接管这些样式 —
内部 padding/font-size/color 全保留。需用户视觉验收。
