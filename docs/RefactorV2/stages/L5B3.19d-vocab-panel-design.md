## 0. 上下文

### 0.1 V1 vocab panel 行为

- 📖 按钮 toggle:点开 → 显示在 play-tab 右下角的滚动卡片(window=5 个 entry,当前时间附近的生词)
- 数据源:`buildVocabTimeline(transcriptCues, vocabWords)` → 在每条 cue 文本里匹配生词,生成 `{word, definition, time}[]` 时间轴
- 渲染:`getVocabWindow(timeline, currentTime, 5)` 取当前 ±5 个 entry,active 高亮
- 触发刷新:
  - 📖 toggle 打开时一次性 buildTimeline + 渲染首屏
  - currentTime 推进时更新 active 高亮(轮询)
  - **vocab 变化时 rebuildTimeline**(V1 无主动订阅 onVocabChanged — 用户加词时 panel 不会刷,需手动关再开;V2 改进:订阅 onVocabChanged 自动刷新,**这是本段的特例旁路**)
  - **transcript cues 变化时 rebuildTimeline**(用户编辑 transcript 后)

### 0.2 a/b/c 段已就位的依赖

- `time-tracker.onTimeUpdate(cb)`:订阅 currentTime
- `time-tracker.onDurationReady(cb)`:duration 异步推送
- `framework.cues`:transcript 内存派生 cues(P1 真相源派生)
- `framework.translations.<lang>.cues`:翻译 cues(b 段)
- `learning.vocabList()` + `learning.onVocabChanged(cb)`:capability 直 import(B3.20b 已落地)

### 0.3 vocab 数据源选择(transcript vs translation)

V1 默认用 transcript cues 匹配。但本段(B3.19.d)考虑到用户切翻译 Tab 的场景:
- **决策 Qd-X**:vocab panel timeline 用哪份 cues?
  - A. **总是 transcript cues**(英文原文匹配生词 — 对齐 V1)
  - B. 跟随 CC 当前 lang(transcript / 翻译)
  - C. 用户在 vocab panel 内独立切

V1 是 A,本段也走 A — 简化,生词主要是英文学习用。

## 2. 模块边界 + 关键 API

### 2.1 helpers/vocab-timeline.ts

```typescript
import type { SubtitleCue } from './subtitle-parser';

export interface VocabTimeEntry {
  word: string;
  definition: string;
  /** 该词在 cue 里出现的时间(秒)*/
  time: number;
}

/** 在 cues 中扫描 vocab 词,生成时间轴(对齐 V1)*/
export function buildVocabTimeline(
  cues: SubtitleCue[],
  vocabWords: Array<{ word: string; definition: string }>,
): VocabTimeEntry[];

/** 当前时间附近的 ±windowSize 条 entry + active 索引 */
export function getVocabWindow(
  timeline: VocabTimeEntry[],
  currentTime: number,
  windowSize?: number,
): { entries: VocabTimeEntry[]; currentIndex: number };
```

V1 算法直迁(cue.text 全小写 + 词正则匹配)。

### 2.3 components/vocab-panel.ts

```typescript
import type { TimeTracker } from '../helpers/time-tracker';
import type { SubtitleCue } from '../helpers/subtitle-parser';

export interface VocabPanelDeps {
  /** 取当前 transcript cues(由 node-view 提供 closure)*/
  getCues: () => SubtitleCue[];
  /** time-tracker(可能 null,无 src 时)*/
  getTracker: () => TimeTracker | null;
}

export interface VocabPanel {
  /** 浮层 DOM,挂到 play-tab.overlayMount */
  el: HTMLElement;
  /** 显示 + 启动三源订阅(toggle on)*/
  show(): void;
  /** 隐藏 + 解除订阅(toggle off)*/
  hide(): void;
  /** 主动 rebuild timeline(transcript cues 变化时 node-view 调用)*/
  rebuild(): void;
  isVisible(): boolean;
  destroy(): void;
}

export function createVocabPanel(deps: VocabPanelDeps): VocabPanel;
```

**内部状态**:
- `timeline: VocabTimeEntry[]`
- `currentVocabWords: Array<{word, definition}>`(从 capability 同步)
- `timeUnsub / vocabChangedUnsub`(订阅句柄)
- `lastWindowKey`(优化:仅当 window 内容变化时 re-render DOM)

**生命周期**:
- `show()`:取 vocabList → 订阅 onVocabChanged → 订阅 onTimeUpdate → buildTimeline → render
- `hide()`:解除订阅 → 隐藏 DOM
- `rebuild()`:外部触发(transcript cues 变),重 buildTimeline + re-render
- `destroy()`:hide + remove DOM

**渲染策略**:每次 onTimeUpdate(300ms)取 window,只在 window 内容真变(不同 entries / currentIndex)时重 render DOM(避免每 300ms 抖动)。

### 2.4 driver 内部 onVocabChanged 订阅(特例旁路)

总设计 § 1.1 已声明:**video-block vocab-panel 内部直订阅 `learning.onVocabChanged` 是特例旁路**(driver 层 NodeView 局部 UI),跟 B3.20b view-integration 路径(给全 PM decoration 推送)职责分离。

实现位置:`vocab-panel.ts.show()` 内调 `onVocabChanged(cb)` 拿 unsub;`hide()` / `destroy()` 时 unsub。

订阅范围**严格限于 vocab-panel 内**;不在 node-view 顶层 / 模块级订阅(否则跟 B3.20b 重复)。

## 6. 风险

### 6.1 driver 直订阅 onVocabChanged 跟 B3.20b 重复
**职责分离**:B3.20b 给全 PM decoration 推;本段给单 NodeView 局部 panel 推。两条订阅独立 cleanup,不会跨干扰。总设计 § 1.1 已记录约束:**driver 直订阅严格限于 NodeView 内部局部 UI;不得在 view 层 / 顶层模块级出现**。

### 6.2 timeline 大词表性能
buildVocabTimeline 是 O(cues × words),典型 60 分钟视频 ~500 cues × ~100 词 = 50k 次正则,实测 < 100ms。对齐 V1 实测可接受。如出问题加 worker 解析(留监测)。

### 6.3 短语生词的 word boundary 匹配
含空格的 vocab(如 "machine learning")用 `\\b` 边界匹配 — 对短语两端要求 word boundary,中间空格不影响。Qd-7=A 跟 B3.20b vocab-highlight plugin 行为对齐。

### 6.4 onVocabChanged 触发 rebuild 的成本
用户每加一个词,panel 整个 timeline 重建。500 cues × 100 词 < 100ms,体感无延迟。**接受**。
