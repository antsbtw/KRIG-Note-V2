## 一、两个维度

**主维度 = 块的类型**（段落、列表、容器、代码、媒体…语义不同）。

**次维度 = 光标状态**（退格的关键状态）：
- **块首/行首**：光标在块开头退格（最关键——决定「合并/退出/降级」）
- **空块**：块内无内容时退格
- **块中/行中**：删前一个字符（平凡，所有块一致）
- **选中整块**（NodeSelection，尤其原子块）：退格删块

> 下表重点定义「块首退格」「空块退格」；「块中删字符」平凡，不逐块重复。

## 二、逐 block 定义（第一步 · 核心）

> ✅ = 已确认；🔲 = 待确认/定义；当前实现见 §五附录。「期望」列写用户视角，不写技术机制。

### 2.0 全局规则 ✅ 已确认（2026-06-07）

#### 核心总纲：块首/空行退格的优先级链（所有块的统一抽象）

光标在**块首 / 空行**按退格时，按以下优先级**从高到低依次尝试**，**每按一次退格只走一步**
（渐进反馈），命中即停：

1. **去格式化 / 退出容器（脱外壳，能脱就先脱）**
   - 有缩进 `indent>0` → 先减一级缩进（`indent−1`）
   - 是标题 heading → 降级为正文段（保留文字）
   - 是列表项 → 退出列表变正文段
   - 在容器内（引用 / 标注 / toggle / 分栏）→ 退出该层容器变正文段
2. **逐级上提对齐**：若当前块仍嵌在更上层 → 回退到上一级、与上一级对齐，
   **一直退到一级（顶级）block 为止**。
3. **跨块合并**：已脱光成顶级朴素正文段后，才与上一个 block 合并。

> 一句话：**能脱一层就先脱一层，一路退到顶级朴素正文段，最后才跨块合并。**
> 段落 / 标题 / 列表 / 容器都是这条链的特例（见各自小节）。**这是 Backspace 的核心抽象。**

#### 其它全局规则

- **块中/行中退格 = 删前一个字符**（所有块一致，下表不重复）。
- **选中原子块退格 = 删该块**（NodeSelection → 删整块）。

### 2.1 文本类 ✅ 已定义（2026-06-07）

> **重要**：段落**不是**「直接合并」的特例 —— 段落本身也可能有缩进 / 在容器内，所以段落
> 退格**也走 §2.0 核心优先级链**（先脱缩进 → 脱容器 → 顶级了才合并）。下表「块首/空块退格」
> 列均指**走完优先级链、已到顶级朴素正文段那一步**的行为。

| Block | 块首退格 | 空块退格 | 备注 |
|-------|---------|---------|------|
| **段落 paragraph** | 走优先级链（§2.0）：indent>0 先减缩进 → 在容器内先退出容器 → **顶级时才与上一块合并** | 同左：先脱缩进/容器；顶级空段才删段、光标回上一块尾 | 段落非特例，按链走 |
| **标题 heading** | 走链；脱到 heading 这层时 = **先降级为正文段**（保留文字，本次只降级）；再按一次才继续（合并/脱壳） | 同上：先降级正文段，再退才合并 | 渐进：标题不一步消失，先退化正文段给缓冲，对称 Enter 标题降级 |
| **文档标题 isTitle** | ✅ **保护**：不删、不合并（标题必须保证存在） | ✅ 无法再回退（保护，什么都不发生） | title-guard 保证 doc 首块恒 isTitle=true |

> - **标题降级两步走**：第一次退格 heading→paragraph（文字不动），下次退格才继续脱壳/合并。
> - **容器内段落退出容器**：首块行首退格 → 把该段从容器顶部**提出到容器上方**（变顶级段落）；
>   容器内若还有其他块则**保留容器**；**仅当容器只剩这一块（提出后空了）才解散整个容器**。
> - **空段退格同样走优先级链**：空段若有缩进/在容器内，也先脱缩进/退容器，而不是直接删段跳上块。

### 2.2 列表类 ✅ 已定义（2026-06-07）

| Block | 项首退格（indent=0 时） | 空项退格 | 备注 |
|-------|----------------------|---------|------|
| **无序 / 有序 / 任务 列表项** | ✅ **当前行退出列表，变正文段**（只退出，不与上一项合并） | ✅ 严格按 §2.0 优先级链：① 退出列表变正文段 → ② 若还嵌在更上层（容器内等）则**上提对齐上一级 block（空行还在，不合并）** → ③ 一路上提到顶级，**到顶级才考虑合并** | indent>0 先减缩进（2.0） |

> **统一模式（「先解构、后合并」—— 为抽象埋线）**：Backspace 在块首/空块时，先把当前块**降一档结构**
> （脱掉标题/列表/容器外壳 → 变朴素正文段），**已是朴素正文段才执行跨块合并**。三者对照：
> - 标题：退格 → 先降级正文段 → 再退才合并
> - 列表项：退格 → 先退出列表变正文段 → 再退才合并（除非已顶级）
> - 段落：本身已是朴素正文段 → 退格直接合并
>
> 即：**结构外壳一次脱一层，脱光成一级正文段后才跨块合并。** 这是 Backspace 的核心抽象。

### 2.3 容器类 ✅ 已定义（2026-06-07）

> **核心模型（用户拍板）**：**容器内部 = 一个独立的「小 note」**。前面定义的所有 Backspace 规则
> （§2.0 优先级链：合并 / 标题降级 / 列表退出 / 减缩进）在容器内部**原样适用**，就跟在顶级 note 里一样。
> **容器边界 = 这个小 note 的「顶级」**；容器的直接子块 = 小 note 的一级 block。
>
> 关键：**「退出容器」不是首块退格的直接动作**，而是「容器内小 note 走到尽头（退到容器内顶级、indent=0、
> 再无可退）」时，**那一次退格**才把当前块提出容器、回到文档级。容器内非首块的退格 = 容器内部的合并/降级
> （跟普通 note 一样），**不越过容器边界**。

| Block | 容器内退格（按容器内小 note，§2.0 链原样适用） | 退出容器的那一退 | 备注 |
|-------|------------------------------------------|----------------|------|
| **引用块 blockquote** | ✅ 容器内正常走：非首块行首退格=与容器内上一块合并/降级；有缩进先减缩进 | ✅ 容器内首块、已 indent=0、无可再退 → **这一退把该块提到容器上方**（逐块退，容器保留其余块） | 修「退不出容器」缺陷 |
| **标注块 callout** | ✅ 同上 | ✅ 同上 | |
| **折叠块 toggle（展开态）** | ✅ 同上 | ✅ 同上 | 收起态光标只在标题行，无「容器内首块」状态 |
| **分栏 column** | ✅ 同上 | ✅ 同上 | column 空列已有专属处理，与此一致 |

> - **「退出容器」仅发生在容器内首块、且容器内已无更内层可退时**（小 note 退到顶级）。容器内其余块的
>   退格都是容器内部操作（合并/降级/减缩进），**不退出容器**。
> - **逐块退 + 解散**：退出容器是逐块的（只提当前块）；**容器只剩这一块时**（提出后空了）→ 解散整个容器。
> - **对称 Enter**：Enter「空段回车跳出容器」 ↔ Backspace「容器内退到顶级再退=退出容器」，进出对称。
> - **实现缺陷（待修）**：当前容器的 `isolating` 阻挡 joinBackward，使「容器内退格」连内部合并都退化成
>   「只删字符」、更退不出容器。本定义要求：容器内 Backspace 走完整 §2.0 链（内部正常合并/降级），
>   到顶级再退才 unwrap 退出（需专属 keymap / 自定义命令）。

### 2.5 媒体 / 嵌入 / 原子类 ✅ 已定义（2026-06-07）

> **防误删原则（用户拍板，对齐 Notion）**：媒体块是有分量的内容，**无意的退格永不删块**。
> 删块只走两条**有意**入口：**整块选中（NodeSelection）按退格** / **handle 菜单删除**。

| Block | caption 内退格 | 选中块退格 | 删块入口 |
|-------|--------------|----------|---------|
| **image / html / math-visual（有 caption）** | ✅ 只删 caption 字符；**caption 空后再退格 → 光标移出到块上方**（无内容可删则无反应），**绝不删块** | ✅ 删块（用户明确选中=有意） | 选中块退格 / handle |
| **audio / video / tweet（有 caption）** | ✅ 同上（caption 退格不删块） | ✅ 删块 | 选中块退格 / handle |
| **文件 file-block / 外链 external-ref / 分割线 hr（atom）** | （无 caption，光标进不去） | ✅ 选中→删块 | 选中块退格 / handle |
| **公式块 math-block（LaTeX 代码区）** | （内部删字符走代码区，同 §2.4 代码块） | ✅ 空块退格 → 变普通段落（同代码块空块） | 内部编辑 / 选中 / handle |

> - **caption 退格不删块**：caption 内退格 = 普通删字符；caption 删空后继续退格 = 光标移到块**上方**
>   （而非删块）。要删媒体块必须**选中整块退格**或用 **handle 菜单**。修「caption 空后误删整块」缺陷。
> - **选中块退格删块**保留（NodeSelection，用户有意），与 Notion 一致。
> - math-block：内部是 LaTeX 代码区（同 §2.4），空块退格变段落；非空内部退格删源码字符。

### 2.6 表格 ✅ 已定义（2026-06-07）

> **核心模型（用户拍板，对齐 Notion）**：**cell = 一个小 note**，内部各块按 §2.0 优先级链**原样适用**
> （合并 / 标题降级 / 列表退出 / 减缩进，跟顶级 note 一样）。
> **但 cell 与容器的关键区别：cell 不允许退出**。cell 是表格网格的固定结构，提不出去（提出表格就破了），
> 故 cell 边界是**硬墙**：退到 cell 内顶级（indent=0、无可再退）再退格 = **无反应**（不退出 cell、不跨
> cell 合并、不删表格）。即 §2.0 链照走，但**最后一步「退出/合并」在 cell 边界被禁用**。

| 场景 | 退格 | 备注 |
|------|------|------|
| **cell 内非顶级块** | ✅ 按 §2.0 链：cell 内合并 / 降级 / 减缩进（cell 内小 note 正常操作） | 不跨 cell |
| **cell 内已到顶级再退格** | ✅ **无反应**（cell 是硬墙，不退出、不跨 cell 合并、不删表格） | 对齐 Notion |

> 删表格不靠 cell 内退格：靠选中表格 / 表格 handle 菜单（同媒体块「有意才删」原则）。
> cell 与容器对照：容器退到顶级再退 = unwrap 退出；cell 退到顶级再退 = 无反应（硬墙不可穿越）。

## 五、附录：Backspace 当前实现现状（技术，供排查/对照）

Backspace 几乎全靠 PM baseKeymap 兜底，**只有 3 处主动拦截**：

| Block / 场景 | 文件:行 | 机制 |
|-------------|---------|------|
| 代码块空块 | plugins/build-code-block-keymap.ts:94-103 | handleKeyDown：空 codeBlock → 替换为 paragraph |
| column 空列 | blocks/column-list/keymap.ts:85-106 | Command：删空列 / 列表只剩 1 列则解散 |
| math-visual 画布 | blocks/math-visual/node-view.ts:162-164 | nodeView keydown 吃掉所有键，防 atom 误删 |
| 兜底（其余全部） | prosemirror-commands baseKeymap | joinBackward（行首合并）/ selectNodeBackward（删 atom）/ deleteSelection（删选区） |
| 装载顺序 | editor-view-builder.ts | blockPlugins(column/table/toggle)→listKeymap→codeBlock→…→baseKeymap |

> **关键**：容器（column/table/toggle）+ 媒体（image/html/math-visual）的 `isolating` 属性会阻挡
> joinBackward 跨块合并，使「容器/caption 首块行首退格」退化成「只删字符」——这是「退不出容器」的根因。
> 全项目**无** Delete 键、无 Mod-Backspace 的自定义处理。
