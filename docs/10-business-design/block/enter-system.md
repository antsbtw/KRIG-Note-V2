## 一、两个维度

定义每个 block 的 Enter 行为，看两个维度：

**主维度 = 块的类型**（最重要）—— 段落、列表、容器、代码、媒体…各类语义不同。

**次维度 = 光标状态**：
- **空块**：块内没有内容（或只有占位）时按回车
- **非空·块中**：块有内容，光标停在文字中间
- **非空·块尾**：光标在块末尾按回车（最常见的「继续写下一段」）

> 部分块只有一种状态有意义（如分割线无文字、媒体块靠 caption）。逐块按实际情况填。

## 二、逐 block 定义（第一步 · 核心）

> ✅ = 已确认行为；🔲 = 待你确认/定义；当前实现见 §五附录。

### 2.1 文本类 ✅ 已定义（2026-06-07）

**「格式」的定义**：段落/标题的排版属性 = `indent`（块缩进）+ `textIndent`（首行缩进）+
`align`（左/中/右对齐）。**不含** `id`（身份）、`isTitle`/`level`（块类型身份）、`bookAnchor`（元数据）。

| Block | 空块 回车 | 非空·块中 回车 | 非空·块尾 回车 | 备注 |
|-------|----------|---------------|---------------|------|
| **段落 paragraph** | ✅ 新建空段落（下一行），继承格式 | ✅ 从光标处拆两段，后半进新段，继承格式 | ✅ 新建下一段（空），继承格式 | 新段永远继承源段全部格式（indent+textIndent+align），三态一致 |
| **标题 heading** | ✅ 降级为正文段（level 丢弃，继承 textIndent/align） | ✅ 拆两段：光标**前**仍标题（同级），光标**后**变正文段 | ✅ 新建**正文段**（非同级标题），继承 textIndent/align | 不延续标题：新产出块永远是正文段（Notion）；块中回车光标前那半是原标题留存 |

### 2.2 列表类 ✅ 已定义（2026-06-07）

**统一规则**：
- **空项回车 = 跳出列表**：该项变普通正文段，**与列表上一级对齐**（继承该项的 indent）。
  对齐容器「空段回车跳出」+ Notion 行为。
  - 边缘：列表项 schema（content='block+'）允许从粘贴/导入带进真嵌套子列表；空项回车
    **统一跳出当前所在那一层列表**变正文段，不做旧模型的「逐级 lift」（缩进已改 indent attr，
    交互层不再产生 sink 嵌套，见 [`indent-system.md`](./indent-system.md)）。
- **非空·项中回车 = 拆成两项**：后半内容进新项，**新项继承格式（indent）**。
- **非空·项尾回车 = 新建下一项**：空项，**继承格式（indent）**。

| Block | 空项 回车 | 非空·项中 回车 | 非空·项尾 回车 | 备注 |
|-------|----------|---------------|---------------|------|
| **无序列表项** | ✅ 跳出列表 → 正文段，继承 indent | ✅ 拆两项，新项继承 indent | ✅ 新建下一项，继承 indent | splitListItem |
| **有序列表项** | ✅ 同上 | ✅ 拆两项，编号顺延 | ✅ 新建下一项，编号 +1 | |
| **任务列表项** | ✅ 同上 | ✅ 拆两项 | ✅ 新建下一项（新项 checked=false） | checked 不继承（新项默认未勾） |

> 一个列表项内可装多个块（listItem content='block+'）。项内某个块的回车走该块的标准行为
> （如项内有段落则段落回车）；上表「拆项/新建项」针对项的主文本段。

### 2.3 容器类（内部可放多个块）✅ 已定义（2026-06-07）

**统一规则**：
- **容器内空段回车 = 退出容器**：删掉该空段，在**容器下方**新建一个普通正文段，光标移过去。
  任意位置的空段都跳出（不区分中间/末尾，规则简单）。代价：容器内无法敲纯空行做视觉间距
  （需要间距用块缩进/段间距，不靠空段堆）。对齐列表「空项回车跳出」+ Notion 行为。
- **容器内非空块（块中/块尾）回车 = 在容器内拆段/新建段，继承格式**（同 §2.1 段落规则）。

| Block | 容器内空段 回车 | 内部块中 回车 | 内部块尾 回车 | 备注 |
|-------|----------------|--------------|--------------|------|
| **引用块 blockquote** | ✅ 退出引用块 → 容器下方新建正文段 | ✅ 引用块内拆段，继承格式 | ✅ 引用块内新建段，继承格式 | — |
| **标注块 callout** | ✅ 退出 callout → 容器下方新建正文段 | ✅ 内拆段，继承格式 | ✅ 内新建段，继承格式 | 同 blockquote |
| **折叠块 toggle（展开态）** | ✅ 退出 toggle → 容器下方新建正文段 | ✅ 内拆段，继承格式 | ✅ 内新建段，继承格式 | 同 blockquote |
| **折叠块 toggle（收起态）** | （此状态不存在：内部已折叠隐藏，光标只在标题行） | — | ✅ 标题行回车 → 在 toggle **后**新建下一个 toggle（继承缩进） | 收起态光标只在标题行 |
| **分栏 column** | ✅ 退出分栏 → 分栏下方新建正文段 | ✅ 栏内拆段，继承格式 | ✅ 栏内新建段，继承格式 | — |

### 2.4 代码块 ✅ 已定义（2026-06-07）

代码块是「整块编辑源码」的特殊块（content='text*' code），回车语义与文本块完全不同：

| 光标状态 | 回车后用户看到 |
|---------|--------------|
| **任意位置（含空块、块中、块尾）单次回车** | 插入换行 `\n`，**不拆块、不跳出**（始终留在代码块内编辑） |
| **跳出条件：光标在末尾 且 最后一字符已是 `\n`（即上次回车留下的空行）** | 删掉那个末尾 `\n`，在代码块**下方**新建正文段，光标过去（双回车跳出） |
| **跳出时若代码块已变空** | 顺带删掉这个空代码块（不留空壳） |

> - 「双回车跳出」精确条件 = 末尾 + 末字符是 `\n`，不是简单「连按两次」。
> - 空代码块回车 = 仍插 `\n`（用户拍板保持现状），**不**直接跳出；要退出空代码块用 Backspace
>   （空块 Backspace → 变回普通段落，见附录）。
> - Tab/Shift-Tab 在代码块内是 2 空格缩进/反缩进（不属回车，归 indent-system.md 例外项）。

### 2.5 媒体 / 嵌入类 ✅ 已定义（2026-06-07）

分两类：**有 caption（content:'block'，单段落）** 和 **atom 无 caption（纯卡片，光标进不去）**。

**caption 约束（用户拍板）**：caption **只能单段落，不能多段落**。所以回车不在 caption 内拆出第二段，
而是**光标后的内容切到块下方新建的正文段**。三种光标状态统一为一句话：

> **caption 保留光标前内容；光标后内容（可能为空）切出来，移到块下方新建的正文段，光标跟过去。**

| caption 光标状态 | 回车后用户看到 |
|----------------|--------------|
| **空 caption 回车** | caption 留空；块下方新建空正文段，光标过去 |
| **非空·段中 回车** | caption 保留光标**前**内容；光标**后**内容切出，放进块下方新建正文段，光标跟到那段 |
| **非空·段末 回车** | 光标后为空 → 块下方新建空正文段，光标过去（同空行回车） |

> 与普通段落一致点：段中回车都把光标后内容分出去。
> 与普通段落区别点：普通段落分到「同级新段」；caption 单段落不能多段，故光标后内容分到**块下方**的
> 正文段（不是留在 caption 内）。

**各块实现状态**：

| Block | caption | caption 回车（按上表统一） | 实现状态 |
|-------|---------|--------------------------|---------|
| **图片 image** | 有（block?） | ✅ 三态统一跳出，下方新建正文段 | ✅ 已有 keymap |
| **HTML 块 html-block** | 有（block） | ✅ 同上 | ✅ 已有 keymap |
| **数学公式块 math-visual** | 有（block） | ✅ 同上 | ✅ 已有 keymap |
| **音频 audio** | 有（block） | ✅ 同上 | ❌ **缺 keymap → 当前回车删块 bug，待修** |
| **视频 video** | 有（block） | ✅ 同上 | ❌ **缺 keymap → 同 bug，待修** |
| **推文 tweet** | 有（block） | ✅ 同上 | ❌ **缺 keymap → 同 bug，待修** |
| **文件 file-block** | 无（atom:true） | 走原子块回车（§2.6） | n/a（无 caption，光标进不去） |
| **外链 external-ref** | 无（atom:true） | 走原子块回车（§2.6） | n/a |

> **核对结论（用户 2026-06-07 反馈）**：audio/video/tweet 在 caption 回车时会删掉整个 block —— 确实是
> bug，与 image/html/math-visual 同一个坑（缺专属 keymap）。修法：补同款 keymap（三态统一跳出、下方新建段）。
> file-block / external-ref 是 atom 卡片无 caption，归 §2.6 原子块。

### 2.6 原子 / 无文字块 ✅ 已定义（2026-06-07）

**统一规则**：原子块（atom，光标进不去）按回车的唯一场景 = **整块被选中（NodeSelection）时** →
在该块**下方新建正文段**，光标过去（同 caption 跳出、容器跳出，全局一致「回车=下方新正文段」）。

| Block | 选中该块 回车 | 备注 |
|-------|--------------|------|
| **分割线 horizontal-rule** | ✅ 下方新建正文段 | atom，光标进不去，只有选中态有回车 |
| **文件 file-block** | ✅ 下方新建正文段 | atom 卡片，同上 |
| **外链 external-ref** | ✅ 下方新建正文段 | atom 卡片，同上 |
| **公式块 math-block** | （非 atom：content='text*' + code，是 LaTeX 源码编辑区） | 回车在其内 = 编辑公式插换行，**走代码区编辑逻辑，不改**（见 §2.4 代码块同理） |

> math-block 不是无文字块：它内部是装 LaTeX 源码的代码区（`content:'text*' code:true`），
> 光标能进去，回车 = 源码换行。归类放这里仅因「外观像原子块」，行为上同 §2.4 代码块（编辑逻辑自洽，不动）。

### 2.7 表格 ✅ 已定义（2026-06-07）

**结论（用户拍板）**：表格单元格 `content='block+'`，cell 内装的就是**标准 block**（paragraph/heading/
list/…）。所以 **cell 内回车 = 光标所在那个标准块的标准回车行为**（段落拆段、列表新建项等，照 §2.1–2.6），
**无需为表格单独定义回车**。

| 场景 | 回车 | 备注 |
|------|------|------|
| **表格单元格内** | ✅ 走 cell 内当前块的标准回车（不独立） | Tab 在表格是单元格导航；回车不导航，走块内标准行为 |

> 对照：Tab 在表格内是「单元格间导航」（特化），但回车不特化——cell 装标准块，回车就是块的标准行为。

### 2.8 特殊（inline / 占位 UI，可能不纳入「块 Enter」范围）

| 项 | 回车 | 是否纳入收敛 |
|----|------|-------------|
| 行内公式 math-inline 编辑弹窗 | 保存公式关弹窗 | ❌ 纯 UI |
| 媒体块 placeholder URL 输入框 | 提交 URL | ❌ 纯 UI |
| 软换行（Shift-Enter，任意块内） | 插 `<br>` 不拆块 | 单独键，非 Enter |
| 文档末空白区（失焦时回车） | 末尾新建空段 | 边界场景 |
