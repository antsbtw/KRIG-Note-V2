# 需求：NoteView 行内链接体系

> 类型：功能增强  
> 创建日期：2026-04-20  
> 状态：已实现  
> 分支：`feature/noteview-enhancement`  
> 前置：Stage 1 ChatGPT 提取验证

---

## 一、背景

当前 NoteView 的文件引用只有**块级**形式：

| 语法 | 渲染结果 | 位置要求 |
|------|---------|---------|
| `!attach[filename](media://...)` | FileBlock（带打开/Finder 按钮） | 必须独占一行 |
| `!file[title](path)` | ExternalRef 卡片 | 必须独占一行 |
| `![alt](media://...)` | ImageBlock | 必须独占一行 |

**缺失的能力**：行内文件链接。当文件引用出现在段落文字中间时（例如 ChatGPT 的 `"请参阅附件 [report.py](sandbox:...)"` ），目前只能降级为纯文本加粗，无法点击打开。

## 二、实际场景

ChatGPT Code Interpreter 经常在段落中间引用沙箱文件：

```markdown
这是一个混合内容段落：其中包含**粗体**、*斜体*、`inline code`，
以及一个链接样式的附件引用：[sample_attachment.py](sandbox:/mnt/data/sample_attachment.py)。
```

提取到 Note 后，独占一行的文件引用可以渲染为 FileBlock，但行内引用只能退化为加粗文本 `**sample_attachment.py**`，丢失了可交互性。

## 三、已实现的行为

### 3.1 两条路径并存

**路径 A — AI 提取自动生成（fileLink inline atom）**

ChatGPT 提取时，行内 sandbox 文件引用输出为 `[filename](media://...)` 格式，解析为 `fileLink` inline atom node，渲染为 📎 chip。

**路径 B — 用户手动插入（link mark + media:// / file://）**

用户通过 FloatingToolbar 🔗 按钮插入链接，`link-click.ts` 根据 href 协议自动分发点击行为。

### 3.2 统一点击行为

| 链接类型 | 单击 | 右键 |
|---------|------|------|
| `krig://note/{id}` | 编辑器内导航 | — |
| `krig://block/{noteId}/{anchor}` | 导航 + 滚动到目标 block（带高亮） | — |
| `https://` / `http://` | 系统浏览器打开 | — |
| `file://` | 系统默认应用打开 | — |
| `media://` | 解析路径后系统默认应用打开 | — |
| `fileLink` inline atom | 系统默认应用打开 | 打开 / 在 Finder 中显示 / 复制路径 |

### 3.3 右键通用格式移除

光标在任意带格式的文字上右键，自动检测所有 mark（粗体、斜体、下划线、删除线、行内代码、链接、文字颜色、高亮、标注）并显示对应的「移除 X」菜单项。无需选中文字。

## 四、技术方案

### 4.1 fileLink inline atom node

`src/plugins/note/blocks/file-link.ts` — inline atom，用于 AI 提取场景：

```typescript
fileLink: {
  group: 'inline',
  inline: true,
  atom: true,
  attrs: {
    src: { default: '' },        // media://files/... 
    filename: { default: '' },   // 显示名
  },
  toDOM(node) {
    return ['span', { class: 'file-link', 'data-src': node.attrs.src }, 
      ['span', { class: 'file-link-icon' }, '📎'],
      ['span', { class: 'file-link-name' }, node.attrs.filename],
    ];
  },
}
```

### 4.2 Markdown 语法

复用标准链接语法，通过 `media://` 协议区分：

```markdown
段落文字 [filename.py](media://files/file-abc123.py) 更多文字。
```

解析规则：当 `[text](href)` 的 href 以 `media://` 开头时，渲染为 `fileLink` inline node 而非普通超链接。

### 4.3 link-click.ts 协议分发

统一链接点击插件，根据 href 协议决定行为：

- `krig://note/` → 笔记导航（带历史栈）
- `krig://block/` → 笔记导航 + 滚动到目标 block（跨文档使用 pendingAnchor 机制）
- `file://` → 解析路径后 `shell.openPath`
- `media://` → `mediaResolvePath` + `shell.openPath`
- `http(s)://` → `shell.openExternal`

### 4.4 LinkPanel Tab 分区

FloatingToolbar 🔗 链接面板重构为三 Tab：

| Tab | 功能 |
|-----|------|
| 📄 笔记 | 搜索笔记列表 + 点击 ▶ 展开标题二级列表；搜索框也接受直接粘贴任意协议链接 |
| 📎 文件 | 系统文件选择器 + 链接到原文件（`file://`）/ 导入到笔记（`media://`）选项 |
| 🔗 网页 | URL 输入框 |

### 4.5 Block Handle 复制链接

HandleMenu 新增 🔗 Copy Link 菜单项，复制 `krig://block/{noteId}/{anchor}` 到剪贴板：

- 标题 block → 锚点为标题文本（最长 60 字符）
- 普通 block → 锚点为 `索引:前缀30字符`

## 五、验收标准

| # | 标准 |
|---|------|
| 1 | 段落中的 `[file.py](media://files/...)` 渲染为可点击的 📎 inline chip |
| 2 | 点击 chip / link mark 能用系统默认应用打开对应文件 |
| 3 | 不影响普通 `[text](https://...)` 超链接的渲染 |
| 4 | 块级 `!attach[...]` FileBlock 行为不变 |
| 5 | ChatGPT 提取的行内 sandbox 文件引用输出 `[filename](media://...)` 并正确渲染 |
| 6 | FloatingToolbar 🔗 面板分为笔记/文件/网页三个 Tab |
| 7 | 笔记 Tab 支持展开标题列表，点击插入 `krig://block/` 段落链接 |
| 8 | 笔记 Tab 搜索框接受粘贴 `krig://`、`https://`、`media://`、`file://` 链接并直接应用 |
| 9 | 文件 Tab 支持系统文件选择器 + 链接/导入两种模式 |
| 10 | HandleMenu 🔗 Copy Link 复制 `krig://block/` 链接到剪贴板 |
| 11 | `krig://block/` 链接支持跨文档导航 + 滚动定位 + 高亮闪烁 |
| 12 | 右键菜单自动检测光标处所有格式 mark 并提供移除选项 |
| 13 | `removeLink` 在光标模式（无选区）下正确移除完整 link mark |
