## 二、开发阶段

### Stage 1：ChatGPT 提取验证

**目标**：验证通过注入脚本 fetch 获取 ChatGPT 对话数据的完整链路。

#### 1.1 验证 Bearer token 获取

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 1.1.1 | 在 ChatGPT 页面注入 `fetch('/api/auth/session')` | 获取 accessToken | 返回有效 JWT token |
| 1.1.2 | 用 Bearer token fetch conversation API | `fetch('/backend-api/conversation/{uuid}')` | 返回完整 mapping 树 JSON |
| 1.1.3 | 验证 token 有效期和刷新 | 长时间使用后 token 是否过期 | 记录 token 有效时长 |

#### 1.2 对话数据解析

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 1.2.1 | mapping 树遍历 | 从当前路径（非分支）提取线性消息序列 | 消息顺序和 ChatGPT 页面一致 |
| 1.2.2 | parts 解析 | 处理 string / asset_pointer / widget 等类型 | 所有 parts 类型都有对应处理 |
| 1.2.3 | turn 合并 | user + tool[] + assistant 合并为一个 turn | 多 tool 调用的消息正确合并 |
| 1.2.4 | hidden 消息过滤 | 跳过 `is_visually_hidden_from_conversation` | 和页面显示一致 |

#### 1.3 内容提取

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 1.3.1 | Markdown 文本 | 从 parts 拼接 | 格式完整，含加粗/列表/标题 |
| 1.3.2 | LaTeX 公式 | `\(...\)` / `$...$` / widget 三种形式 | 所有形式正确转换 |
| 1.3.3 | 代码块 | fenced code block | 语言标注保留 |
| 1.3.4 | DALL·E 图片 | `sediment://file_xxx` → estuary API | 图片 base64 正确获取 |
| 1.3.5 | Code Interpreter 图片 | `aggregate_result.messages[].image_url` | 图片正确提取 |
| 1.3.6 | Canvas 文档 | `/textdocs` API | 内容完整提取 |
| 1.3.7 | 用户上传文件 | `attachments[]` | 文件引用记录 |

#### 1.4 Note 导入验证

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 1.4.1 | 右键单条提取 | 点击 ChatGPT 回复 → 提取到 Note | 文本 + 图片正确导入 |
| 1.4.2 | 整页提取 | 提取整个对话 | 所有 turn 按顺序导入 |
| 1.4.3 | 图片渲染 | DALL·E / Code Interpreter 图片 | 在 Note 中正确显示 |

### Stage 2：Gemini 提取验证

**目标**：验证通过注入脚本 fetch 获取 Gemini 对话数据的完整链路。

#### 2.1 验证 HttpOnly cookie fetch

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 2.1.1 | 在 Gemini 页面注入 fetch batchexecute | `credentials: 'include'` + `X-Same-Domain: 1` | 返回有效 hNvQHb 响应 |
| 2.1.2 | 构造 batchexecute POST body | 正确的请求参数格式 | 和 CDP 捕获的请求体一致 |
| 2.1.3 | 如果 fetch 失败，记录原因 | CSP / cookie 问题 / 请求格式错误 | 有明确的错误信息 |

#### 2.2 对话数据解析

| # | 任务 | 说明 | 验收标准 |
|---|------|------|---------|
| 2.2.1 | 多帧流解析 | 剥离 XSRF 前缀 + 贪心帧分割 + 双重 JSON 解析 | 所有帧正确解析 |

| 2.2.3 | schema 路径验证 | 当前路径是否仍然有效 | 文本/图片/grounding 路径都能命中 |

## 五、风险与应对

| 风险 | 可能性 | 影响 | 应对 |
|------|--------|------|------|
| ChatGPT Bearer token fetch 返回 401 | 低 | Stage 1 阻塞 | 退回 CDP 模式（现有提取器已可用） |
| Gemini HttpOnly cookie fetch 被 CSP 阻止 | 中 | Stage 2 阻塞 | 退回 CDP 模式（现有提取器已可用） |
| Gemini batchexecute POST body 构造错误 | 中 | Stage 2.1 延迟 | 从 CDP 捕获的请求中复制格式 |
| Gemini schema 路径变化 | 低 | Stage 2.2 延迟 | 现有侦察脚本可快速重新定位 |
| ChatGPT 分支对话遍历错误 | 中 | Stage 1.2 延迟 | 先取当前路径，忽略分支 |
| 图片 URL 过期（Imagen） | 高 | Stage 2.3.4 | 立即 base64 内联，不存 URL |
| 三平台抽象过度 | 中 | Stage 3 复杂度 | 只抽象验证通过的公共部分 |
