## B. 需求二：一年期免费授权 grant（新增，请实现）

### B1. 要解决的产品问题
测试期所有功能全免费，但要求注册。**目的是为未来计费兜底**：怕用户装了第一版就不再更新、永久白嫖。所以「免费一年」的到期时间**必须由后端按账号记录**（不能记在客户端本地——否则用户改本机时钟/不联网就绕过了，新策略也推不到他）。一年后后端把 grant 失效，客户端自动进入计费判定，**与客户端版本无关**。

### B2. 请实现
1. **发放**：用户**注册成功时**（或首次登录时），自动为该账号发放一个 KRIG-Note 的**一年期免费授权**记录，含到期时间 `expiresAt`。
   - 已注册过的老用户：若还没有该 grant，首次登录时补发（或按你们更顺的时机）。
2. **查询接口**：客户端登录后查询「我这个账号的免费授权状态」。返回结构建议：
   ```json
   {
     "app": "krig-note",
     "grant": {
       "active": true,
       "expiresAt": 1830000000000,
       "issuedAt": 1798464000000
     }
   }
   ```
   - 可以是**独立接口**（如 `GET /api/.../krig-note/free-grant`），
   - **或**并入已有的 `GET /api/v1/me`（在返回里加一段 grant 概览，客户端从中筛）。哪种对你们更省就哪种，告诉我们路径即可。
3. **失效语义**：`now > expiresAt` 时 `active=false`（读时计算，参照订阅的 `effectiveStatus()` 范式）。grant 失效后，客户端改走订阅判定（free/pro）——**那是下一期的事，本期不涉及**，你们现在只要保证 grant 到期后 `active` 变 false 即可。

### B3. ⚠️ 希望（不强制，但很有价值）：到期时间带签名
若 `expiresAt` 能**带服务端签名**（客户端可验签），则客户端离线兜底时也无法靠改本机时钟伪造「还没到期」。
- 能做：客户端会验签后才采信离线缓存的到期时间。
- 不能做：客户端就只在「联网查到的 active」为准，离线兜底体验稍弱但不影响安全（联网一次即纠正）。
- 请告知能否支持，影响客户端离线策略的实现。

### B4. 测试期参数
- 免费期时长：**一年**（请做成后端可配置，方便日后延长/调整，不用改客户端）。
- 是否给所有 `app_source=krig-note` 的新账号无条件发放：**是**（Freemium，无 trial 门槛）。

## C. 客户端会怎么用（便于你们对齐）

```
注册/登录(A) ──▶ 拿 JWT ──▶ 查 grant(B2) 
   grant.active=true  → 客户端解锁全功能,显示「测试版·免费授权剩余 N 天」
   grant.active=false → (下一期)改读订阅判 free/pro
```

- 客户端只在「启动 / 恢复前台 / 注册登录完成」时查 grant，不轮询。
- 全部走 portal 相对路径，不直连独立 IP/端口。
