# eBook 模块评估（2026-04-09）

## 范围
- Renderer（L3 View）：`src/plugins/ebook/components/EBookView.tsx` + 子组件/钩子（Fixed/Reflowable/Toolbar/Search/Outline）
- 主进程存储/加载：`src/main/ebook/file-loader.ts`、`src/main/ebook/bookshelf-store.ts`、`src/main/ebook/annotation-store.ts`
- 参考原则：分层设计、模块自包含、插件化、UI 配置化、命名/可描述性。

## 发现（按严重度排序）
1) **UI 逻辑与渲染/样式耦合，缺少主题化**  
   - `EBookView` 内联大量 UI 状态与样式控制（fitWidth 计算、annotation picker 绝对定位、loading/empty 文案），`ebook.css` 为单一主题，无 tokens/配置。  
   - 影响：难以换肤或复用，违反“UI 表现层配置化”。  

2) **插件/框架分离不足：提取逻辑嵌入 View**  
   - `handleExtract` 直接调用 `(viewAPI as any).extractionOpen()` 并在视图层处理结果，属于跨层业务；违背“框架与插件分离/模块自包含”。应通过协议/服务抽象或右槽路由调用。  

3) **状态恢复与 Workspace 同步不完整**  
   - EBookView 使用本地 state + `viewAPI.ebookRestore()`，但 WorkspaceState 未包含 ebook 相关字段（此前 Workspace 评估已指出）；切换 Workspace/重启可能丢失侧边栏开关、注释模式等 UI 状态。  

4) **可访问性与键盘支持不足**  
   - 搜索/书签快捷键仅 Cmd+F/D；按钮缺 aria-label；annotation 颜色选取仅鼠标。  

5) **数据层约束与错误处理薄弱**  
   - file-loader/bookshelf/annotation-store 未检查文件存在性/类型冲突的错误反馈路径；EBookView 加载失败仅 console.error。  

6) **渲染职责集中于单文件，缺少分层/可测试性**  
   - EBookView 同时管理渲染器创建、加载/恢复、进度保存、键盘监听、提取、UI 控制，文件 >400 行，缺少 service/hooks 分层（部分 hooks 已有，但核心装配仍在同文件）。  

## 改进建议
- 主题化：建立 ebook 主题 tokens（背景/文本/边框/按钮/overlay/annot picker 尺寸与色板），迁移 inline style 到 tokens/CSS 变量。 
- 提取/跨层：将 PDF 提取逻辑封装为 service 或协议消息，View 仅触发命令/路由。 
- 状态同步：将书籍状态（activeBookId、fitWidth、scale、sidebarOpen、annotationMode、restorePage/CFI）纳入 WorkspaceState 或专用存储，切换 Workspace 可恢复。 
- 可访问性：为 Toolbar/搜索/标注按钮增加 aria-label + 键盘快捷键；标注颜色提供键盘循环选择。 
- 错误与健壮性：加载/导入失败显示 UI 提示；文件不存在/类型不支持时的回退；bookshelf 操作添加冲突检查。 
- 结构化分层：
  - View 负责 UI 组装与事件分发；
  - hooks/service 负责数据/状态（book load/save、进度、提取）；
  - renderer factory/adapter 负责 fileType → renderer 创建与生命周期；
  - overlay/annotation picker 独立组件。
