# WebView 模块评估（2026-04-09）

## 范围
- WebView 组件：`src/plugins/web/components/WebView.tsx`
- Toolbar：`src/plugins/web/components/WebToolbar.tsx`
- 存储：`src/plugins/web/main/bookmark-store.ts`、`src/plugins/web/main/history-store.ts`
- 参考原则：分层设计、模块自包含、插件化、UI 配置化、命名/可描述性、安全性。

## 发现与代码证据（按严重度排序）
1) **UI 主题/样式硬编码**  
   - `web.css` + inline 样式（书签星标颜色、按钮图标字符），无主题 tokens；难以换肤或一致化。  

2) **业务逻辑与 View 直接耦合（静默吞没异常）**  
   - 位置：`WebView.tsx` **第62行** `viewAPI.webHistoryAdd().catch(() => {})`。  
   - 问题：直接调用底层 API 时遇到异常直接静默，没有任何界面或 service 层的隔离来向用户抛出问题。  

3) **安全性裸奔模式**  
   - 位置：`WebView.tsx` **第141行** `allowpopups={'true' as any}`。  
   - 问题：完全放开了弹出式窗口而不进行域白名单拦截或审计；另外 URL 输入也未校验潜在危险协议（如 file:// 等）。  

4) **状态同步/恢复缺失**  
   - 当前 URL/标签页状态未与 WorkspaceState 或 session 对接；切换 Workspace 后 WebView 不会恢复到上次页面。  

5) **可访问性能力缺失**  
   - 按钮缺 aria-label；URL 输入未提供确认/撤销提示；快捷键仅在 title 文案中隐含。  

6) **日用浏览器基础能力缺失（功能性）**  
   - 单标签限制不能并行浏览；无痕/隐身策略及Cookie管理缺失。缺少 find-in-page 和安全域锁标提示等。

## 改进建议
- **搭建错误边界与统一处理（Error Boundary）**：为 WebView 及其调用的 viewAPI 配置 Service Adapter，保证请求失败或存储冲突能抛出友好的 UI 错误提醒（如 Toast/Dialog），而不是 catch 空处理。
- **落实主进程级安全拦截屏障**：必须针对 `allowpopups` 进行 URL 信任审计。在主进程中拦截来自该 partition 的 window open 请求，限制到受信任源域。
- 主题化：提取颜色/间距/圆角/字号到 tokens 或 CSS 变量，统一按钮图标。 
- 状态恢复：将当前 URL、历史栈等纳入 WorkspaceState 或独立 session。
