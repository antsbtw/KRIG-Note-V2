# NoteView 组件评估（2026-04-09）

## 范围
- 文件：`src/plugins/note/components/NoteView.tsx`
- 相关契约：`viewAPI`（`src/main/preload/view.ts`）、NoteStore IPC（`IPC.NOTE_*`）
- 参考原则：分层设计、模块自包含、UI 表现层配置化、命名/可描述性、存储格式。

## 发现（按严重度排序）
1) **视觉样式内联硬编码，缺少主题化**  
   - 位置：`styles` 对象包含颜色、边框、圆角、字号。  
   - 影响：无法通过统一主题/配置切换外观，违背“UI 表现层配置化”。  

2) **导入/保存入口未与存储格式契合且缺少错误反馈**  
   - Toolbar 的 “Import JSON” 直接读文件并派发 `note:import-json`，未校验结构或提示失败原因；保存按钮仅触发 `note:save`，无加载/禁用状态。  

3) **导航历史耦合外部插件且无错误处理**  
   - back/forward 调用来自 `../plugins/link-click`，若未初始化或抛错 UI 无提示；navState 初始值固定 false，未在挂载时同步。  

4) **可访问性与命名不足**  
   - 按钮无 aria-label，标题截断无 tooltip，closeSlot 按钮仅用符号“×”。  

5) **状态同步缺口**  
   - 仅在 note 打开时 setTitle，未监听 activeNoteId 丢失/关闭时清空；dirty 状态由全局事件驱动，缺少来源标识与防抖。  

## 改进建议
- 提取样式到共享主题（CSS 变量或 `theme.ts`），按钮/背景/边框复用同一视觉源。 
- Import/Save：对导入 JSON 做 schema 验证和错误提示；保存时显示进行中/失败反馈。 
- 在挂载时同步 navState（调用 canGoBack/Forward），并为导航操作添加 try/catch + 用户提示。 
- 补充 aria-label、tooltip，关闭按钮文本化（如 “Close”）。 
- 监听 activeNoteId 变化清理标题；dirty 事件增加 detail/来源校验或节流。
