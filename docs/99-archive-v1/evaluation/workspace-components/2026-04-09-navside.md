# NavSide 组件评估（2026-04-09）

## 范围
- 文件：`src/renderer/navside/NavSide.tsx`
- 相关契约：`navSideAPI`（由 `src/main/preload/navside.ts` 暴露）、`NavSideRegistration` & `WorkModeRegistration`（`src/shared/types.ts`）。
- 参考原则：分层设计、模块自包含、框架与插件分离、UI 表现层配置化、Workspace 是核心调度单元、命名/可描述性。

## 发现（按严重度排序）
1) **框架层硬编码插件实现（违反“框架与插件分离 / 单向依赖”）**  
   - 位置：顶层 `import { WebPanel } from '../../plugins/web/navside/WebPanel';`。NavSide 直接依赖具体插件组件，而非通过注册表/工厂。  
   - 影响：contentType 扩展需改框架代码，破坏可替换性和单向依赖。  

2) **单文件职责过载，耦合数据/状态/渲染（违反“分层设计”“模块自包含”）**  
   - 近 900 行同时处理：WorkMode 切换、笔记/文件夹 CRUD、拖放、多选、上下文菜单、3 类 contentType 渲染。  
   - 影响：难以替换某个面板或数据源，缺少清晰接口契约；测试与维护成本高。  

3) **UI 视觉大量内联硬编码（违反“UI 表现层配置化”）**  
   - 样式对象包含颜色、间距、圆角、字体，缺少主题变量或共享样式源。  
   - 影响：无法通过主题切换或集中配置调整视觉。  

4) **Workspace 状态同步不完整**  
   - 仅将 `expandedFolders` 同步到 Workspace；`activeNoteId` 同步依赖 noteOpenInEditor，电子书书架的 `ebookExpandedFolders` 由 NavSide 自持状态但未写回 WorkspaceState；右侧 slot 状态缺席。  
   - 影响：切换 Workspace 或重启后部分状态丢失，违背“Workspace 驱动 NavSide 和 Slot 内容”。  

5) **数据适配与渲染耦合，缺少接口抽象**  
   - Note、Folder、EBook、Web 数据获取直接调用 navSideAPI；无 adapter 层，也无类型守卫/错误处理。  
   - 影响：切换数据源或后端时需要改 UI 代码；异常时组件静默失败。  

6) **交互一致性与可访问性不足**  
   - context menu、拖放高亮、键盘可访问性缺乏可描述性（无 aria/role）；搜索框只读且无占位逻辑实现，属于未完成功能。  

## 改进建议
- 引入 NavSide 面板注册/懒加载：插件注册 contentType → 组件工厂；NavSide 根据注册表渲染，移除对插件实现的 import。 
- 拆分模块：
  - 状态与数据层（note/folder/ebook/web adapters + hooks）
  - 交互状态（selection/drag/drop/context menu）
  - 纯视图组件（列表、ActionBar、Search、书架/Web 面板容器）
- 抽离样式到主题/变量，减少 inline style。 
- 完善 WorkspaceState 同步：activeNoteId、expandedFolders、activeBookId、ebookExpandedFolders、right slot 信息统一读写。 
- 为 API 调用添加错误处理与 loading/empty 态；对 navSideAPI 返回值做 schema 校验。 
- 补充可访问性：aria-label/role/keyboard 支持；搜索框实现或隐藏。
